## 1. 产品概述

### 1.1 背景与痛点

作为 A 股散户，你当前面临的核心痛点：

1. **凭感觉交易**：没有经过验证的交易规则，追涨杀跌、频繁操作，导致长期收益跑不赢大盘

2. **策略无法验证**：想到一个交易想法，不知道过去历史上是否有效，不敢直接实盘试错

3. **情绪干扰决策**：盘中受市场情绪影响，明明制定了计划却临时变卦，错过止盈止损点

4. **信息处理效率低**：全市场 5000 多只股票，手动筛选标的效率极低，无法快速找到符合条件的机会

### 1.2 产品目标

我们要打造一套**个人自用的规则化量化辅助系统**，核心目标是：

- 把 10 年 A 股实战经验程序化，用简单可解释的规则 + 因子替代主观判断

- 不引入复杂的机器学习 / 深度学习，所有策略逻辑透明可解释，符合散户的交易认知

- 实现 “数据获取→指标计算→策略验证→信号提醒→风控执行” 的完整闭环

- 帮助你实现科学炒股，告别瞎买瞎卖，长期获得稳定可复制的超额收益

### 1.3 产品定位

这是一套**个人低频量化辅助决策工具**，聚焦日 / 周级别的中低频交易策略，优先覆盖保守型、趋势型的散户常用策略，不做高频、套利等复杂策略，优先保证系统的稳定性、可解释性和易用性。

## 2. 核心功能需求

### 2.1 数据管理模块（Java 侧实现）

#### 2.1.1 数据获取

Java 后端通过 Tushare Pro REST API 直接获取全市场 A 股数据，覆盖以下数据类型：

|数据类型|说明|更新频率|
|---|---|---|
|行情数据|个股 / 指数的日 K 线数据（开盘 / 收盘 / 最高 / 最低 / 成交量 / 成交额）|每日收盘后|
|标的基础信息|股票代码、名称、行业、是否 ST、是否退市、上市日期|每日|
|财务数据|季度 / 年度财报数据：净利润、ROE、资产负债率、营收增速等|财报发布后|
|资金情绪数据|北向资金每日净流入、龙虎榜上榜数据、换手率数据|每日|
|交易日历|A 股交易日、节假日信息，用于策略时间对齐|每月更新|

**数据源选型**：

- 主数据源：`Tushare Pro`，数据标准化程度高、接口稳定可靠，提供标准 REST API，Java 后端可直接调用获取数据（免费版积分足够满足基础需求）

- 调用约束：严格控制请求频率，遵守 Tushare 的接口调用规则，避免触发频率限制，所有数据仅用于个人研究使用

#### 2.1.2 数据清洗与预处理

- 数据清洗在 Java 后端完成，Tushare 返回的数据标准化程度高，清洗工作量较小
- 自动处理复权：对历史 K 线数据进行前复权处理，保证价格的连续性，避免除权除息导致的指标失真

- 异常值处理：自动剔除停牌、涨跌停无法交易的异常数据，标记 ST、*ST、退市风险标的

- 数据对齐：将不同来源的数据按交易日、标的代码进行对齐，保证策略计算时数据的一致性

- 缓存机制：本地缓存已获取的数据，避免重复请求，提升数据访问速度

#### 2.1.3 数据存储

- 本地存储结构化数据，支持快速查询

- MVP 阶段使用 SQLite 即可满足个人单机需求，后续可扩展为 PostgreSQL

- 数据按时间分区存储，方便历史数据的快速检索

### 2.2 指标计算模块（Python 侧实现）

Java 后端通过 Tushare 获取原始行情数据后，调用 Python 计算服务完成技术指标与科学指标的计算。Python 服务专注于量化计算，不负责数据获取，分为三大类：

#### 2.2.1 技术面指标（趋势 / 择时用）

所有技术指标都是老股民日常看盘的常用指标，逻辑完全透明：

|指标|说明|常用参数|
|---|---|---|
|均线（MA）|5 日 / 10 日 / 20 日 / 60 日 / 120 日 / 250 日均线，用于判断趋势|5,20,60|
|MACD|指数平滑异同移动平均线，用于判断趋势转折|12,26,9<br>|
|KDJ|随机指标，用于判断超买超卖|9,3,3|
|RSI|相对强弱指标，判断市场强弱|14|
|BOLL|布林带，判断价格波动区间|20|
|成交量均线|成交量的 5 日 / 20 日均线，判断量能变化|5,20|

#### 2.2.2 基本面指标（选股用）

用于筛选优质、低估的公司，是中长期持股的核心依据：

|指标|说明|
|---|---|
|PE-TTM|滚动市盈率，剔除一次性损益，反映当前估值水平|
|PB|市净率，衡量资产折价程度，适合周期、银行类行业|
|ROE|净资产收益率，衡量公司盈利能力，核心质量指标|
|净利润增速|归母净利润同比增速，反映公司成长能力|
|资产负债率|衡量公司财务健康度，排除高负债风险标的|
|格雷厄姆安全边际指标|结合流动比率、盈利稳定性的综合安全指标|

#### 2.2.3 情绪面指标（择时 / 过滤用）

A 股散户非常看重的市场情绪指标，用来判断资金的动向：

|指标|说明|
|---|---|
|换手率|个股的日换手率，反映标的的活跃程度|
|北向资金净流入|北向资金当日 / 近 3 日 / 近 5 日的净流入金额，判断外资动向|
|龙虎榜上榜|个股是否登上龙虎榜，反映短期资金关注度|
|成交额排名|个股全市场成交额排名，过滤流动性不足的标的|

### 2.3 策略管理模块

这是系统的核心，我们支持**可视化的规则配置**，你可以通过组合不同的因子和规则，搭建自己的交易策略，所有规则都是可解释的，没有黑箱。

#### 2.3.1 风险前置过滤规则

所有策略默认先执行风险过滤，这是老股民的经验总结，先把不能碰的标的排除掉：

1. 自动剔除：ST/*ST 股、退市整理股、停牌股

2. 自动剔除：上市不满 30 天的次新股（避免次新股波动过大）

3. 自动剔除：近 20 日平均成交额低于 5000 万的标的（避免流动性风险，买了卖不出）

4. 可选剔除：科创板 / 北交所 / 创业板标的（如果你只做主板的话）

#### 2.3.2 策略规则配置

支持你通过组合 “条件” 来构建策略，条件支持：

- 指标比较：比如`PE-TTM < 20`、`5日均线 > 20日均线`、`北向资金近3日净流入 > 0`

- 多条件组合：支持 AND/OR 逻辑，比如`(MACD金叉) AND (换手率 > 3%) AND (ROE > 15%)`

- 规则排序：支持按指标排序选股，比如选符合条件的标的中，ROE 最高的前 20 只

#### 2.3.3 内置策略模板

我们内置了经过 A 股市场验证的、散户常用的经典策略模板，你可以直接使用，也可以在此基础上修改：

|策略模板|适用场景|核心规则|
|---|---|---|
|双均线趋势策略（新手首选）|中长线趋势交易，适合 ETF / 龙头股|1. 5 日均线上穿 20 日均线（金叉）→ 买入<br>2. 5 日均线下穿 20 日均线（死叉）→ 卖出<br>3. 调仓频率：日度，持仓最多 10 只标的|
|低 PE 价值策略|中长期价值投资，适合消费 / 医药行业|1. 选股条件：PE-TTM <行业平均、PB < 2、ROE> 15%<br>2. 每月调仓一次，持有符合条件的最低估的 30 只标的|
|MACD 短线趋势策略|短线波段交易|1. 买入条件：MACD 金叉、成交量放大 1.5 倍、股价突破 20 日均线<br>2. 止损：跌破 20 日均线止损，止盈：盈利 10% 止盈|
|小市值动量策略|A 股特色小市值效应策略|1. 选股条件：流通市值 50-300 亿、非 ST、近 1 月涨幅 > 5%<br>2. 周度调仓，持有前 20 只动量最强的标的|
|北向资金跟随策略|跟随外资动向的保守策略|1. 买入条件：北向资金连续 3 日净流入、个股被北向加仓<br>2. 持有直到北向资金连续 2 日减仓卖出|

### 2.4 策略回测模块

在你把策略用到实盘之前，先要用历史数据验证这个策略过去是否有效，避免实盘试错。我们的回测完全基于规则，不用复杂模型，结果透明可解释。

#### 2.4.1 回测配置

- 支持选择回测的时间范围：建议至少选择 3 年以上的历史数据，覆盖牛熊周期

- 支持选择基准：比如沪深 300 指数，用来对比策略是否跑赢大盘

- 支持配置交易成本：默认模拟 A 股真实交易成本，包括佣金（万 2.5）、印花税（卖出千 1）、过户费

- 支持配置初始资金：默认 10 万，模拟你的真实账户资金

#### 2.4.2 回测执行流程

1. 按时间顺序遍历历史每一天的行情数据

2. 每天收盘后，根据你的策略规则，计算当天的目标持仓

3. 模拟真实交易：考虑 T+1 规则、涨跌停无法成交、最小交易单位（100 股 / 手）

4. 记录每天的账户净值、持仓变化、交易记录

#### 2.4.3 回测报告

回测完成后，生成完整的回测报告，帮你判断这个策略是否靠谱：

1. **收益类指标**：

    - 总收益率：回测周期内的总收益

    - 年化收益率：复利计算的年化收益，15%-25% 属于优秀策略，超过 30% 要警惕过拟合

    - 超额收益：策略收益相对基准（沪深 300）的超额收益

2. **风险类指标**：

    - 最大回撤：账户从最高点到最低点的最大跌幅，反映策略的风险，越小越好，一般要求不超过 20%

    - 夏普比率：单位风险带来的收益，大于 1.5 属于优秀，大于 2 属于非常优秀

    - 胜率：盈利交易的占比

    - 盈亏比：平均盈利 / 平均亏损，一般大于 2 属于不错的策略

3. **可视化图表**：

    - 策略净值曲线 vs 基准净值曲线，直观看到策略的收益情况

    - 最大回撤区间标记，让你看到策略在什么行情下会亏钱

4. **交易记录**：完整的历史买卖记录，你可以复盘每一笔交易的逻辑

#### 2.4.4 回测注意事项

系统内置防过拟合提醒：

- 如果你回测的参数过多、规则过于复杂，会提醒你可能存在过拟合

- 建议你用样本外数据做验证：比如用 2019-2022 年的数据做回测，用 2023-2024 年的数据做验证，看策略是否依然有效

### 2.5 信号服务与提醒模块

当策略验证通过后，就可以上线实盘使用了，系统会自动帮你监控市场，当出现交易信号的时候，及时提醒你。

#### 2.5.1 信号生成

- 每日收盘后，Java 后端从 Tushare 获取当日行情数据，结合 Python 计算服务生成的指标，自动运行策略生成当日的交易信号：

    - 买入信号：哪些标的符合你的买入条件，建议买入的仓位

    - 卖出信号：哪些持仓标的符合你的卖出条件，建议卖出

    - 目标持仓：根据策略，你当前应该持有的标的和仓位

- 每个信号都会附带原因解释：比如 “买入贵州茅台，原因：MACD 金叉，北向资金加仓，PE 处于历史低位”，完全透明

#### 2.5.2 信号推送

支持多种提醒方式，确保你不会错过信号：

- 企业微信 / 钉钉推送：把信号推送到你的聊天软件，盘中盘后都能及时收到

- 邮件推送：把每日的信号报告发送到你的邮箱，方便复盘

- 网页端提醒：Java 业务系统的首页展示当日的信号，你打开就能看到

### 2.6 模拟交易模块

在你实盘之前，你可以先用模拟交易测试策略的实盘表现，避免直接上实盘踩坑。

#### 2.6.1 模拟撮合

- 完全模拟 A 股的真实交易规则：T+1、涨跌停限制、最小交易单位

- 用真实的行情数据做模拟撮合，和实盘的成交情况几乎一致

- 支持模拟不同的资金量，测试策略的资金容量

#### 2.6.2 模拟账户

- 模拟账户的资金、持仓、收益完全和实盘一样

- 记录所有的模拟交易记录，方便你复盘

- 每日生成模拟账户的收益报告，让你看到策略的实盘表现

### 2.7 实盘辅助与风控模块

当你准备实盘交易的时候，系统会帮你做风控，避免你犯错误，同时帮你规划交易订单。

#### 2.7.1 风控检查

所有交易信号在执行之前，都会先经过风控检查，这是系统的最后一道防线：

1. **仓位风控**：

    - 单票最大仓位：个股持仓不超过总资金的 8%，避免黑天鹅风险

    - 总仓位限制：根据市场情况，支持设置最大仓位，比如震荡市不超过 80% 仓位

2. **亏损风控**：

    - 单日最大亏损：单日亏损超过 1%，停止开新仓，避免情绪化加仓

    - 最大回撤熔断：账户回撤超过 8%，自动触发降仓，把风险控制住

3. **交易约束**：

    - 自动检查 T+1 规则：当天买入的股票不能当天卖出

    - 自动检查涨跌停：涨跌停的标的不建议交易，避免无法成交

    - 自动检查流动性：避免交易流动性不足的标的

#### 2.7.2 订单规划

根据策略生成的目标仓位，自动帮你计算需要买卖的股数：

- 自动对齐最小交易单位（100 股 / 手）

- 自动计算可用资金，确保不会超买

- 如果你设置了换手限制，会自动拆分订单，避免单日换手过大

#### 2.7.3 实盘对接（MVP 后扩展）

后续可以对接券商的 API，实现半自动化交易：

- 自动获取你的账户持仓、资金情况

- 自动生成下单指令，你确认后就可以直接下单

- 自动对账，同步券商的订单状态和持仓情况

## 4. 风险说明

作为老股民，必须要提醒你，量化不是万能的，有这些风险你需要知道：

1. **回测不等于未来**：历史有效不代表未来一定有效，市场是会变化的，不要过度迷信回测结果

2. **过拟合风险**：不要为了回测结果去过度优化参数，那样的策略实盘一定会翻车

3. **市场风险**：A 股市场波动大，再好的策略也会有回撤，要做好仓位控制，不要加杠杆

4. **黑天鹅风险**：比如突发的政策变化、退市风险，系统的风控只能防范常规风险，无法防范极端黑天鹅

5. **合规风险**：所有数据仅用于个人研究，不要用于商用，遵守数据源的使用规则
