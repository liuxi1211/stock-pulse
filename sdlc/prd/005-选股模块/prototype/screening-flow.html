3. 规则树结构详解（Rule Tree）

规则树支持三种节点类型：逻辑节点（AND/OR组合）、比较节点（条件判断）、表达式节点（静态值 / 因子引用 / 算术运算）。

Logic 节点（逻辑组合）
  • type: "logic"
  • operator: "AND" | "OR"
  • children: RuleNode[]
  • → 可以无限嵌套，实现复杂逻辑组合
Compare 节点（比较判断）
  • type: "compare"
  • left: ExprNode
  • comparator: ">" | "<" | ">=" | "<=" | "==" | "cross_up" | "cross_down"
  • right: ExprNode
  • → cross_up/cross_down 需要前一日数据 → 触发自 JOIN
Expr 节点（表达式）
  • value: BigDecimal ← 静态值
  • factor: String + params: {} + outputIndex?: number ← 因子引用
  • op: "*" | "+" | "-" | "/" + left/right: ExprNode ← 算术运算
  • → 三者互斥，一个 ExprNode 只能是其中一种
示例：复杂规则树及其 JSON

树形结构：

AND（顶层）
├─ MA(5) cross_up MA(20)
├─ VOLUME > VOL_MA(20) * 1.5
└─ OR
   ├─ RSI(14) > 70
   └─ KDJ(J) > 80

对应的 JSON：

{ "rules": [ { "type": "compare", "left": { "factor": "MA", "params": {"timeperiod": 5} }, "comparator": "cross_up", "right": { "factor": "MA", "params": {"timeperiod": 20} } }, { "type": "compare", "left": { "factor": "VOLUME" }, "comparator": ">", "right": { "op": "*", "left": { "factor": "VOL_MA", "params": {"timeperiod": 20} }, "right": { "value": 1.5 } } }, { "type": "logic", "operator": "OR", "children": [ { "type": "compare", "left": { "factor": "RSI", "params": {"timeperiod": 14} }, "comparator": ">", "right": { "value": 70 } }, { "type": "compare", "left": { "factor": "KDJ", "outputIndex": 2 }, "comparator": ">", "right": { "value": 80 } } ] } ], "option": { ... } }
4. ConditionSqlBuilder 工作原理

ConditionSqlBuilder 是本模块的核心翻译引擎。它通过递归遍历规则树，将每个节点翻译为对应的 SQL 片段，最终拼接成完整的 SELECT 查询。

扫描阶段

先整棵树扫描一次，检查是否有 cross_up / cross_down。如果有，则需要 LEFT JOIN stock_indicator_daily AS prev 自关联获取前一日数据。

递归翻译阶段

buildCondition()：处理 logic/compare 节点
buildExpression()：处理 expr 节点
FactorColumnMapper：因子 → 列名映射

附加前置过滤

根据 option 附加：
• NOT LIKE '%ST%'（剔ST）
• list_date <= ...（剔新股）
• close * volume >= ...（成交额门槛）

执行

JdbcTemplate.query() 执行查询
典型节点翻译对照表
规则树节点 翻译后的 SQL 片段 备注
factor: "MA", params: {timeperiod: 5} sid.ma_5 因子映射
factor: "MACD", outputIndex: 0 sid.macd_dif 多输出因子
value: 1.5 1.5 静态值
op: "*", left: VOL_MA(20), right: 1.5 (sid.vol_ma_20 * 1.5) 算术运算
compare: MA(5) > MA(20) (sid.ma_5 > sid.ma_20) 简单比较
compare: MA(5) cross_up MA(20) (sid.ma_5 > sid.ma_20 AND prev.ma_5 <= prev.ma_20) ⚠ 需要自JOIN
logic: AND(cond1, cond2, cond3) (cond1 AND cond2 AND cond3) 逻辑组合
factor: "MA", params: {timeperiod: 8} sid.ma_10 + warning ⚠ 参数降级
注意：ConditionSqlBuilder 会自动收集所有被引用的因子列（如 ma_5、ma_20、vol_ma_20 等），并在 WHERE 中附加 AND sid.ma_5 IS NOT NULL AND sid.ma_20 IS NOT NULL ...，确保 NULL 值不会参与比较导致意外结果。

6. 因子 → 数据库列名映射表

由 FactorColumnMapper 实现（002-因子库模块负责维护，005-选股模块直接调用）。以下是标准参数对照表：

factor params outputIndex SQLite 列名 备注
MA {timeperiod: 5|10|20|60|120|250} - ma_5 / ma_10 / ma_20 / ma_60 / ma_120 / ma_250 简单移动平均线
EMA {timeperiod: 5|10|20|60} - ema_5 / ema_10 / ema_20 / ema_60 指数移动平均
MACD -（标准参数） 0|1|2 macd_dif / macd_dea / macd_hist 0=DIF(差值), 1=DEA(信号), 2=HIST(柱值)
KDJ -（标准参数9,3,3） 0|1|2 kdj_k / kdj_d / kdj_j 0=K值, 1=D值, 2=J值
RSI {timeperiod: 6|14|28} - rsi_6 / rsi_14 / rsi_28 相对强弱指数
SAR - - sar 抛物线转向
VOL_MA {timeperiod: 5|20|60} - vol_ma_5 / vol_ma_20 / vol_ma_60 成交量均线
CLOSE - - sid.close 收盘价（基础行情）
VOLUME - - sid.volume 成交量（基础行情）
HIGH / LOW - - sid.high / sid.low 最高/最低价
非标准参数降级规则：如 MA({timeperiod: 8}) 无对应列，将降级到最接近的标准参数（MA10），同时在响应的 warnings 字段中返回降级提示。如完全无法匹配（如 MA({timeperiod: 2})），抛出异常。
