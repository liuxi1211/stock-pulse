# 板块行情模块 PRD：行业板块视角

> **模块**：015 · 板块行情（申万一级行业单 Tab 独立菜单页）
> **状态**：规划中，待 `index_daily` 表（由 tushare `index_daily` 接口采集）落地后进入实施
> **来源**：拆分自 [`主要功能模块整体设计计划.md`](./../../.trae/documents/主要功能模块整体设计计划.md) §3.1
> **原则**：严格基于 tushare ≤2000 积分可用接口；不引入派生聚合数据；前端复用现有组件（StockApp/SearchSuggest/ChartsTheme/renderKline）；本文仅做"需求级"定稿，**具体接口字段、错误码、前端交互稿待落地 spec 展开**。

---

## 0. 拆分说明（为什么单独立项）

总计划文档把 6 个功能模块（数据采集 / 个股诊断（017）/ 板块行情（015）/ 资金流向（016）/ 现有模块补齐（012/013/014）/ 联调）混排，本文把 **板块行情模块（015）** 单独拎出独立成稿，因为：

- **它是独立的菜单项**：路由 `/page/sector` 在侧边栏「主要功能」下第 4 项，与仪表盘/行情中心/自选股并列，是一个完整的独立视角（行业板块轮动），不依赖其他新模块的前端页面即可独立交付。
- **Tab 裁剪决策集中**：原双 Tab（行业板块 + 概念板块）砍掉概念板块（`concept` / `concept_detail` 接口不在 tushare 接口汇总表）；原「行业板块资金流向柱状图」砍掉（`moneyflow_dc` 不存在）；原「今日板块资金净流入 TOP 10」砍掉（同上）。所有裁剪理由需统一记录。
- **数据依赖清晰且最小**：仅依赖 1 张新表 `index_daily`（由数据采集任务交付）+ 2 张已有表 `sw_industry` / `sw_industry_member`（已有 Service，本模块只补 Controller 对前端暴露），不跨多个模块。
- **下游依赖明确**：仪表盘（012）的「板块概览快照」卡片依赖本模块的行业排行能力，本 PRD 需明确该卡片复用本模块 Controller 的方式。

> **不在本 PRD 范围**：资金流向页面（016）、个股诊断页面（017）、仪表盘补齐（012）--各自独立排期；本 PRD 只描述板块行情页面本身及其直接依赖的后端 Controller。

## 1. 背景与目标

### 1.1 业务背景

- **老股民最看重板块轮动视角**：A 股是结构性行情主导，老股民看盘第一步是「今日哪个行业领涨、哪个领跌」，板块轮动是选股的前置信号。当前 stock-pulse 没有独立的板块行情页面，用户只能跳到东方财富/同花顺看，跳出本系统。
- **申万一级行业是标准切分**：申万一级行业分类共 28 个（不含概念板块），是 A 股板块轮动的标准视角；老股民对"银行/非银/地产/有色/煤炭/医药/电子/计算机"等行业名有直觉，不需要概念板块（题材）视角。
- **现有 `sw_industry` / `sw_industry_member` 数据完整但未对前端暴露**：Service 完整（目前只给 engine 量化层用），本模块需新建 `SwIndustryController` 把行业树与成分股查询对前端 REST 暴露。
- **行业指数行情需新增 `index_daily` 表**：原后端无行业指数日线数据，需新增 `index_daily` 表（申万行业指数 + 大盘指数共用一张表，由 tushare `index_daily` 接口采集），本模块通过 `IndexDailyController` 查询行业指数日线用于 K 线图。

### 1.2 总体目标

- **目标一**：访问 `/page/sector` 看到单 Tab 完整页面（行业板块 Tab），顶部有板块轮动热度图，下方有行业涨跌排行表、成分股展开列表、行业板块 K 线图。
- **目标二**：行业涨跌排行表展示申万一级 28 个行业的「行业名 / 今日涨跌幅 / 成交额 / 领涨股 / 领跌股」，支持按涨跌幅/成交额排序，点击行业展开成分股列表（支持分页+搜索）。
- **目标三**：行业板块 K 线图渲染行业指数日线（基于 `index_daily`），含 MA5/10/20/60 主图叠加 + 成交量副图。
- **目标四**：顶部板块轮动热度图以网格图形式展示 28 个行业板块，每格一个行业，颜色深浅表示涨跌幅（红涨绿跌），鼠标悬浮显示行业详情。
- **质量准绳**：`.trae/rules/系统设计理念(用法建议).md` + A 股实战派认可（不接受"纸面能跑、实战不能用"）。

### 1.3 非目标（明确不做）

| 非目标项 | 原因 | 备注 |
|---|---|---|
| **概念板块 Tab** | `concept` / `concept_detail` 接口不在 `tushare 接口汇总.md` 中，不存在 | 页面由原双 Tab 砍为单 Tab（仅行业板块） |
| **行业板块资金流向柱状图** | `moneyflow_dc` 接口不存在 | 不引入派生聚合数据（数据单源约束） |
| **今日板块资金净流入 TOP 10** | `moneyflow_dc` 接口不存在 | 顶部全局区只保留板块轮动热度图 |
| **板块成分股的实时分时** | 不在本期范围 | 成分股列表用 `daily_quote` 最新收盘，不做日内分时 |
| **概念板块/题材/热度榜** | 同 `concept` 不存在 | 不做任何概念类视角 |
| **跨市场板块对比** | 不在本期范围 | 只看 A 股申万行业，不做港股/美股行业对比 |
| **板块内个股加权指数自算** | 数据单源约束 | 板块指数统一用 `index_daily` 已有的申万行业指数 ts_code，不自算 |

### 1.4 典型用户旅程

**旅程 1：看板块轮动找领涨行业**
开盘后访问 `/page/sector` -> 顶部热度图一眼看红色深浅找今日领涨行业 -> 下方排行表按涨降幅降序 -> 看到领涨行业的领涨股 -> 点击该行业展开成分股 -> 搜索框输入关键字过滤 -> 点击成分股跳转个股诊断

**旅程 2：看行业指数 K 线判断趋势**
行业排行表点击某行业 -> 右侧 K 线图加载该行业指数日线 -> 切 MA60 看长期趋势 -> 看成交量副图判断放量缩量 -> 决定是否进成分股找个股

**旅程 3：仪表盘下钻**
仪表盘（012）板块概览快照卡片展示行业 TOP 5 -> 点击「查看全部」-> 跳转 `/page/sector` 完整页面

**旅程 4：从行情中心行业筛选进入**
行情中心（013）行业筛选下拉选「银行」-> 看到银行股列表 -> 想看银行板块整体 -> 点击侧边栏「板块行情」-> 在行业排行表找「银行」-> 展开成分股对照

### FR-2 Tab A 行业板块排行表

**需求**：单 Tab「行业板块」，展示申万一级 28 个行业的涨跌排行表。

**Tab 结构**：仅 1 个 Tab（「行业板块」），不显示 Tab 切换栏（原双 Tab 砍掉概念板块后只剩单 Tab，UI 上不显示 Tab 头，直接展示内容）。

**排行表字段**：

| 列 | 数据来源 | 说明 |
|---|---|---|
| 行业名 | `sw_industry`（level=1） | 申万一级行业名（如「银行」「非银金融」） |
| 行业代码 | `sw_industry.industry_code` | 用于关联成分股与行业指数 ts_code |
| 今日涨跌幅 | `index_daily.pct_chg`（最新交易日） | 按行业指数 pct_chg 展示，红涨绿跌 |
| 成交额 | `index_daily.amount`（最新交易日） | 行业指数成交额（万元） |
| 领涨股 | `sw_industry_member` JOIN `daily_quote`（最新交易日，按 pct_chg 降序 TOP 1） | 该行业成分股中今日涨幅最大的股票（名称+代码+涨幅） |
| 领跌股 | 同上，按 pct_chg 升序 TOP 1 | 该行业成分股中今日跌幅最大的股票（名称+代码+跌幅） |

**排序**：
- 默认按「今日涨跌幅」降序（领涨行业在前）
- 支持点击表头切换排序：今日涨跌幅 / 成交额，升序/降序可切换
- 领涨股/领跌股列不参与排序

**交互**：
- 表格行可点击（鼠标悬停高亮，鼠标指针变手型）
- 点击某行 -> 展开下方成分股列表（FR-3）+ 右侧 K 线图加载该行业指数（FR-4）
- 默认选中第一行（涨跌幅最高的行业），K 线图默认加载第一行行业指数

**数据加载**：
- 页面初始化时一次性拉取 28 个行业的排行数据（1 个接口调用）
- 接口设计建议：`GET /api/sector/industry-ranking?tradeDate={latest}` 返回 28 行排行数据

**验收**：
- 页面加载后表格展示 28 行（申万一级 28 个行业）
- 默认按涨跌幅降序，第一行为今日领涨行业
- 点击表头「今日涨跌幅」「成交额」可切换升降序
- 领涨股/领跌股列展示股票名称+代码+涨跌幅，红涨绿跌颜色正确

### FR-3 行业成分股展开列表

**需求**：点击行业排行表某行，展开该行业的成分股列表，支持分页 + 搜索。

**展开方式**：
- 行内展开（点击行后，下方滑出成分股列表区域，再次点击收起）
- 或右侧抽屉展开（具体交互在落地 spec 定）

**成分股列表字段**：

| 列 | 数据来源 | 说明 |
|---|---|---|
| 股票代码 | `sw_industry_member.con_code` | 如 `000001.SZ` |
| 股票名称 | `stock_basic.name` | 如「平安银行」 |
| 最新价 | `daily_quote.close`（最新交易日） | |
| 今日涨跌幅 | `daily_quote.pct_chg`（最新交易日） | 红涨绿跌 |
| 今日成交量 | `daily_quote.vol`（最新交易日） | 手数 |
| 今日成交额 | `daily_quote.amount`（最新交易日） | 万元 |
| 所属市场 | `stock_basic.market`（主板/创业板/科创板/北交所） | |

**分页**：
- 默认每页 20 条
- 支持切换 20/50/100 每页
- 分页器在列表底部，显示「共 X 条 · 第 Y/Z 页」

**搜索**：
- 列表顶部有搜索框，支持按「股票代码」或「股票名称」模糊搜索
- 输入即过滤（debounce 300ms），不清空分页状态

**跳转**：
- 股票代码列改为链接，点击跳转 `/page/stock-detail/{code}`（依赖个股诊断（017）上线；未上线时点击无响应或 toast 提示「个股诊断即将上线」）

**数据加载**：
- 点击行业行时拉取该行业成分股（1 个接口调用，带分页参数）
- 接口设计建议：`GET /api/sector/industry-members?industryCode={code}&page={p}&size={s}&keyword={k}` 返回分页成分股列表

**验收**：
- 点击行业行后下方/右侧展开成分股列表
- 分页器正常工作，切换每页条数正常
- 搜索框输入关键字后列表实时过滤
- 点击股票代码跳转个股诊断（017 上线后）

### FR-4 行业板块 K 线图

**需求**：展示当前选中行业的指数日线 K 线图，含 MA 主图叠加 + 成交量副图。

**K 线图规格**：

| 项 | 说明 |
|---|---|
| 数据来源 | `index_daily`（行业指数 ts_code，如 `801010.SI` 农林牧渔） |
| 默认周期 | 日线 |
| 默认范围 | 近 250 个交易日（约 1 年） |
| 主图 | K 线（蜡烛图）+ MA5 / MA10 / MA20 / MA60 主图叠加（可切换显隐） |
| 副图 | 成交量柱状图（红涨绿跌） |
| 十字光标 | 鼠标移动显示 OHLCV 浮窗 |
| 主题 | 复用 `ChartsTheme`，三主题（azure/mist/cyber）切换不破坏图表 |
| 渲染库 | ECharts（与仪表盘 K 线一致，复用 `renderKline` 函数） |

**交互**：
- 切换行业（点击排行表其他行）-> K 线图重新加载该行业指数
- MA 线显隐切换（顶部按钮组：MA5/MA10/MA20/MA60，可多选）
- 时间范围切换：近 60 日 / 近 120 日 / 近 250 日 / 全部
- 不在本期做周期切换（日/周/月/分钟）-- 个股诊断 Tab 1 已有完整周期切换，本模块只看日线即可

**数据加载**：
- 切换行业时拉取该行业指数日线（1 个接口调用）
- 接口设计建议：`GET /api/index-daily?tsCode={industry_index_code}&limit={N}&orderBy=trade_date&order=desc` 返回行业指数日线数据

**验收**：
- 默认加载第一行（涨跌幅最高行业）的指数 K 线
- K 线蜡烛图 + MA 线 + 成交量副图正常渲染
- 切换行业行后 K 线图重新加载
- MA 线显隐切换正常
- 时间范围切换正常
- 三主题切换后图表颜色同步变化

### FR-5 顶部板块轮动热度图

**需求**：页面顶部全局区展示板块轮动热度图（网格图），每个格子代表一个申万一级行业，颜色深浅表示今日涨跌幅。

**热度图规格**：

| 项 | 说明 |
|---|---|
| 布局 | 网格图（Grid Heatmap），28 个行业按 7×4 或 6×5 网格排列（具体布局落地 spec 定） |

## 4. 非功能需求

### NFR-1 热度图性能
- 28 个行业格子的热度图首屏渲染 < 500ms（数据已随排行表一次性拉取，不额外发请求）
- 鼠标悬浮浮窗响应 < 100ms
- 颜色梯度计算在客户端完成（基于涨跌幅阈值映射，不依赖后端算颜色）

### NFR-2 长列表分页
- 行业排行表 28 行一次性渲染，不分页（数据量小）
- 成分股列表强制分页（单行业成分股可能几百只，不分页会卡顿），默认每页 20 条
- 分页接口返回 `PageResult`（含 `total` / `page` / `size` / `records`），前端分页器正确展示

### NFR-3 K 线图性能
- 默认加载近 250 日 K 线，渲染 < 1s
- 切换行业重新加载 K 线 < 1s
- MA 线显隐切换无感（不重新拉数据，客户端重算）

### NFR-4 主题统一
- 所有图表（热度图 / K 线 / 成交量副图）复用 `ChartsTheme`
- 三主题（azure/mist/cyber）切换后所有图表颜色同步变化
- 红涨绿跌全局统一（`theme.css` 的 `.stock-up` / `.stock-down` 类）

### NFR-5 数据降级
- 某行业指数在 `index_daily` 缺数据时，K 线图显示「暂无数据」占位，不报错
- 某行业成分股在 `sw_industry_member` 缺数据时，展开列表显示「暂无成分股」，不报错
- 行业排行接口失败时，页面显示重试按钮，不白屏

### NFR-6 接口限流
- 所有接口遵循后端统一限流（如适用），不单独设计
- 前端 debounce：搜索框 300ms、行业切换 500ms（避免快速点击发多次请求）

## 6. 实施建议

### 6.1 建议开发顺序

按功能模块的子任务顺序：

| 子任务 | 内容 | 依赖 | 预估 |
|---|---|---|---|
| **任务 1** | 后端：`SwIndustryController` 对前端暴露（行业树 + 成分股 + 行业排行） | 已有 `sw_industry` / `sw_industry_member` Service | 先做 |
| **任务 2** | 后端：`IndexDailyController`（行业指数行情查询） | 已有 `index_daily` Service（由数据采集任务落地） | 与任务 1 并行 |
| **任务 3** | 前端：`sector.html` + `sector.js` + `sector.css` 页面骨架 + 路由 + 侧边栏菜单 | 任务 1 + 任务 2 接口就绪 | 任务 1/2 完成后 |
| **任务 4** | 前端：行业板块 Tab 内容（热度图 + 排行表 + 成分股展开 + K 线图） | 任务 3 骨架就绪 | 任务 3 完成后 |

**建议理由**：
- 任务 1/2 后端 Controller 可并行开发（不同表），互不阻塞
- 任务 3 页面骨架依赖 Controller 接口就绪才能联调
- 任务 4 是最重的前端工作（热度图 + K 线图 + 分页 + 联动），建议集中一个迭代

### 6.2 与其他模块的关系

| 关联模块 | 关系 | 说明 |
|---|---|---|
| **数据采集任务** | 硬前置 | `index_daily` 表 + `INDEX_DAILY` 枚举 + Service + 定时拉取任务必须先落地（由 tushare `index_daily` 接口采集，2000 起可用） |

| 风险 | 影响 | 应对 |
