# 行情中心模块 PRD：现有页面功能补齐（S1-S8）

> **模块**：013 · 行情中心（全市场列表入口，路由 `/page/stock-list`，已在侧边栏菜单）
> **状态**：规划中，待 `daily_basic` Controller 与个股诊断模块（017）下钻目标就绪后进入实施
> **来源**：拆分自 [`主要功能模块整体设计计划.md`](./../../.trae/documents/主要功能模块整体设计计划.md) §2.2 行情中心补齐（S1-S8）、§1.1 页面形态、§7 假设与约束、§10 附录接口核查
> **原则**：从**用户可见功能角度**描述需求；数据依赖直接写表名/接口名；不写代码实现；接口必须来自 `tushare 接口汇总.md` 且 ≤2000 积分。

## 0. 拆分说明（为什么单独立项）

总计划文档把行情中心补齐混排在现有 3 模块补齐清单里，与仪表盘（D1-D5）、自选股（W1-W10）同列。本文把 **行情中心 8 项功能补齐**（S1-S8）单独拎出独立成稿，因为：

- **行情中心定位独特**：路由 `/page/stock-list` 是项目「全市场列表入口」，老股民浏览 A 股全市场行情的第一站，8 项补齐围绕同一页面、同一用户场景（全市场扫描 + 多维筛选 + 下钻个股），天然是一组。
- **8 项补齐相互关联**：S1 榜单 Tab / S2 行业筛选 / S3 市场筛选 共享同一张全市场表格；S4 列排序 / S7 表格加列 共享同一组表头与列定义；S5 跳转 / S6 搜索 是同一顶部工具条的两个入口；S8 CSV 导出依赖 S7 新增列的数据完整性。单独立项便于统一表格结构与交互。
- **数据依赖集中**：S1/S4/S7/S8 共同依赖 `daily_basic` 表（PE/PB/换手率/市值/量比）；S2/S3 依赖 `stock_basic` + 申万行业分类；S5 依赖个股诊断页面路由；S6 复用已有 `SearchSuggest` 组件。8 项在数据层面高度耦合。

> **不在本 PRD 范围**：仪表盘补齐（D1-D5）、自选股补齐（W1-W10）、板块行情页面本身（015）、个股诊断页面本身（017）--各自独立排期；本 PRD 只描述行情中心页面 `/page/stock-list` 的功能补齐。

## 1. 背景与目标

### 1.1 行情中心定位

- **全市场列表入口**：路由 `/page/stock-list`，侧边栏「主要功能」第 2 项，老股民浏览 A 股全市场行情的第一站。
- **老股民选股前置环节**：老股民习惯「先扫全市场榜单找热点 -> 按行业/市场筛选缩小范围 -> 看多维指标（PE/PB/换手率/量比）精选 -> 点击个股进入诊断」；行情中心是这个使用路径的起点，下钻终点是个股诊断。
- **现有页面已实现但功能单薄**：行情中心页面骨架已存在（`stock-list.html` + 内联脚本），但仅展示基础行情列表 + 涨幅/跌幅/成交额 3 榜，缺 8 项关键能力（详见 §1.2）。

### 1.2 现状与功能缺口

| # | 现状 | 缺口 |
|---|---|---|
| S1 | 顶部仅有涨幅榜 / 跌幅榜 / 成交额榜 3 个 Tab | 需扩展至 6 榜：涨幅榜 / 跌幅榜 / 换手率榜 / 成交额榜 / 量比榜 / 振幅榜 |
| S2 | 无行业筛选 | 需加申万一级行业筛选下拉，按行业缩小列表范围 |
| S3 | 无市场筛选 | 需加沪 / 深 / 创 / 科 / 北 5 大市场筛选下拉 |
| S4 | 表头不可排序 | 需支持点击表头按 涨跌幅 / 成交量 / 换手率 / 市值 / PE 排序 |
| S5 | 股票代码是纯文本，不可点击 | 需改为链接，点击跳转 `/page/stock-detail/{code}` |
| S6 | 顶部搜索框无联想 | 需集成 `SearchSuggest` 组件，输入代码/名称实时联想 |
| S7 | 表格只有基础行情列（代码/名称/价/涨跌幅/量/额） | 需增加 市值 / PE / PB / 换手率 / 量比 / 行业 6 列 |
| S8 | 「导出 CSV」按钮点击只 toast 占位 | 需实现真实 CSV 导出，按当前筛选 + 排序结果导出 |

### 1.3 非目标（明确不做）

| 非目标项 | 原因 | 备注 |
|---|---|---|
| **概念板块筛选** | `concept` / `concept_detail` 接口不在 `tushare 接口汇总.md` 中，不存在 | 行业筛选只做申万一级行业，不做概念板块 |
| **盘中实时分时刷新** | 不在本期范围 | 行情中心采用页面手动刷新或定时轮询（每日收盘后数据更新），不做 WebSocket 推送 |
| **分钟级 K 线内嵌** | K 线能力归个股诊断 Tab 1 | 行情中心只做列表，不做 K 线图（个股下钻后才看 K 线） |
| **财务指标深度筛选** | 财务数据（ROE/毛利率等）归选股中心 | 行情中心只做行情维度筛选（榜单/行业/市场），不做财务因子筛选 |
| **跨市场对比（港股/美股）** | 不在本期范围 | 仅 A 股沪/深/创/科/北 5 大市场 |
| **自选股一键加入** | 自选股管理归自选股页面 | 行情中心只提供「跳转个股诊断」入口，不提供「加自选」按钮（加自选在个股诊断顶部摘要条） |
| **行情数据反向写入** | 数据单源约束 | 行情中心只读消费，不写入任何数据 |

### 1.4 总体目标

- **目标一**：让老股民在行情中心能从 6 个维度（榜单/行业/市场/排序/搜索/指标列）快速缩小全市场范围，找到目标个股。
- **目标二**：让列表展示多维指标（市值/PE/PB/换手率/量比/行业），一眼判断个股估值与活跃度，不用再切到其他页面查基本面。
- **目标三**：让老股民从列表无缝下钻个股诊断，形成「列表扫描 -> 多维筛选 -> 指标精选 -> 下钻诊断」的完整使用闭环。
- **目标四**：让筛选结果可一键导出 CSV，支持离线分析或导入其他工具。
- **质量准绳**：`.trae/rules/系统设计理念(用法建议).md` + A 股实战派认可（不接受"纸面能跑、实战不能用"）。

## 2. 典型用户场景

### 2.1 主场景：老股民开盘后浏览全市场列表

**时间**：交易日 9:30 开盘后 / 15:00 收盘后两个时点
**用户**：3 年以上经验的 A 股散户/老股民，习惯用同花顺/东方财富看「涨幅榜/换手率榜/振幅榜」找热点，希望在本系统内一站式扫描全市场

**使用路径**：

1. 登录后点击侧边栏「行情中心」进入 `/page/stock-list`
2. **看榜单找热点（S1）**：
   - 默认看「涨幅榜」TOP 50，扫一眼今日领涨个股
   - 切到「换手率榜」看资金活跃个股
   - 切到「量比榜」看放量异动个股（量比 > 2 = 明显放量）
   - 切到「振幅榜」看波动剧烈个股（振幅大 = 多空博弈激烈）
3. **按行业缩小范围（S2）**：
   - 行业筛选下拉选「银行」-> 列表只显示银行股
   - 老股民判断「今日银行板块领涨，要在银行股里挑个股」
4. **按市场进一步过滤（S3）**：
   - 市场筛选下拉选「沪」-> 列表只显示沪市银行股
5. **按指标排序精选（S4 + S7）**：
   - 点击表头「PE」列 -> 按 PE 升序排列（低 PE 优先）
   - 看市值/PE/PB/换手率/量比 6 列，综合判断
   - 老股民经验：PE < 10 且 PB < 1 且 换手率 > 3% = 低估值活跃股
6. **搜索特定股票（S6）**：
   - 顶部搜索框输入「平安」-> 联想出「平安银行 000001.SZ」
   - 点击联想项 -> 直接跳转该股票的个股诊断页
7. **点击下钻诊断（S5）**：
   - 列表中点击某股票代码（如 `600036.SH` 招商银行）
   - 跳转 `/page/stock-detail/600036.SH` 进入个股诊断做深入分析
8. **收盘后导出 CSV（S8）**：
   - 收盘后回到行情中心，按当前筛选 + 排序结果
   - 点击「导出 CSV」按钮 -> 下载 CSV 文件
   - 在 Excel 里离线分析今日筛选出的候选股池

### 2.2 异常场景：数据未拉取或依赖模块未上线

- **`daily_basic` 表无数据**：PE/PB/换手率/市值/量比 5 列显示「-」占位，不影响基础行情列（代码/名称/价/涨跌幅/量/额）展示
- **个股诊断模块（017）未上线**：股票代码列显示为纯文本（不可点击），不报错；017 上线后自动变为可点击链接
- **板块行情模块（015）未上线**：行业筛选下拉为空或显示「暂无行业数据」，不阻塞其他筛选能力
- **榜单数据缺失**：对应榜单 Tab 显示「暂无数据」占位，其他 Tab 正常切换
- **CSV 导出失败**：toast 提示「导出失败，请重试」，不阻塞页面其他功能

## 3. 功能需求

### FR-1：顶部榜单 Tab 扩展至 6 榜（对应 S1）

#### 3.1.1 功能描述

行情中心顶部 Tab 从现有 3 榜（涨幅榜 / 跌幅榜 / 成交额榜）扩展为 **6 榜**：

| Tab | 排序字段 | 排序方向 | 数据来源 | 说明 |
|---|---|---|---|---|
| 涨幅榜 | `pct_chg` | 降序 | `daily_quote` | 今日涨幅 TOP 50 |
| 跌幅榜 | `pct_chg` | 升序 | `daily_quote` | 今日跌幅 TOP 50 |
| 换手率榜 | `turnover_rate` | 降序 | `daily_basic` JOIN `daily_quote` | 换手率 TOP 50，资金活跃 |
| 成交额榜 | `amount` | 降序 | `daily_quote` | 成交额 TOP 50，资金体量 |
| 量比榜 | `volume_ratio` | 降序 | `daily_basic` JOIN `daily_quote` | 量比 TOP 50，放量异动 |
| 振幅榜 | `(high - low) / pre_close * 100` | 降序 | `daily_quote` | 振幅 TOP 50，波动剧烈 |

每个 Tab 默认展示 TOP 50，支持翻页加载更多（每页 50，最多 500）。

#### 3.1.2 用户价值

- **多维榜单覆盖 A 股老股民所有看榜场景**：老股民看盘习惯同时看 6 个榜单--涨幅榜找热点、跌幅榜找错杀、换手率榜找活跃、成交额榜找资金、量比榜找放量、振幅榜找波动。
- **量比榜与振幅榜是老股民刚需**：量比 > 2 = 明显放量（可能有消息面驱动）；振幅 > 5% = 多空博弈激烈（可能有主力资金进出）。这两个榜单在现有 3 榜里缺位，是本次补齐的核心增量。

#### 3.1.3 数据依赖

- **复用现有 `/market/ranking` 接口**，扩展支持 6 种榜单类型（接口参数 `type` 取值扩展）。
- **依赖 `daily_quote` 表**（已有数据，由 tushare `daily` 接口采集，120 起积分）：提供 `pct_chg`（涨跌幅）/ `amount`（成交额）/ `high`（最高）/ `low`（最低）/ `pre_close`（前收）/ `vol`（成交量）/ `close`（收盘价）。
- **依赖 `daily_basic` 表**（已有数据，由 tushare `daily_basic` 接口采集，2000 起积分）：提供 `turnover_rate`（换手率）/ `volume_ratio`（量比）。
- 振幅榜由 `daily_quote` 的 `high` / `low` / `pre_close` 字段前端或后端计算，公式：`(high - low) / pre_close * 100`。
- **无需新建表**，纯 JOIN 查询扩展。

#### 3.1.4 前端改造点

- `stock-list.html`：顶部 Tab 切换栏从 3 个扩展为 6 个 Tab。
- `stock-list.js`（从内联抽出）：扩展榜单渲染逻辑，支持 `type=turnover` / `type=volume_ratio` / `type=amplitude` 参数调用 `/market/ranking` 接口。

#### 3.1.5 后端改造点

- 扩展 `MarketController.ranking` 接口，`type` 参数新增 3 个取值：`turnover` / `volume_ratio` / `amplitude`。
- `MarketService.getRanking(type, tradeDate, limit)` 方法加 3 个分支：
  - `turnover`：JOIN `daily_quote` + `daily_basic`，按 `turnover_rate` 降序取 TOP 50
  - `volume_ratio`：JOIN `daily_quote` + `daily_basic`，按 `volume_ratio` 降序取 TOP 50
  - `amplitude`：查 `daily_quote`，按 `(high - low) / pre_close * 100` 降序取 TOP 50（计算可在 SQL 或 Service 层做）
- **无需新建表**，纯 JOIN 查询扩展。

### FR-2：行业筛选下拉（对应 S2）

#### 3.2.1 功能描述

行情中心顶部工具条加「行业筛选」下拉框：

| 项 | 说明 |
|---|---|
| 下拉位置 | 顶部工具条，榜单 Tab 下方、搜索框左侧 |
| 下拉选项 | 申万一级行业列表（28 个）+「全部行业」选项（默认选中） |
| 选项文案 | 行业中文名（如「银行」「非银金融」「医药生物」），不显示行业代码 |
| 交互 | 选择某行业后，列表立即按该行业过滤；选「全部行业」恢复全市场 |
| 默认值 | 「全部行业」（不筛选） |

#### 3.2.2 用户价值

- **按行业缩小列表范围**：A 股全市场 5000+ 股票，老股民看盘时习惯先选行业再看个股，行业筛选是必备能力。
- **配合榜单使用**：先切「换手率榜」找资金活跃方向 -> 再切「银行」行业筛选 -> 看银行板块今日活跃个股，形成「榜单 + 行业」二维筛选组合。

#### 3.2.3 数据依赖

- **依赖申万一级行业分类数据**：由 tushare `index_classify` 接口（2000 积分）+ `index_member_all` 接口（2000 积分）采集，落库到 `sw_industry` 表（行业树）+ `sw_industry_member` 表（成分股关系）。
- **依赖 `SwIndustryController`** 暴露的 `/api/industry/list?level=1` 接口，返回申万一级行业列表（28 个），含 `industry_code` / `industry_name`。
- **行业过滤逻辑**：列表查询时 JOIN `sw_industry_member` 表，按所选 `industry_code` 过滤 `ts_code` 属于该行业的股票。
- `sw_industry` / `sw_industry_member` 表已有数据（已有 Service），本 PRD 只消费 Controller 暴露的查询接口。

#### 3.2.4 前端改造点

- `stock-list.html`：顶部工具条加「行业」下拉框（Bootstrap dropdown 或 select2）。
- `stock-list.js`：页面加载时调用 `/api/industry/list?level=1` 拉取行业列表填充下拉；选择行业后带 `industryCode` 参数重新查询列表接口。

#### 3.2.5 后端改造点

- **复用板块行情模块（015）提供的 `SwIndustryController`**，调用 `GET /api/industry/list?level=1` 获取行业列表。本模块不重复实现行业树查询。
- 行情中心的列表查询接口（`/market/ranking` 或新的 `/market/stock-list` 接口）需扩展支持 `industryCode` 参数，JOIN `sw_industry_member` 表做行业过滤。

### FR-3：市场筛选下拉（对应 S3）

#### 3.3.1 功能描述

行情中心顶部工具条加「市场筛选」下拉框：

| 项 | 说明 |
|---|---|
| 下拉位置 | 顶部工具条，行业筛选下拉右侧 |
| 下拉选项 | 全部市场（默认）/ 沪市 / 深市 / 创业板 / 科创板 / 北交所，共 6 选项 |
| 选项文案 | 中文（「沪市」「深市」「创业板」「科创板」「北交所」） |
| 交互 | 选择某市场后，列表立即按该市场过滤；选「全部市场」恢复全市场 |
| 默认值 | 「全部市场」（不筛选） |

#### 3.3.2 用户价值

- **按市场板块缩小范围**：老股民习惯区分沪/深/创/科/北，不同市场板块有不同的涨跌停规则与风险偏好。创业板/科创板涨跌停 20%，主板 10%，北交所 30%，筛选市场便于按规则分类看盘。

#### 3.3.3 数据依赖

- **依赖 `stock_basic` 表**（已有数据，由 tushare `stock_basic` 接口采集，2000 积分）：`market` 字段标识市场板块，取值如 `主板` / `创业板` / `科创板` / `北交所`（具体枚举以 `stock_basic` 表实际取值为准）。
- **沪/深市场判断**：`stock_basic.ts_code` 后缀 `.SH` = 沪市，`.SZ` = 深市；`market` 字段进一步区分主板/创业板/科创板/北交所。
- 市场筛选通过 JOIN `stock_basic` 表按 `market` 字段过滤实现。

#### 3.3.4 前端改造点

- `stock-list.html`：顶部工具条加「市场」下拉框。
- `stock-list.js`：选择市场后带 `market` 参数重新查询列表接口。

#### 3.3.5 后端改造点

- 行情中心的列表查询接口扩展支持 `market` 参数，JOIN `stock_basic` 表按 `market` 字段过滤。
- **无需新建表**，纯 JOIN 查询扩展。

### FR-4：表头列排序（对应 S4）

#### 3.4.1 功能描述

行情中心列表表头支持点击排序，可排序的列：

| 列 | 排序字段 | 默认方向 | 数据来源 |
|---|---|---|---|
| 涨跌幅 | `pct_chg` | 降序 | `daily_quote` |
| 成交量 | `vol` | 降序 | `daily_quote` |
| 成交额 | `amount` | 降序 | `daily_quote` |
| 换手率 | `turnover_rate` | 降序 | `daily_basic` |
| 市值 | `total_mv` | 降序 | `daily_basic` |
| PE | `pe_ttm` | 升序（低估值优先） | `daily_basic` |
| PB | `pb` | 升序 | `daily_basic` |
| 量比 | `volume_ratio` | 降序 | `daily_basic` |
| 振幅 | `(high - low) / pre_close * 100` | 降序 | `daily_quote` |

**交互规则**：
- 点击表头第一次：按该列默认方向排序
- 点击表头第二次：反序
- 点击表头第三次：取消排序（恢复榜单默认排序）

#### 3.4.4 前端改造点

## 4. 非功能需求

### 4.1 长列表分页性能

- **列表默认每页 50 条**，支持翻页加载更多（最多 500 条）。
- 翻页响应时间 ≤ 500ms（含网络 + 后端查询 + 前端渲染）。
- 后端查询接口必须分页（`page` + `size` 参数），禁止一次返回全量数据。
- `daily_quote` + `daily_basic` + `sw_industry_member` 三表 JOIN 查询需有 `(ts_code, trade_date)` 联合索引，确保 JOIN 性能。
- 前端渲染大数据量时用虚拟滚动或分页渲染（避免一次渲染 500 行 DOM 卡顿）。

### 4.2 主题统一

- 列表所有数字、涨跌色、链接色必须复用 `theme.css` 与 `ChartsTheme` 主题变量。
- 三主题（azure / mist / cyber）切换后，列表颜色同步变化，不允许出现「切换主题后表格颜色不更新」的 bug。
- 涨跌色统一用 `theme.css` 的 `.stock-up`（红）/ `.stock-down`（绿）类名，不硬编码颜色值。
- 表头排序箭头、下拉框样式、搜索框样式与全局组件样式一致（复用 Bootstrap 主题变量）。

### 4.3 CSV 导出性能

- 导出 ≤ 500 条数据，前端生成 CSV 文件时间 ≤ 2 秒。
- 导出过程中按钮显示 loading 状态，禁止重复点击（防多次下载）。
- 导出失败时 toast 提示，不阻塞页面其他功能。
- CSV 文件大小预估 ≤ 200KB（500 条 × 12 列），浏览器可直接生成不卡顿。

### 4.4 数据降级

- **`daily_basic` 表无数据**：PE/PB/换手率/市值/量比 5 列显示「-」占位，基础行情列正常展示。
- **`sw_industry_member` 表无数据**：行业列显示「-」，行业筛选下拉为空。
- **`stock_basic` 表无数据**：市场筛选下拉为空，列表股票名称显示代码。
- **个股诊断模块（017）未上线**：股票代码列降级为纯文本（不可点击），其他功能不受影响。
- **板块行情模块（015）未上线**：行业筛选下拉为空或显示「暂无行业数据」，其他筛选能力正常。
- **榜单数据缺失**：对应 Tab 显示「暂无数据」占位，其他 Tab 正常切换。
- **数据时效提示**：列表顶部显示「数据日期 YYYY-MM-DD」，让用户清楚看到数据是何日的（避免周末看到周五数据以为今日数据）。

### 4.5 兼容性

- 浏览器：Chrome / Edge / Firefox 最新版（不兼容 IE）。
- 分辨率：最低 1366x768，推荐 1920x1080。
- 移动端：不做响应式适配（行情中心是 PC 端专业工具，不在手机使用）。

## 7. 风险与注意事项

### 7.1 数据时效风险

- **行情数据延迟**：`daily` / `daily_basic` 接口每日收盘后采集，盘中行情中心显示的是前一日收盘数据，需在列表顶部明确标注「数据日期」避免误导。
- **降级方案**：盘中访问行情中心时，列表顶部显示「截至 YYYY-MM-DD 收盘」提示。

### 7.2 三表 JOIN 性能风险

- **`daily_quote` + `daily_basic` + `sw_industry_member` 三表 JOIN** 在全市场 5000+ 股票场景下可能性能不足。
- **应对**：确保 `daily_quote` 与 `daily_basic` 表都有 `(ts_code, trade_date)` 联合索引，`sw_industry_member` 表有 `con_code` 索引；Service 层做缓存（如 Redis 缓存当日榜单 + 筛选结果，TTL 到当日收盘）；分页查询避免一次返回全量。

### 7.3 PE/PB 为 NULL 或负值的处理

- **亏损股 PE 为负或 NULL**：`daily_basic.pe_ttm` 对亏损股为负值或 NULL，排序时需统一处理（NULL 排最后，负值按数值排序）。
- **破净股 PB < 1**：PB 可能为负（资不抵债）或 NULL，展示时负值显示「-」或原值（老股民习惯看破净股，PB < 1 是机会信号，不能简单隐藏）。
- **建议实现**：展示时 NULL 显示「-」，负值原值显示（保留小数 2 位）；排序时 NULL 统一排最后，负值按数值参与排序。

### 7.4 量比字段口径

- **`daily_basic.volume_ratio` 是 tushare 标准量比字段**，口径与同花顺一致（当日平均每分钟成交量 / 过去 5 日平均每分钟成交量）。
- 量比为 NULL 的情况：新股上市不足 5 日、停牌复牌首日等，展示时显示「-」。

### 7.5 行业归属的时效性

- **`sw_industry_member` 是申万行业成分股关系表**，成分股会定期调整（申万每年调整 1-2 次）。
- 行业列展示的是当前最新归属，不反映历史归属变更（不做 point-in-time 行业归属，那是轮动回测的事）。
- 老股民看盘时行业归属按最新即可，无需历史回溯。

### 7.6 股票代码跳转的降级方案

- **个股诊断模块（017）未上线时**：FR-5 股票代码跳转需做降级处理，避免点击后 404 影响体验。
- **建议方案**：前端读取全局配置 `window.STOCK_DETAIL_ENABLED`（或调用 `/api/stock-detail/health` 健康检查），017 未上线时渲染为纯文本 `<span>`，017 上线后渲染为 `<a>` 链接。具体降级方案落地 spec 定。
