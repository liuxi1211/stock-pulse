## 1. 产品定位与设计哲学

### 1.1 核心认知

做了 10 年 A 股，最深刻的体会是：**散户亏钱不是因为不懂技术，而是因为没有纪律**。

量化系统的本质不是发明一套神奇的赚钱公式，而是：

1. **把你的经验变成可执行的规则** — 你知道"均线多头排列买入"是对的，但你做不到每次都严格执行
2. **用历史数据验证你的判断** — 你觉得某个策略有效，但你需要数字证明它过去 5 年到底赚了多少
3. **让机器替你盯盘** — 5000 只股票人眼看不过来，机器可以每天扫描全部
4. **控制风险保住本金** — 活着比赚钱重要，单票重仓、追涨杀跌这些毛病系统帮你拦住

### 1.2 设计哲学

| 原则 | 说明 |
|---|---|
| **数据归一** | Java 是唯一的数据入口和出口。所有数据通过 Tushare 获取、清洗、存储在 SQLite。Python 不直连数据库，只通过 Java 提供的数据接口获取计算所需数据 |
| **规则即策略** | 策略通过可视化界面配置规则组合来定义，而非手写代码。组合因子条件、设置买卖规则、调整参数，全程 UI 操作，无需改代码。高级用户可扩展自定义因子 |
| **引擎即服务** | Python 作为纯计算服务存在，以 akquant 为核心引擎暴露标准 API。Java 不关心 Python 内部实现，只关心输入输出的 JSON 契约 |
| **配置优于硬编码** | 策略参数、风控规则、信号推送渠道全部可配置，改配置不改代码 |
| **简单策略优先** | 先跑通简单的均线策略，再逐步迭代复杂策略。不要一上来就搞多因子模型 |

### 1.3 产品边界

**做什么**：

- 日线级别的中低频策略研究、回测、信号生成
- 个人持仓管理和风控
- 策略绩效追踪和复盘

**不做什么**：

- 不做高频交易 / 秒级策略
- 不做自动下单（MVP 阶段）
- 不做多人协作 / 社区功能
- 不做实时行情推送（用收盘后信号替代）

### 2.2 核心设计变更：数据流与职责划分

与 V1.0 方案最大的区别：**Python 不直连数据库**。

**为什么这样设计**：

1. **数据一致性**：Java 负责数据获取、清洗、复权，是唯一的数据真相源（Single Source of Truth）。如果 Python 也直连 SQLite，可能出现数据理解不一致（如复权方式、停牌处理）
2. **解耦数据与计算**：Python 不需要关心数据怎么来的、存在哪里。它只关心输入是标准格式的 JSON/DataFrame，输出是计算结果
3. **Python 服务无状态化**：Python 服务不依赖数据库文件路径、不依赖 SQLite 驱动，部署更简单，未来甚至可以跑在远程服务器上
4. **数据安全**：Java 控制所有数据访问权限，可以做频率限制、数据过滤

**数据流转方式**：

```
Java 向 Python 传递数据的两种模式：

模式 1：计算请求内嵌数据（小数据量，单股/少量股票）
  Java 调用 Python API 时，在请求 body 中直接携带行情数据
  适用：单股回测、因子计算、信号生成

模式 2：数据文件传递（大数据量，全市场扫描）
  Java 先将数据导出为临时文件（Parquet/CSV），告知 Python 文件路径
  Python 从文件读取，计算完成后删除临时文件
  适用：全市场信号扫描、参数优化、Walk-forward 分析
```

## 3. 核心功能模块

### 3.1 因子库 (Factor Library)

#### 3.1.1 设计目标

因子是策略的积木。系统提供一套可视化可配置的因子，你通过组合因子条件来构建策略，而不是写代码。

#### 3.1.2 因子分类体系

```
因子库
├── 技术面因子 (Technical)        — 量价关系，择时用
│   ├── 趋势类: MA, MACD, DMI, SAR
│   ├── 动量类: RSI, KDJ, WR, CCI
│   ├── 波动类: BOLL, ATR, KELCH
│   └── 量能类: OBV, VOL_MA, AMOUNT_RANK
│
├── 基本面因子 (Fundamental)      — 公司质量，选股用（Phase 2）
│   ├── 估值类: PE_TTM, PB, PS_TTM, EV/EBITDA
│   ├── 质量类: ROE, ROA, 净利润率, 毛利率
│   ├── 成长类: 净利润增速, 营收增速, EPS增速
│   └── 安全类: 资产负债率, 流动比率, 经营现金流/净利润
│
├── 情绪面因子 (Sentiment)        — 市场资金行为，过滤用（Phase 2）
│   ├── 换手率, 量比
│   ├── 北向资金净买入, 北向资金持股占比变化
│   ├── 龙虎榜上榜次数, 上榜买入额
│   └── 融资融券余额变化
│
└── 统计面因子 (Statistical)      — 高阶计算（Phase 2）
    ├── 波动率 (历史/隐含)
    ├── 偏度/峰度
    ├── 相关性矩阵
    └── 因子 IC/IR 值
```

#### 3.1.3 因子注册机制

每个因子通过标准注册声明定义，Python 端自动扫描注册。**因子注册信息同时会同步给 Java 端**，用于前端渲染因子选择器和参数配置表单。

```python
from engine.factor import FactorRegistrar

@FactorRegistrar.register(
    name="MA",
    display_name="简单移动均线",
    category="TECHNICAL",
    sub_category="TREND",
    params={"period": {"type": "int", "default": 5, "range": [2, 250]}},
    dependencies=[],
    # 前端展示配置
    ui_config={
        "comparators": [">", "<", ">=", "<=", "cross_up", "cross_down"],
        "default_comparator": "cross_up",
        "description": "均线值或均线交叉信号",
    }
)
def compute_ma(df, period=5):
    """df 为标准 OHLCV DataFrame，由 Java 传入"""
    return df["close"].rolling(period).mean()
```

**关键设计**：
- 因子元信息（名称、参数、UI 配置）通过 `/api/compute/factors/registry` 接口暴露给 Java
- Java 拿到因子注册表后，在前端动态渲染策略配置界面
- 新增因子只需在 Python 端添加 `@register` 函数，Java 前端自动适配

### 3.2 策略管理 (Strategy Management)

#### 3.2.1 设计思路：可视化规则配置

**策略不需要写代码**。系统提供可视化的策略配置界面，你通过组合因子条件来定义买卖规则。

每个策略由以下要素组成：

1. **选股范围（Universe）**：在哪些股票中筛选（全市场 / 沪深 300 成分 / 自选股 / 自定义条件）
2. **风险前置过滤（Pre-filter）**：排除 ST、次新股、流动性不足等标的
3. **买入规则（Entry Rules）**：一组因子条件的 AND/OR 组合
4. **卖出规则（Exit Rules）**：一组因子条件的 AND/OR 组合 + 止损止盈
5. **仓位管理（Position Sizing）**：单票仓位、最大持仓数、调仓频率
6. **参数配置（Params）**：均线天数、止损比例等可调参数

#### 3.2.2 策略规则的数据模型

策略的核心是一棵**规则树**，用 JSON 存储：

**数据库表 — 策略元数据**：

```sql
CREATE TABLE quant_strategy (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT NOT NULL UNIQUE,          -- 策略英文标识，如 sma_cross
    display_name    TEXT NOT NULL,                 -- 策略中文名，如 "双均线交叉"
    description     TEXT,
    category        TEXT NOT NULL,                 -- TREND / MEAN_REVERT / MOMENTUM / VALUE / MULTI_FACTOR / CUSTOM
    -- ★ 策略规则定义（核心）
    entry_rules     TEXT NOT NULL,                 -- JSON: 买入规则树
    exit_rules      TEXT NOT NULL,                 -- JSON: 卖出规则树
    risk_rules      TEXT,                          -- JSON: 策略级风控规则（止损止盈等）
    position_rules  TEXT,                          -- JSON: 仓位管理规则
    pre_filter      TEXT,                          -- JSON: 前置过滤条件
    -- 参数
    params_schema   TEXT NOT NULL,                 -- JSON Schema: 策略参数定义（前端动态渲染表单）
    params_default  TEXT NOT NULL,                 -- JSON: 默认参数值
    param_space     TEXT,                          -- JSON: 参数优化搜索空间
    -- 选股范围
    universe        TEXT NOT NULL DEFAULT 'ALL',   -- ALL / INDEX_300 / INDEX_500 / WATCHLIST / CUSTOM
    universe_filter TEXT,                          -- JSON: 自定义股票池过滤条件
    -- 状态
    status          TEXT NOT NULL DEFAULT 'DRAFT', -- DRAFT / ACTIVE / PAUSED / ARCHIVED
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2.3 规则树的 JSON 结构

规则树采用递归结构，支持 AND / OR 逻辑嵌套：

```json
{
    "operator": "AND",
    "conditions": [
        {
            "type": "factor_compare",
            "factor": "MA",
            "params": {"period": 5},
            "comparator": "cross_up",
            "target_factor": "MA",
            "target_params": {"period": 20}
        },
        {
            "type": "factor_threshold",
            "factor": "VOL_MA",
            "params": {"period": 5},
            "comparator": ">",
            "multiplier": 1.5,
            "description": "成交量放大 1.5 倍"
        },
        {
            "operator": "OR",
            "conditions": [
                {
                    "type": "factor_threshold",
                    "factor": "RSI",
                    "params": {"period": 14},
                    "comparator": ">",
                    "value": 50
                },
                {
                    "type": "factor_compare",
                    "factor": "MACD",
                    "params": {},
                    "comparator": ">",
                    "value": 0
                }
            ]
        }
    ]
}
```

**规则节点类型**：

| 类型 | 说明 | 示例 |
|---|---|---|
| `factor_compare` | 因子与因子比较 | MA(5) 上穿 MA(20) |
| `factor_threshold` | 因子与阈值比较 | RSI(14) > 70 |
| `factor_multiplier` | 因子与另一因子倍数比较 | 成交量 > VOL_MA(5) × 1.5 |
| `price_compare` | 价格与因子比较 | 收盘价 > MA(20) |
| `AND` / `OR` | 逻辑组合 | 多个条件的与/或组合 |

**卖出规则额外支持**：

| 类型 | 说明 | 示例 |
|---|---|---|
| `stop_loss` | 固定止损 | 买入价 × (1 - 5%) |
| `trailing_stop` | 移动止损 | 从最高点回撤 8% |
| `take_profit` | 固定止盈 | 买入价 × (1 + 10%) |
| `max_holding_days` | 最大持仓天数 | 持有超过 10 天自动卖出 |

#### 3.2.5 策略生命周期

```

                  │
                  └─────→ ARCHIVED
                          (归档，不再运行但保留历史)
```

- **DRAFT**：策略规则已配置，参数已设置，但尚未进行回测验证
- **ACTIVE**：策略已通过回测验证，纳入每日信号生成
- **PAUSED**：临时暂停，不参与信号生成，保留所有历史数据
- **ARCHIVED**：策略下线，保留回测历史和信号记录供复盘

#### 3.2.6 策略版本管理

每次修改策略规则或参数后，系统自动记录变更：

### 3.3 回测中心 (Backtesting)

#### 3.3.1 设计原则

回测是量化系统最核心的功能，也是你验证策略想法的唯一手段。设计上必须做到：

- **快速验证**：一个想法 → 配好参数 → 点击回测 → 5 分钟内看到结果
- **结果可信**：严格模拟 A 股真实交易规则，避免回测结果虚高
- **对比分析**：支持策略之间横向对比，支持与基准（沪深 300）对比

#### 3.3.2 回测配置

#### 3.3.7 Walk-forward 分析

Walk-forward 是最接近实盘的回测验证方法，利用 akquant 的 Walk-forward 功能：

```json
POST /api/compute/walk-forward
{
    "task_id": "wf-20260526-001",
    "strategy": { ... },
    "data_mode": "file",
    "data_file": "/data/temp/wf-20260526-001.parquet",
    "config": {
        "start_date": "2018-01-01",
        "end_date": "2026-05-25",

        "anchor": false,                     // 滚动窗口（true=扩展窗口）
        "metric": "sharpe_ratio",
        "param_grid": {
            "sma_short": [3, 5, 8, 10],

        "initial_cash": 100000
```

**防过拟合机制**：

### 3.6 风控系统 (Risk Management)

#### 3.6.1 设计思路

风控是系统的安全网，分为两层：
- **策略级风控**：每个策略自己的止损止盈规则（在策略规则 JSON 的 `exit_rules` 中配置）
- **账户级风控**：全局风控规则，覆盖所有策略（Java 端执行）

#### 3.6.2 风控规则配置

```sql
CREATE TABLE quant_risk_rule (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT NOT NULL UNIQUE,
    display_name    TEXT NOT NULL,
    category        TEXT NOT NULL,                  -- POSITION / LOSS / TRADE / LIQUIDITY
    description     TEXT,
    rule_config     TEXT NOT NULL,                  -- JSON: 规则参数
    severity        TEXT NOT NULL DEFAULT 'WARN',   -- WARN / BLOCK
    enabled         INTEGER DEFAULT 1,
    priority        INTEGER DEFAULT 0,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**内置风控规则**：

| 规则 | 类别 | 默认参数 | 说明 |
|---|---|---|---|
| `MAX_SINGLE_POSITION` | POSITION | `{"max_pct": 0.08}` | 单票仓位不超过 8% |
| `MAX_TOTAL_POSITION` | POSITION | `{"max_pct": 0.90}` | 总仓位不超过 90% |
| `MAX_INDUSTRY_CONCENTRATION` | POSITION | `{"max_pct": 0.25}` | 同行业仓位不超过 25% |
| `DAILY_LOSS_LIMIT` | LOSS | `{"max_pct": 0.01}` | 单日亏损超 1% 停止开新仓 |
| `MAX_DRAWDOWN_CIRCUIT_BREAKER` | LOSS | `{"max_pct": 0.08}` | 最大回撤超 8% 触发降仓 |
| `STOP_LOSS_CHECK` | LOSS | `{"default_pct": 0.05}` | 持仓浮亏超 5% 自动提醒 |
| `T_PLUS_1_CHECK` | TRADE | `{}` | 当日买入不可当日卖出 |
| `LIMIT_PRICE_CHECK` | TRADE | `{}` | 涨跌停标的不建议交易 |
| `MIN_LIQUIDITY` | LIQUIDITY | `{"min_amount": 50000000}` | 日均成交额低于 5000 万不交易 |

#### 3.6.3 风控执行流程

```
信号产生 → 信号进入风控检查队列
    │
    ├─→ 遍历所有 enabled 的风控规则（按 priority 排序）
    │     │
    │     ├─→ BLOCK 规则触发 → 信号标记为"风控拦截"
    │     │
    │     └─→ WARN 规则触发 → 信号保留，附加警告信息
    │
    └─→ 风控通过的信号 → 推送到信号中心
```

### 3.7 引擎适配层 (Engine Adapter)

#### 3.7.1 设计目标

Python 端定义引擎抽象接口，akquant 作为默认引擎，未来可以无缝替换。

#### 3.7.2 适配器接口

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass

    """回测结果标准输出"""
    total_return_pct: float
    annualized_return: float
    max_drawdown_pct: float
    sharpe_ratio: float
    sortino_ratio: float
    calmar_ratio: float
    win_rate: float
    profit_factor: float
    total_trades: int
    avg_trade_return_pct: float
    volatility: float
    excess_return_pct: float | None = None
    alpha: float | None = None
    beta: float | None = None
    information_ratio: float | None = None
    tracking_error: float | None = None
    var_95: float | None = None
    cvar_95: float | None = None
    max_consecutive_losses: int | None = None
    equity_curve: list[dict] | None = None
    trades: list[dict] | None = None
    report_html_path: str | None = None

@dataclass
class SignalResult:
    """信号生成标准输出"""
    signals: list[dict]

@dataclass
class GridSearchResult:
    """参数优化标准输出"""
    best_params: dict
    best_score: float
    best_result: BacktestResult | None = None
    all_results: list[dict] | None = None

@dataclass
class WalkForwardResult:
    """Walk-forward 分析标准输出"""
    folds: list[dict]
    aggregate: dict
    report_html_path: str | None = None


class EngineAdapter(ABC):
    """回测引擎适配器抽象基类"""

    @abstractmethod
    def run_backtest(
        self,
        strategy_class: type,
        data: "pd.DataFrame",
        params: dict,
        config: dict,
    ) -> BacktestResult:
        """执行回测"""
        ...

    @abstractmethod
    def run_signal(
        self,
        strategy_class: type,
        data: "pd.DataFrame",
        params: dict,
        signal_date: str,
        current_positions: list[dict] | None = None,
    ) -> SignalResult:
        """生成信号"""
        ...

    @abstractmethod
```

### 3.8 内置策略模板

#### 3.8.1 双均线交叉 (sma_cross)

| 项目 | 说明 |
|---|---|
| 适用场景 | 趋势跟踪，适合 ETF 和龙头股 |
| 买入规则 | MA(短期) 上穿 MA(长期)（金叉） |
| 卖出规则 | MA(短期) 下穿 MA(长期)（死叉） 或 触发止损 |
| 默认参数 | `sma_short=5, sma_long=20, stop_loss_pct=0.05` |

**entry_rules JSON**：
```json
{
    "operator": "AND",
    "conditions": [
        {
            "type": "factor_compare",
            "factor": "MA",
            "params": {"period": "{{sma_short}}"},
            "comparator": "cross_up",
            "target_factor": "MA",
            "target_params": {"period": "{{sma_long}}"}
        }
    ]
}
```

**exit_rules JSON**：
```json
{
    "operator": "OR",
    "conditions": [
        {
            "type": "factor_compare",
            "factor": "MA",
            "params": {"period": "{{sma_short}}"},
            "comparator": "cross_down",
            "target_factor": "MA",
            "target_params": {"period": "{{sma_long}}"}
        },
        {
            "type": "stop_loss",
            "pct": "{{stop_loss_pct}}"
        }
    ]
}
```

#### 3.8.2 MACD 趋势突破 (macd_breakout)

| 项目 | 说明 |
|---|---|
| 适用场景 | 中短线波段交易 |
| 买入规则 | MACD 金叉 AND 成交量放大 1.5 倍 AND 收盘价 > MA(20) |
| 卖出规则 | MACD 死叉 OR 跌破 MA(20) OR 盈利 10% |
| 默认参数 | `macd_fast=12, macd_slow=26, macd_signal=9, vol_ratio=1.5, profit_target=0.10` |

#### 3.8.3 低估值价值策略 (value_pick)

| 项目 | 说明 |
|---|---|
| 适用场景 | 中长期价值投资（Phase 2） |
| 选股条件 | PE_TTM < 行业中位数 AND ROE > 15% AND 营收增速 > 0 |
| 调仓频率 | 月度调仓，持有前 20 只 |
| 默认参数 | `max_pe=30, min_roe=0.15, top_n=20` |

#### 3.8.4 小市值动量策略 (small_cap_momentum)

| 项目 | 说明 |
|---|---|
| 适用场景 | A 股特色小市值效应（Phase 2） |
| 选股条件 | 流通市值 50-300 亿 + 非 ST + 近 20 日涨幅 > 5% |
| 调仓频率 | 周度调仓，持有前 10 只 |
| 默认参数 | `min_market_cap=50e8, max_market_cap=300e8, momentum_days=20, top_n=10` |

#### 3.8.5 北向资金跟随策略 (northbound_follow)

| 项目 | 说明 |
|---|---|
| 适用场景 | 跟随外资动向的保守策略（Phase 2） |
| 买入规则 | 北向资金连续 3 日净流入 AND 个股被北向加仓 |
| 卖出规则 | 北向资金连续 2 日减仓 |
| 默认参数 | `consecutive_days=3, sell_consecutive=2` |

> **注意**：策略 3、4、5 依赖基本面/情绪面数据（财务、北向资金），MVP 阶段先实现策略 1、2。

## 9. 风险与注意事项

### 9.1 投资风险

1. **回测不等于未来**：历史有效 ≠ 未来有效。策略的衰变是常态，需要持续监控
2. **过拟合**：参数优化最容易过拟合。防范：Walk-forward 验证 + 参数鲁棒性检查
3. **幸存者偏差**：回测时使用的股票池是当前存续的股票，已经剔除了退市的
4. **流动性风险**：小市值策略回测收益可能很好，但实盘时冲击成本很高

### 9.2 技术风险

1. **akquant 版本稳定性**：akquant 是较新的项目，API 可能变动。建议锁定版本
2. **数据传递性能**：Java 向 Python 传递大量行情数据时，需注意 JSON 序列化的性能瓶颈，大数据量使用文件传递模式
3. **Python 服务运维**：需要考虑进程守护、异常重启。建议用 systemd 或 supervisor 管理
4. **规则解析兼容性**：可视化规则 JSON 到 akquant Strategy 的动态转换需要充分测试

### 9.3 合规提醒

- 所有数据仅用于个人研究，不得商用
- 不从事任何违反证券法规的活动
- 系统产生的信号仅供参考，投资决策由用户自行负责
