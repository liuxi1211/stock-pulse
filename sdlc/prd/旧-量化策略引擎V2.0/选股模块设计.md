# 6. 选股模块设计

## 6.1 架构概述

```
┌──────────────────────────────────────────────────────────┐
│  每日收盘后（批量预计算）                                   │
│                                                          │
│  Java 遍历股票池 ──→ POST /api/indicators/compute ──→ Python │
│                     ←── 返回指标值字典 ←──              │
│  Java 存入 stock_indicator_daily 表                      │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│  用户选股时（实时查询）                                    │
│                                                          │
│  前端条件编辑器（复用因子注册表）                            │
│      │                                                   │
│      │  JSON 条件                                        │
│      ▼                                                   │
│  Java ConditionSqlBuilder → SQL WHERE 子句               │
│      │                                                   │
│      ▼                                                   │
│  SELECT ... FROM stock_indicator_daily WHERE ...          │
│      │                                                   │
│      ▼                                                   │
│  返回匹配的股票列表                                        │
└──────────────────────────────────────────────────────────┘
```

**核心分工：**

| 层 | 职责 | 不做什么 |
|---|---|---|
| Python | 接收 OHLCV → 调 TA-Lib → 返回指标值 | 不管存储、不管查询、不管筛选逻辑 |
| Java | 存指标值、条件转 SQL、查询、展示 | 不碰 TA-Lib 计算 |
| 前端 | 复用因子注册表 + 条件编辑器 | 和策略配置器共用一套 UI |

## 6.2 支持的筛选条件

### 6.2.1 条件分类

所有条件复用策略配置的 JSON Schema（表达式节点 + compare 条件树），无需定义新格式。

| 分类 | 典型条件 | 表达式形态 |
|---|---|---|
| **因子 vs 阈值** | RSI < 30, MACD柱 > 0, ADX > 25 | `factor` vs `value` |
| **因子 vs 因子** | MA5 > MA20, PLUS_DI > MINUS_DI | `factor` vs `factor` |
| **穿越事件** | MA5 上穿 MA20（金叉）, DIF 上穿 DEA | `cross_up` / `cross_down` |
| **算术组合** | volume > VOL_MA(20) × 2, (HIGH-LOW)/CLOSE > 0.03 | `factor` vs `op` 表达式 |
| **多条件组合** | 金叉 AND 放量 AND RSI < 70 | AND/OR 条件树 |

### 6.2.2 常用筛选场景

| 场景 | 条件 JSON | 预计算表列引用 |
|---|---|---|
| 均线多头排列 | `ma_5 > ma_10 AND ma_10 > ma_20 AND ma_20 > ma_60` | 多个 compare AND |
| 双均线金叉 | `ma_5 cross_up ma_20` | 当日 + 前一日自 JOIN |
| MACD 金叉 | `macd_dif cross_up macd_dea` | 同上 |
| RSI 超卖 | `rsi_14 < 30` | 简单比较 |
| 放量突破 | `volume > vol_ma_20 * 2` | 算术表达式 |
| 布林带收窄后突破 | `boll_upper - boll_lower < close * 0.05 AND close > boll_upper` | 算术 + 比较 |
| DMI 多头 | `plus_di_14 > minus_di_14 AND adx_14 > 25` | 因子 vs 因子 + 阈值 |
| KDJ 超买 | `kdj_j > 80` | 多输出指标取值 |
| 创 N 日新高 | `close >= high_nday_max_60` | 需扩展列（见 6.2.3） |

### 6.2.3 扩展指标列（非 TA-Lib，Java 侧计算）

除了 Python 算的 TA-Lib 指标，还有一类条件是纯 SQL 可算的统计值：

| 扩展列 | 计算方式 | 用途 |
|---|---|---|
| `change_pct` | `(close - pre_close) / pre_close * 100` | 涨跌幅筛选 |
| `amplitude` | `(high - low) / pre_close * 100` | 振幅筛选 |
| `turnover_rate` | 来自行情数据源 | 换手率筛选 |
| `high_nday_max_20` | `MAX(high) OVER (ORDER BY trade_date ROWS 19 PRECEDING)` | 创 20 日新高 |
| `high_nday_max_60` | 同上，窗口 59 | 创 60 日新高 |
| `low_nday_min_20` | `MIN(low) OVER (... WINDOW 19)` | 创 20 日新低 |
| `vol_ratio` | `volume / vol_ma_5` | 量比 |
| `ma5_slope` | `(ma_5 - LAG(ma_5, 3) OVER (...)) / LAG(ma_5, 3) * 100` | 均线斜率（方向性） |

这些不需要 Python 计算，Java 写入指标表时用 SQL 窗口函数直接算。

### 6.2.4 时间维度：快照模式 vs 区间模式

**第一版只支持快照模式（Snapshot）**：所有条件在指定日期同一天求值。

```
screen_date = 2026-05-27

条件：MA5 > MA20 AND RSI < 70 AND volume > vol_ma_20 * 2

含义：在 2026-05-27 这一天，三个条件同时成立
```

cross_up / cross_down 在快照模式下自动包含前一交易日对比：

```
MA5 cross_up MA20：
  当天（5/27）：ma_5 > ma_20
  前一天（5/26）：ma_5 <= ma_20
  → 判定 5/27 发生金叉
```

**第二期扩展区间模式（Window）**：给 compare 条件加可选 `within` 字段：

```json
{
  "type": "compare",
  "left": { "factor": "MA", "params": {"timeperiod": 5} },
  "comparator": "cross_up",
  "right": { "factor": "MA", "params": {"timeperiod": 20} },
  "within": 5
}
```

含义：最近 5 个交易日内，出现过 MA5 上穿 MA20。SQL 实现为 `EXISTS` 子查询。

### 6.3.4 Python 侧实现要点

```python
@app.post("/api/indicators/compute")
async def compute_indicators(req: ComputeRequest):
    # 1. 将 ohlcvs 转为 numpy arrays
    import numpy as np
    close = np.array([b["close"] for b in req.ohlcvs], dtype=float)
    high  = np.array([b["high"]  for b in req.ohlcvs], dtype=float)
    low   = np.array([b["low"]   for b in req.ohlcvs], dtype=float)
    open_ = np.array([b["open"]  for b in req.ohlcvs], dtype=float)
    volume = np.array([b["volume"] for b in req.ohlcvs], dtype=float)

    # 2. 按 indicators 列表逐个计算
    values = {}
    for ind in req.indicators:
        key = ind["factorKey"]
        reg = FACTOR_REGISTRY[key]
        params = ind.get("params", {})

        if reg["func"] is None:
            # 价格/成交量直取
            field = reg["inputs"][0]
            col_name = _to_column_name(key, params)
            values[col_name] = float({"close": close, "high": high, "low": low, "volume": volume}[field][-1])
            continue

        # 构建 TA-Lib 输入
        input_map = {"close": close, "high": high, "low": low, "open": open_, "volume": volume}
        inputs = {f: input_map[f] for f in reg["inputs"]}
        result = reg["func"](**inputs, **params)

        if reg["multi_output"]:
            names = _multi_output_names(key, params)
            for i, name in enumerate(names):
                values[name] = float(result[i][-1])
        else:
            col_name = _to_column_name(key, params)
            values[col_name] = float(result[-1])

    return {"stock": req.stock, "values": values}
```

列名生成函数：

```python
def _to_column_name(factor_key: str, params: dict) -> str:
    """因子引用 → 预计算表列名"""
    SINGLE_PARAM_KEYS = {"MA", "EMA", "RSI", "ATR", "ADX", "PLUS_DI", "MINUS_DI", "WILLR", "CCI"}
    if factor_key in SINGLE_PARAM_KEYS and "timeperiod" in params:
        return f"{factor_key.lower()}_{params['timeperiod']}"
    if factor_key == "VOL_MA" and "timeperiod" in params:
        return f"vol_ma_{params['timeperiod']}"
    if factor_key == "VOL_EMA" and "timeperiod" in params:
        return f"vol_ema_{params['timeperiod']}"
    if factor_key == "OBV":
        return "obv"
    return factor_key.lower()


def _multi_output_names(factor_key: str, params: dict) -> list[str]:
    """多输出指标 → 列名列表"""
    if factor_key == "MACD":
        return ["macd_dif", "macd_dea", "macd_hist"]
    if factor_key == "KDJ":
        return ["kdj_k", "kdj_d", "kdj_j"]
    if factor_key == "BOLL":
        return ["boll_upper", "boll_mid", "boll_lower"]
    raise ValueError(f"Unknown multi-output factor: {factor_key}")
```

| 指标 | 参数 | 预热 bar 数 |

| MACD(12,26,9) | 默认 | 33 |

### 6.4.3 数据写入流程

```
每日收盘后（如 15:30）：

1. Java 从数据源获取全市场当日行情 + 最近 300 天历史 OHLCV
2. 遍历股票池（可并行），对每只股票：
   a. 组装 POST /api/indicators/compute 请求
   b. 调用 Python 计算指标
   c. 收到响应后，INSERT INTO stock_indicator_daily
3. 写入完成后，用 SQL 窗口函数更新扩展列：
   UPDATE stock_indicator_daily s
   JOIN (
       SELECT id,
              (close - pre_close) / pre_close * 100 AS change_pct,
              (high - low) / pre_close * 100         AS amplitude,
              volume / vol_ma_5                       AS vol_ratio
       FROM stock_indicator_daily
       WHERE trade_date = CURDATE()
   ) calc ON s.id = calc.id
   SET s.change_pct = calc.change_pct,
       s.amplitude  = calc.amplitude,
       s.vol_ratio  = calc.vol_ratio;
```

## 6.5 条件转 SQL 核心代码

### 6.5.1 因子表达式 → 列名映射

Java 侧维护一个映射表，将 JSON 表达式节点映射到预计算表的列名：

```java
public class FactorColumnMapper {

    /**
     * 将因子表达式节点映射到 stock_indicator_daily 表的列名
     *
     * @param node 表达式节点，如 {"factor": "MA", "params": {"timeperiod": 5}}
     * @return 列名，如 "ma_5"
     */
    public static String toColumn(Map<String, Object> node) {
        String factor = (String) node.get("factor");
        Map<String, Object> params = (Map<String, Object>) node.getOrDefault("params", Map.of());

        switch (factor) {
            // 单参数指标 — 列名 = factor_参数值
            case "MA":    return "ma_" + params.get("timeperiod");
            case "EMA":   return "ema_" + params.get("timeperiod");
            case "RSI":   return "rsi_" + params.get("timeperiod");
            case "ATR":   return "atr_" + params.get("timeperiod");
            case "ADX":   return "adx_" + params.get("timeperiod");
            case "PLUS_DI":  return "plus_di_" + params.get("timeperiod");
            case "MINUS_DI": return "minus_di_" + params.get("timeperiod");
            case "WILLR":    return "willr_" + params.get("timeperiod");
            case "CCI":      return "cci_" + params.get("timeperiod");
            case "VOL_MA":   return "vol_ma_" + params.get("timeperiod");
            case "VOL_EMA":  return "vol_ema_" + params.get("timeperiod");

            // 多输出指标 — 需要 output_index
            case "MACD": return macdColumn(getOutputIndex(node));
            case "KDJ":  return kdjColumn(getOutputIndex(node));
            case "BOLL": return bollColumn(getOutputIndex(node));

            // 无参数指标
            case "OBV":    return "obv";
            case "CLOSE":  return "close";
            case "HIGH":   return "high";
            case "LOW":    return "low";
            case "VOLUME": return "volume";

            default:
                throw new IllegalArgumentException("Unknown factor: " + factor);
        }
    }

    private static int getOutputIndex(Map<String, Object> node) {
        return ((Number) node.getOrDefault("output_index", 0)).intValue();
    }

    private static String macdColumn(int outputIndex) {
        switch (outputIndex) {
            case 0:  return "macd_dif";
            case 1:  return "macd_dea";
            case 2:  return "macd_hist";
            default: return "macd_dif";
        }
    }

    private static String kdjColumn(int outputIndex) {
```

### 6.5.2 表达式节点 → SQL 表达式

```java
public class ExpressionSqlBuilder {

    /**
     * 将表达式节点转为 SQL 表达式片段
     *
     * @param node 表达式节点
     * @param alias 表别名（"a" = 当日, "p" = 前一日）
     * @return SQL 表达式，如 "a.ma_5", "(a.volume / a.vol_ma_20 * 2)"
     */
    public static String toSql(Map<String, Object> node, String alias) {
        // ① 静态值 → 直接返回数字
        if (node.containsKey("value")) {
            return String.valueOf(node.get("value"));
        }

        // ② 因子引用 → 表别名.列名
        if (node.containsKey("factor")) {
            String column = FactorColumnMapper.toColumn(node);
            return alias + "." + column;
        }

        // ③ 算术运算 → (left op right)
        if (node.containsKey("op")) {
            String op = (String) node.get("op");
            String leftSql = toSql((Map<String, Object>) node.get("left"), alias);
            String rightSql = toSql((Map<String, Object>) node.get("right"), alias);
            return "(" + leftSql + " " + op + " " + rightSql + ")";
        }

        throw new IllegalArgumentException("Unknown expression node: " + node);
    }
}
```

### 6.5.3 条件树 → SQL WHERE

```java
public class ConditionSqlBuilder {

    /**
     * 将条件树转为完整的 SQL 查询
     *
     * @param conditions 条件树 JSON（与策略配置 signals 同结构）
     * @param screenDate 筛选日期
     * @param additionalWhere 额外 WHERE 条件（如股票池过滤）
     * @return 完整 SQL
     */
    public static String buildQuery(Map<String, Object> conditions, String screenDate,
                                     String additionalWhere) {
        // 检测条件树中是否包含 cross_up / cross_down
        boolean needPrevJoin = containsCross(conditions);

        if (needPrevJoin) {
            return buildCrossQuery(conditions, screenDate, additionalWhere);
        } else {
            return buildSimpleQuery(conditions, screenDate, additionalWhere);
        }
    }

    // ── 简单查询（无穿越）──

    private static String buildSimpleQuery(Map<String, Object> conditions,
                                            String screenDate, String additionalWhere) {
        String whereClause = buildWhere(conditions, "a");
        StringBuilder sql = new StringBuilder();
        sql.append("SELECT a.stock_code, a.close, a.change_pct, a.volume ");
        sql.append("FROM stock_indicator_daily a ");
        sql.append("WHERE a.trade_date = '").append(screenDate).append("' ");
        if (additionalWhere != null) {
            sql.append("AND ").append(additionalWhere).append(" ");
        }
        sql.append("AND (").append(whereClause).append(")");
        return sql.toString();
    }

    // ── 穿越查询（需 JOIN 前一日）──

    private static String buildCrossQuery(Map<String, Object> conditions,
                                           String screenDate, String additionalWhere) {
        String whereClause = buildWhere(conditions, "a");
        String prevWhereClause = buildCrossPrevWhere(conditions, "p");

        StringBuilder sql = new StringBuilder();
        sql.append("SELECT a.stock_code, a.close, a.change_pct, a.volume ");
        sql.append("FROM stock_indicator_daily a ");
        sql.append("JOIN stock_indicator_daily p ");
        sql.append("  ON a.stock_code = p.stock_code ");
        sql.append("  AND p.trade_date = (");
        sql.append("    SELECT MAX(t.trade_date) FROM stock_indicator_daily t ");
        sql.append("    WHERE t.stock_code = a.stock_code AND t.trade_date < '").append(screenDate).append("'");
        sql.append("  ) ");
        sql.append("WHERE a.trade_date = '").append(screenDate).append("' ");
        if (additionalWhere != null) {
            sql.append("AND ").append(additionalWhere).append(" ");
        }
        sql.append("AND (").append(whereClause).append(") ");
        sql.append("AND (").append(prevWhereClause).append(")");
        return sql.toString();
    }

    // ── 条件树递归 ──

    /**
     * 递归构建 WHERE 子句
     * AND/OR → SQL AND/OR
     * compare → left_expr comparator right_expr
     * cross_up → 当日条件（前一日的条件单独生成）
     */
    private static String buildWhere(Map<String, Object> condNode, String alias) {
        String operator = (String) condNode.get("operator");

        // 逻辑组合节点
        if ("AND".equals(operator) || "OR".equals(operator)) {
            List<Map<String, Object>> conditions = (List<Map<String, Object>>) condNode.get("conditions");
            StringJoiner joiner = new StringJoiner(
                "AND".equals(operator) ? " AND " : " OR ",
                "(", ")"
            );
            for (Map<String, Object> c : conditions) {
                joiner.add(buildWhere(c, alias));
            }
            return joiner.toString();
        }

        // 叶子节点：compare
        if (!"compare".equals(condNode.get("type"))) {
            return "1=1"; // 未知类型，安全降级
        }

        String comparator = (String) condNode.get("comparator");
        Map<String, Object> left = (Map<String, Object>) condNode.get("left");
        Map<String, Object> right = (Map<String, Object>) condNode.get("right");

        // 穿越信号 — 只生成当日部分（a.left > a.right）
        // 前一日部分由 buildCrossPrevWhere 单独生成
        if ("cross_up".equals(comparator)) {
            return ExpressionSqlBuilder.toSql(left, alias) + " > "
                 + ExpressionSqlBuilder.toSql(right, alias);
        }
        if ("cross_down".equals(comparator)) {
            return ExpressionSqlBuilder.toSql(left, alias) + " < "
                 + ExpressionSqlBuilder.toSql(right, alias);
        }

        // 常规比较
        return ExpressionSqlBuilder.toSql(left, alias) + " " + comparator + " "
             + ExpressionSqlBuilder.toSql(right, alias);
    }

    /**
     * 构建前一日穿越条件
     * cross_up → p.left <= p.right
     * cross_down → p.left >= p.right
     * 非穿越条件不生成（返回 1=1）
     */
    private static String buildCrossPrevWhere(Map<String, Object> condNode, String alias) {
        String operator = (String) condNode.get("operator");

        if ("AND".equals(operator) || "OR".equals(operator)) {
            List<Map<String, Object>> conditions = (List<Map<String, Object>>) condNode.get("conditions");
            StringJoiner joiner = new StringJoiner(
                "AND".equals(operator) ? " AND " : " OR ",
                "(", ")"
            );
            for (Map<String, Object> c : conditions) {
                joiner.add(buildCrossPrevWhere(c, alias));
            }
            return joiner.toString();
        }

        if (!"compare".equals(condNode.get("type"))) {
            return "1=1";
        }

        String comparator = (String) condNode.get("comparator");
        Map<String, Object> left = (Map<String, Object>) condNode.get("left");
        Map<String, Object> right = (Map<String, Object>) condNode.get("right");

        switch (comparator) {
            case "cross_up":
                return ExpressionSqlBuilder.toSql(left, alias) + " <= "
                     + ExpressionSqlBuilder.toSql(right, alias);
            case "cross_down":
                return ExpressionSqlBuilder.toSql(left, alias) + " >= "
                     + ExpressionSqlBuilder.toSql(right, alias);
            default:
                return "1=1"; // 非穿越条件，前一日无需约束
        }
    }

    // ── 工具方法 ──

    private static boolean containsCross(Map<String, Object> condNode) {
        String operator = (String) condNode.get("operator");
        if ("AND".equals(operator) || "OR".equals(operator)) {
            List<Map<String, Object>> conditions = (List<Map<String, Object>>) condNode.get("conditions");
            return conditions.stream().anyMatch(ConditionSqlBuilder::containsCross);
        }
        String comparator = (String) condNode.get("comparator");
        return "cross_up".equals(comparator) || "cross_down".equals(comparator);
    }
}
```

> 注意：OR 分支中的 cross_up 前一日条件 `p.macd_dif <= p.macd_dea` 在语义上是"MACD DIF 前一日 <= DEA"，但 RSI 那个分支不需要前一日约束。当前实现为整棵树生成统一的前一日 WHERE，OR 中的非穿越分支生成 `1=1`（恒真）。实际效果：`(p.macd_dif <= p.macd_dea) OR (1=1)` = 恒真，这会导致 MACD 金叉的前一日约束在 OR 中失效。
>
> **正确做法**：cross_up 的前一日约束必须和当日约束绑定在一起，不能拆成两个独立 WHERE。见 6.5.5 修正方案。

### 6.5.5 OR 中含穿越的修正方案

当 OR 分支中包含 cross_up/cross_down 时，需要将"当日条件 + 前一日条件"绑定为一个整体：

```java
/**
 * 修正版：为每个 cross 生成绑定条件组
 *
 * cross_up 的完整条件 = 当日(left > right) AND 前一日(left <= right)
 * 这两组必须在同一个 AND/OR 层级中绑定
 */
private static String buildWhereWithPrev(Map<String, Object> condNode, String todayAlias, String prevAlias) {
    String operator = (String) condNode.get("operator");

    if ("AND".equals(operator) || "OR".equals(operator)) {
        List<Map<String, Object>> conditions = (List<Map<String, Object>>) condNode.get("conditions");
        StringJoiner joiner = new StringJoiner(
            "AND".equals(operator) ? " AND " : " OR ",
            "(", ")"
        );
        for (Map<String, Object> c : conditions) {
            joiner.add(buildWhereWithPrev(c, todayAlias, prevAlias));
        }
        return joiner.toString();
    }

    if (!"compare".equals(condNode.get("type"))) {
        return "1=1";
    }

    String comparator = (String) condNode.get("comparator");
    Map<String, Object> left = (Map<String, Object>) condNode.get("left");
    Map<String, Object> right = (Map<String, Object>) condNode.get("right");

    switch (comparator) {
        case "cross_up":
            // 绑定当日 + 前一日
            return "(" + ExpressionSqlBuilder.toSql(left, todayAlias) + " > "
                       + ExpressionSqlBuilder.toSql(right, todayAlias)
                       + " AND "
                       + ExpressionSqlBuilder.toSql(left, prevAlias) + " <= "
                       + ExpressionSqlBuilder.toSql(right, prevAlias) + ")";
        case "cross_down":
            return "(" + ExpressionSqlBuilder.toSql(left, todayAlias) + " < "
                       + ExpressionSqlBuilder.toSql(right, todayAlias)
                       + " AND "
                       + ExpressionSqlBuilder.toSql(left, prevAlias) + " >= "
                       + ExpressionSqlBuilder.toSql(right, prevAlias) + ")";
        default:
            return ExpressionSqlBuilder.toSql(left, todayAlias) + " " + comparator + " "
                 + ExpressionSqlBuilder.toSql(right, todayAlias);
    }
}
```

## 6.6 预计算调度设计

### 6.6.1 调度时序

```
15:00  收盘
15:05  数据源更新完毕（行情数据可用）
15:10  Java 定时任务启动
       ├── Step 1: 获取全市场当日行情 + 近 300 天历史 OHLCV
       ├── Step 2: 分批并行调用 Python 计算指标
       │     每批 50 只股票，并行度 = CPU 核数
       │     预计 100 批 × 1s/批 ≈ 2 分钟
       ├── Step 3: 批量 INSERT INTO stock_indicator_daily
       └── Step 4: 更新扩展统计列（change_pct, amplitude, vol_ratio）
15:15  预计算完成，选股查询可用
```

### 6.6.2 增量 vs 全量

| 策略 | 适用场景 | 说明 |
|---|---|---|
| **每日增量**（推荐） | 日常更新 | 每天只计算当天的新指标值 |
| 全量重建 | 新增指标列、修复数据 | 重新计算所有历史数据，低频操作 |

增量计算时，Python 仍需要 300 天 OHLCV（因为 MA250 需要历史数据预热），但只返回最后一根 bar 的指标值。

### 6.6.3 自定义参数处理

预计算表覆盖常用参数组合（24 种），如果用户筛选时用了非标参数（如 MA(13)）：

1. **首选**：检查预计算表中是否有近似列，提示用户使用预置参数
2. **降级**：实时调用 Python 计算，结果不存预计算表
3. **扩展**：如果某自定义参数被高频使用，将其加入预计算列表
