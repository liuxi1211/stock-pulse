# Phase 1 — 量化策略引擎 验收文档

> 对应 PRD：`001-量化策略引擎PRD.md`
> 目标：验证 Phase 1 的核心功能 — 因子库、策略规则配置、回测、信号生成、持仓管理、风控系统是否按需求完成。
> 技术架构：Java Spring Boot（数据管理 / 业务系统 / UI）+ Python FastAPI（计算引擎 / akquant 回测引擎）

## 3. 手动验证清单（必走流程）

### 3.1 必走流程 1 — 策略从配置到回测到信号

```
1. 登录系统 → 进入策略管理页面
2. 创建新策略 "双均线交叉策略"
   - 选股范围：沪深 300 成分股
   - 前置过滤：剔除 ST、次新股、日均成交额 < 5000 万
   - 买入规则：MA(5) 上穿 MA(20) AND 成交量 > VOL_MA(5) × 1.5
   - 卖出规则：MA(5) 下穿 MA(20) OR 止损 5% OR 止盈 10%
   - 仓位管理：单票最大 8%，最多持仓 10 只
   - 参数：short=5, long=20
3. 保存策略（状态 DRAFT）
4. 配置回测：2020-01-01 至 2025-12-31，初始资金 10 万，基准沪深 300
5. 执行回测，等待完成
6. 查看回测报告：
   - 检查权益曲线 vs 基准曲线
   - 检查最大回撤区间标注
   - 检查交易统计（胜率、盈亏比）
   - 检查月度收益热力图
   - 检查风险指标（Sharpe、Sortino、Calmar）
7. 如果策略收益超过基准，设置为 ACTIVE 状态
8. 修改策略参数（如 short=3, long=15）
   - 确认自动创建新版本快照
9. 执行 Walk-forward 分析：训练 2 年 / 测试 1 年
10. 查看 Walk-forward 报告，检查样本内外相关性
11. 设置策略为 ACTIVE，等待每日收盘后自动生成信号
12. 次日收盘后查看信号中心，确认产生 BUY/SELL 信号
13. 点击信号 → 查看信号详情（触发原因、建议仓位、风控检查结果）
14. 将信号转为模拟交易 → 查看持仓、交易记录、每日净值
15. 观察风控规则：当持仓触发止损/止盈时的信号行为
```

### 3.2 必走流程 2 — 参数优化与策略对比

```
1. 选择一个已回测的策略
2. 配置 Grid Search 参数空间：short_ma ∈[3,5,8,10, short_ma ∈ [15,20,30]
3. 执行参数优化
4. 查看最优参数组合，按夏普比率排序
5. 使用最优参数重新回测
6. 对比不同参数组合的回测结果
7. 选择最优参数版本，设置为 ACTIVE
```

### 3.3 必走流程 3 — 风控系统验证

```
1. 配置一组风控规则（单票 8%、总仓位 90%、行业 25%、单日亏损 1%、最大回撤 8%）
2. 模拟一系列极端行情（如连续下跌触发最大回撤）
3. 观察风控检查结果：信号被拦截/警告
4. 触发 T+1 规则：当日买入的股票当日触发卖出信号，被标记限制
5. 触发涨跌停检查：涨跌停的股票触发信号被标记不建议交易
```
