# 仪表盘模块 PRD：现有页面功能补齐（D1-D5）

> **模块**：012 · 仪表盘（项目总览入口，路由 `/`，已在侧边栏菜单）
> **状态**：规划中，待 `index_daily` 表与板块行情模块（015）行业排行能力就绪后进入实施
> **来源**：拆分自 [`主要功能模块整体设计计划.md`](./../../.trae/documents/主要功能模块整体设计计划.md) §2.1 仪表盘补齐（D1-D6）、§1.1 页面形态、§7 假设与约束、§10 附录接口核查
> **原则**：从**用户可见功能角度**描述需求；数据依赖直接写表名/接口名；不写代码实现；接口必须来自 `tushare 接口汇总.md` 且 ≤2000 积分。

## 0. 拆分说明（为什么单独立项）

总计划文档把仪表盘补齐混排在 Phase E（现有 3 模块补齐）的 E1-E6 子任务清单里，与行情中心、自选股补齐混排，没有独立的「用户可见功能」展开。本文把 **仪表盘 5 项功能补齐**（D1-D5，D6 砍掉）单独拎出独立成稿，因为：

- **仪表盘定位独特**：路由 `/` 是项目总览入口，老股民开盘第一眼必看，5 项补齐围绕同一页面、同一用户场景（开盘前快速总览），天然是一组。
- **5 项补齐相互关联**：D1 真实指数 / D2 市场温度 / D3 换手率榜 / D4 K 线指标 / D5 板块概览 同屏展示，前端改动都集中在 `dashboard.html` + `dashboard.js`，单独立项便于统一交互与布局。
- **D6 北向卡片砍掉决策需集中记录**：原计划用 `moneyflow_hsgt` 接口，核查后发现该接口不在 `tushare 接口汇总.md` 中，整体砍掉（北向能力改由 016 资金流向模块的「北向 Tab」用 `hk_hold` 替代实现，不在仪表盘承担）。

> **不在本 PRD 范围**：行情中心补齐（S1-S8）、自选股补齐（W1-W10）、板块行情页面本身（015）、资金流向页面本身（016）--各自独立排期；本 PRD 只描述仪表盘页面 `/` 的功能补齐。

## 1. 背景与目标

### 1.1 仪表盘定位

- **项目总览入口**：路由 `/`，侧边栏「主要功能」第 1 项，用户登录后默认落地页。
- **老股民开盘第一眼**：老股民习惯开盘前快速浏览「大盘指数 / 市场情绪 / 个股榜单 / 板块轮动 / 自选监控」五项总览，仪表盘是**一站式总览面板**，不深入个股细节，细节靠下钻到行情中心/板块行情/个股诊断。
- **现有页面已实现但缺真实数据**：仪表盘页面骨架已存在（`dashboard.html` + `dashboard.js`），但有 5 处功能缺口（详见 §1.2）。

### 1.2 现状与功能缺口

| # | 现状 | 缺口 |
|---|---|---|
| D1 | 大盘指数卡片显示 3368.07 等固定值（`MarketServiceImpl.java:31-44` 硬编码 mock） | 需接真实 `index_daily` 数据，覆盖沪/深/创/科 4 个指数 |
| D2 | 无市场温度卡片 | 需补涨家数/跌家数/平家数/涨停数/跌停数 5 个数字 |
| D3 | `/market/ranking` 仅涨幅榜/跌幅榜/成交额榜 3 榜 | 需扩展加换手率榜 TOP 10 |
| D4 | K 线图仅 MA5/MA10/MA20 三线主图叠加 | 需加 MACD/KDJ/RSI/BOLL 副图切换 |
| D5 | 无板块概览卡片 | 需加「今日行业板块涨跌 TOP 5」卡片 |
| ~~D6~~ | ~~无北向资金卡片~~ | ❌ 砍掉（`moneyflow_hsgt` 接口不存在，详见 §1.3 非目标） |

### 1.3 非目标（明确不做）

| 非目标项 | 原因 | 备注 |
|---|---|---|
| **D6 北向资金卡片** | 原计划用 `moneyflow_hsgt` 接口，**该接口不在 `tushare 接口汇总.md` 中，不存在**；替代方案 `hk_hold`（沪深股通持股明细）数据粒度是「个股持股明细」非「每日净买入」，不适合做仪表盘总览小卡片 | 北向能力改由 016 资金流向模块「北向 Tab」承担，用 `hk_hold` 做持股比例曲线，不在仪表盘 |
| **概念板块/题材快照** | `concept` / `concept_detail` 接口不存在 | 仪表盘 D5 板块概览只做申万一级行业，不做概念板块 |
| **行业板块资金净流入** | `moneyflow_dc` 接口不存在 | 不引入派生聚合数据（数据单源约束） |
| **大盘指数日内分时** | 不在本期范围 | 仪表盘大盘指数用日 K 收盘价，不做日内分时 |
| **多市场对比（港股/美股）** | 不在本期范围 | 仅 A 股沪/深/创/科 4 大指数 |
| **北向资金十大重仓股聚合** | 同 D6 原因 | 改由 016 资金流向模块承担 |
| **盘中实时刷新推送** | 不在本期范围 | 仪表盘采用页面手动刷新或定时轮询（每日收盘后数据更新），不做 WebSocket 推送 |

### 1.4 总体目标

- **目标一**：仪表盘首屏展示**真实**大盘指数数据（沪/深/创/科 4 个指数），消除 3368.07 等固定值。
- **目标二**：仪表盘首屏展示市场温度（涨/跌/平家数 + 涨停/跌停数），让用户一眼判断市场情绪。
- **目标三**：榜单区扩展换手率榜 TOP 10，与现有涨幅/跌幅/成交额三榜同区切换。
- **目标四**：K 线图支持 MACD / KDJ / RSI / BOLL 副图切换，与个股诊断 Tab 1 指标集对齐。
- **目标五**：首页右侧加「今日行业板块涨跌 TOP 5」卡片，让用户在仪表盘就能看到板块轮动方向。
- **质量准绳**：`.trae/rules/系统设计理念(用法建议).md` + A 股实战派认可（不接受"纸面能跑、实战不能用"）。

## 2. 典型用户场景

### 2.1 主场景：老股民开盘前看仪表盘

**时间**：交易日 9:15 集合竞价前 / 9:30 开盘后 / 15:00 收盘后三个时点
**用户**：3 年以上经验的 A 股散户/老股民，习惯用同花顺/东方财富看盘，希望在本系统内一站式总览

**使用路径**：
1. 登录后落地 `/`（仪表盘）
2. **第一眼**看顶部 4 个大盘指数卡片（D1）：
   - 上证指数 / 深证成指 / 创业板指 / 科创50
   - 每张卡片显示：指数名 + 最新点位 + 涨跌额 + 涨跌幅（红涨绿跌）
   - 老股民判断「今日大盘是红是绿、有没有破关键点位」
3. **第二眼**看市场温度卡片（D2）：
   - 涨家数 / 跌家数 / 平家数 / 涨停数 / 跌停数 5 个数字
   - 老股民判断「普涨普跌还是分化、有没有涨停潮」
4. **第三眼**看榜单区（D3）：
   - 切换涨幅榜 / 跌幅榜 / 成交额榜 / 换手率榜 4 个 Tab
   - 换手率榜看「今天谁最活跃、资金在炒谁」
   - 点击榜单中的股票代码 -> 跳转个股诊断（依赖 017 模块）
5. **第四眼**看 K 线图（D4）：
   - 默认看上证指数日 K + MA5/10/20
   - 切到 MACD 副图看柱状图（红柱转绿柱判断短期动能变化）
   - 切到 BOLL 看上下轨（判断是否触破上下轨）
   - 老股民判断「大盘技术面信号」
6. **第五眼**看右侧板块概览卡片（D5）：
   - 今日行业板块涨跌 TOP 5（领涨 5 + 领跌 5）
   - 老股民判断「今日领涨板块是哪个、要不要切到板块行情看成分股」
   - 点击「查看全部」-> 跳转 `/page/sector`（板块行情页）

**收盘后复看**：15:00 后刷新页面，看当日最终的大盘指数 / 市场温度 / 榜单 / 板块涨跌，确认今日盘面整体走势。

### 2.2 异常场景：数据未拉取或拉取失败

- **指数数据未拉取**：D1 卡片显示「暂无数据」占位，**不显示硬编码 mock**（这是 D1 的核心验收点）
- **市场温度数据缺失**：D2 卡片显示「数据加载中...」或「暂无数据」
- **板块行情模块未上线**：D5 卡片显示「板块行情功能开发中」占位，不阻塞其他 4 项功能
- **K 线指标算法异常**：D4 K 线主图正常渲染，副图显示「指标计算异常」占位

## 3. 功能需求

### FR-1：大盘指数真实数据展示（对应 D1）

#### 3.1.1 功能描述

仪表盘顶部展示 4 个 A 股核心大盘指数卡片：

| 指数名 | 指数代码 | 说明 |
|---|---|---|
| 上证指数 | `000001.SH` | 沪市大盘 |
| 深证成指 | `399001.SZ` | 深市大盘 |
| 创业板指 | `399006.SZ` | 创业板大盘 |
| 科创50 | `000688.SH` | 科创板大盘 |

每张卡片显示：
- 指数名称（中文）
- 最新点位（保留 2 位小数）
- 涨跌额（与前收对比，红涨绿跌）
- 涨跌幅（百分比，红涨绿跌）
- 数据日期（YYYY-MM-DD，表明数据时效）

#### 3.1.2 用户价值

- **消除硬编码 mock**：当前固定显示 3368.07 等假数据，老股民一眼识破"假"，失去信任。改真实数据后，仪表盘成为「可信的总览面板」。
- **4 个指数覆盖 A 股全市场**：沪/深/创/科 4 大指数是 A 股老股民最熟悉的 4 个指数，覆盖主板/创业板/科创板三大板块，无需切到其他页面就能一眼看全。

#### 3.1.3 数据依赖

- **依赖 `index_daily` 表**，由 tushare `index_daily` 接口采集（2000 起积分，已确认在 `tushare 接口汇总.md` 中）。
- `index_daily` 表关键字段：`ts_code`（指数代码）/ `trade_date`（交易日）/ `close`（收盘价）/ `pre_close`（前收）/ `change`（涨跌额）/ `pct_chg`（涨跌幅）/ `open` / `high` / `low` / `vol` / `amount`。
- 4 个指数代码通过 `ts_code` 字段过滤查询：`WHERE ts_code IN ('000001.SH','399001.SZ','399006.SZ','000688.SH') AND trade_date = <最新交易日>`。
- 交易日历由 `trade_cal` 表（已有数据）确定，避免周末/节假日显示旧数据。

#### 3.1.4 前端改造点

- `dashboard.html`：4 个指数卡片容器结构保持，移除硬编码 mock 文本占位。
- `dashboard.js`：页面加载时调用 `/api/index-daily/latest?codes=000001.SH,399001.SZ,399006.SZ,000688.SH`（接口名暂定，以最终落地为准），渲染 4 个卡片。
- 涨跌色统一用 `theme.css` 的红涨绿跌色变量（`.stock-up` / `.stock-down`）。

#### 3.1.5 后端改造点

- **移除 `MarketServiceImpl.java:31-44` 的硬编码 mock 大盘指数数据**（这是核心改造点）。
- 改为查询 `index_daily` 表，按 `ts_code` 过滤 4 个大盘指数，取最新 `trade_date` 数据返回。
- 新建 `IndexDailyController` 暴露查询接口（GET，返回 `ApiResponse<List<IndexDailyDTO>>`），支持按 `ts_code` 列表与 `trade_date` 范围查询。
- `index_daily` 表的 schema / DO / Mapper / TushareApiEnum / DTO / Service / 定时拉取任务由「后端数据底座」相关任务统一交付，仪表盘侧只消费。

### FR-2：市场温度卡片（对应 D2）

#### 3.2.1 功能描述

仪表盘首屏加一张「市场温度」卡片，展示当日 A 股市场情绪 5 个数字：

| 数字 | 含义 | 计算规则 |
|---|---|---|
| 涨家数 | 当日上涨股票数 | `COUNT(*) FROM daily_quote WHERE trade_date = <今日> AND pct_chg > 0` |
| 跌家数 | 当日下跌股票数 | `COUNT(*) FROM daily_quote WHERE trade_date = <今日> AND pct_chg < 0` |
| 平家数 | 当日平盘股票数 | `COUNT(*) FROM daily_quote WHERE trade_date = <今日> AND pct_chg = 0` |
| 涨停数 | 当日涨停股票数 | `COUNT(*) FROM daily_quote WHERE trade_date = <今日> AND pct_chg >= 9.9`（按板块区分 10%/20%/30% 涨跌停规则） |
| 跌停数 | 当日跌停股票数 | `COUNT(*) FROM daily_quote WHERE trade_date = <今日> AND pct_chg <= -9.9` |

> 涨停/跌停按板块区分涨跌停规则：主板 10%、创业板/科创板 20%、ST 5%（具体阈值以后端实现为准，建议从 `stock_basic` 表的 `list_status` 字段判断 ST 状态 + 市场板块判断）。

#### 3.2.2 用户价值

- **一眼看市场情绪**：5 个数字直观反映「普涨普跌还是分化」「有没有涨停潮/跌停潮」。
- 老股民经验：涨停数 > 50 = 市场情绪亢奋；跌停数 > 50 = 恐慌；涨跌比 > 3:1 = 普涨；< 1:3 = 普跌。

#### 3.2.3 数据依赖

- **依赖 `daily_quote` 表**（已有数据，由 tushare `daily` 接口采集，2000 积分）。
- `daily_quote` 表关键字段：`ts_code` / `trade_date` / `pct_chg`（涨跌幅）/ `close` / `open` / `high` / `low` / `vol`。
- 按当日 `trade_date` 聚合统计 5 个数字，**无需新表**。
- 涨停/跌停判断需结合 `stock_basic` 表的 `market`（市场板块：主板/创业板/科创板/北交所）与 `name`（含 ST 判断）字段。

#### 3.2.4 前端改造点

- `dashboard.html`：在 4 个指数卡片下方加 1 张「市场温度」卡片（横向 5 格布局）。
- `dashboard.js`：调用市场温度接口，渲染 5 个数字 + 颜色（涨家数红色、跌家数绿色、平家数灰色、涨停数红色加粗、跌停数绿色加粗）。

#### 3.2.5 后端改造点

- 扩展 `MarketController` 加市场温度接口（如 `/api/market/temperature?tradeDate=<今日>`），返回 `{up_count, down_count, flat_count, limit_up_count, limit_down_count}`。
- `MarketService` 加 `getMarketTemperature(tradeDate)` 方法，查询 `daily_quote` 表按 `pct_chg` 分组统计。
- **无需新建表**，纯查询聚合。

### FR-3：换手率榜 TOP 10（对应 D3）

#### 3.3.1 功能描述

在现有 `/market/ranking` 接口支持的「涨幅榜 / 跌幅榜 / 成交额榜」3 个榜单基础上，**扩展加换手率榜 TOP 10**：

| 列 | 说明 |
|---|---|
| 排名 | 1-10 |
| 股票代码 | `ts_code`（如 `000001.SZ`），点击跳转个股诊断 `/page/stock-detail/{code}` |
| 股票名称 | 从 `stock_basic` 表取 `name` |
| 最新价 | `close` |
| 涨跌幅 | `pct_chg`（红涨绿跌） |
| 换手率 | `turnover_rate`（百分比，降序排序） |
| 成交额 | `amount`（亿元，辅助参考） |

#### 3.3.2 用户价值

- **识别资金活跃个股**：换手率高 = 资金进出活跃，老股民看换手率榜找「今天资金在炒谁」「有没有主力资金进场」。
- 老股民经验：换手率 > 10% = 高度活跃；> 20% = 异常活跃（可能有重大消息）；< 1% = 极度冷门。

#### 3.3.3 数据依赖

- **依赖 `daily_quote` 表 + `daily_basic` 表 JOIN 查询**（两张表均已有数据）。
- `daily_basic` 表关键字段：`ts_code` / `trade_date` / `turnover_rate`（换手率）/ `pe` / `pb` / `total_mv`（总市值）/ `circ_mv`（流通市值）。
- `daily_quote` 表提供 `pct_chg` / `close` / `amount`，`daily_basic` 表提供 `turnover_rate`，两表通过 `ts_code + trade_date` 关联。
- **无需新建表**，纯 JOIN 查询。

#### 3.3.4 前端改造点

- `dashboard.html`：榜单区 Tab 切换从 3 个 Tab 扩展为 4 个 Tab（涨幅榜 / 跌幅榜 / 成交额榜 / 换手率榜）。
- `dashboard.js`：扩展榜单渲染逻辑，支持 `type=turnover` 参数调用 `/market/ranking` 接口。

#### 3.3.5 后端改造点

- 扩展 `MarketController.ranking` 接口，新增 `type=turnover` 分支，返回 TOP 10 换手率个股。
- `MarketService.getRanking(type, tradeDate, limit)` 方法加换手率分支，JOIN `daily_quote` + `daily_basic` 两表，按 `turnover_rate` 降序取 TOP 10。
- **无需新建表**，纯 JOIN 查询扩展。

### FR-4：K 线图技术指标切换（对应 D4）

#### 3.4.1 功能描述

仪表盘 K 线图当前仅支持主图叠加 MA5/MA10/MA20 三条均线，**扩展加 4 个副图指标切换**：

| 指标 | 显示方式 | 说明 |
|---|---|---|
| MA（已有） | 主图叠加 | MA5 / MA10 / MA20，可选扩展加 MA60 |
| MACD | 副图 | DIF 线 / DEA 线 / MACD 柱（红绿柱） |
| KDJ | 副图 | K 线 / D 线 / J 线（J = 3K - 2D 自算） |
| RSI | 副图 | RSI6 / RSI12 / RSI24 三条线 |
| BOLL | 主图叠加 | 上轨 / 中轨（MA20）/ 下轨三条线 + 价格通道带状区域 |

用户可在指标切换区选择主图指标（MA / BOLL）与副图指标（MACD / KDJ / RSI），切换后 K 线图实时重渲染。

#### 3.4.2 用户价值

- **仪表盘就能看技术指标组合**：老股民在仪表盘快速看大盘技术面，不用切到个股诊断 Tab 1。
- **与个股诊断 Tab 1 指标集对齐**：仪表盘 K 线指标与个股诊断 Tab 1（K 线技术面）共用同一套指标算法（基于 `akquant.talib` 函数），保证口径一致。

#### 3.4.3 数据依赖

- **无新数据依赖**：K 线图已有 `daily_quote` 表的 OHLCV 数据即可算所有指标。
- `daily_quote` 表字段：`ts_code` / `trade_date` / `open` / `high` / `low` / `close` / `vol`。
- 指标算法：
  - MA：`akquant.talib.MA(close, timeperiod=5/10/20/60)`
  - MACD：`akquant.talib.MACD(close, fastperiod=12, slowperiod=26, signalperiod=9)` 返回 `(dif, dea, hist)`
  - KDJ：`akquant.talib.STOCH(high, low, close, fastk_period=9, slowk_period=3, slowd_period=3)` 返回 `(K, D)`，**J = 3K - 2D 需自算**
  - RSI：`akquant.talib.RSI(close, timeperiod=6/12/24)`
  - BOLL：`akquant.talib.BBANDS(close, timeperiod=20, nbdevup=2.0, nbdevdn=2.0)` 返回 `(upper, middle, lower)`

#### 3.4.4 前端改造点

- `dashboard.js` 的 `renderKline` 函数扩展：
  - 加指标切换 UI（主图指标下拉 + 副图指标下拉）
  - 加副图渲染逻辑（LightweightCharts 的 HistogramStyle / LineSeries 渲染 MACD 柱、KDJ 线、RSI 线）
  - 加 BOLL 主图叠加渲染（带状区域 + 上下轨线）
- 复用 `ChartsTheme` 主题色，三主题（azure/mist/cyber）切换后指标线颜色同步变化。

#### 3.4.5 后端改造点

- **无后端改造**（后端已支持 `daily_quote` 数据查询，K 线指标算法在前端实现或前端调后端计算接口）。
- 可选：若前端算性能不足，后端可加 `/api/indicator/compute` 接口接收 OHLCV 数组返回指标值，但一期建议前端算。

### FR-5：板块概览快照卡片（对应 D5）

#### 3.5.1 功能描述

仪表盘右侧加「今日行业板块涨跌 TOP 5」卡片，展示当日申万一级行业板块涨跌排行：

| 区域 | 内容 |
|---|---|
| 领涨 TOP 5 | 行业名 + 今日涨跌幅（红色，降序） |
| 领跌 TOP 5 | 行业名 + 今日涨跌幅（绿色，升序） |
| 底部 | 「查看全部」链接 -> 跳转 `/page/sector`（板块行情页） |

行业涨跌幅基于申万一级行业指数（`index_daily` 表中 `ts_code` 形如 `801010.SI` 的申万行业指数）的 `pct_chg` 字段。

#### 3.5.2 用户价值

- **仪表盘就能看到板块轮动方向**：老股民进仪表盘第一眼就能看到「今日哪个行业领涨、哪个领跌」，不用切到板块行情页。
- **引导下钻**：点击「查看全部」跳转板块行情页看完整 28 个行业排行 + 成分股。

#### 3.5.3 数据依赖

- **依赖 `index_daily` 表的申万行业指数数据**，由 tushare `index_daily` 接口采集（2000 起积分）。
- **依赖 `sw_industry` 表的申万一级行业分类**（已有数据），提供行业代码与行业名称的映射（如 `801010.SI` -> `农林牧渔`）。
- 行业指数代码格式：`801xxx.SI`（申万一级行业指数，共 28 个）。
- 查询逻辑：取 `index_daily` 表中 `ts_code LIKE '801%.SI' AND trade_date = <今日>` 的数据，按 `pct_chg` 降序取前 5（领涨）+ 升序取前 5（领跌）。

#### 3.5.4 前端改造点

- `dashboard.html`：右侧加「板块概览」卡片（领涨 5 + 领跌 5 + 查看全部链接）。
- `dashboard.js`：调用板块行情模块的行业排行接口，渲染 10 个行业条目。
- 点击「查看全部」跳转 `/page/sector`。

#### 3.5.5 后端改造点

- **复用板块行情模块（015）的行业排行 Controller**，无需新建接口。
- 板块行情模块（015）提供 `/api/industry/ranking?tradeDate=<今日>&limit=5` 接口，返回申万一级行业涨跌 TOP N，仪表盘 D5 直接调用。
- 仪表盘侧**不实现**行业排行逻辑，全部委托给 015 模块。

## 4. 非功能需求

### 4.1 性能

- **首屏加载时间 ≤ 2 秒**（含 4 个指数卡片 + 市场温度 + 榜单 + K 线图 + 板块概览，数据异步并行加载）。
- 4 个指数卡片、市场温度、榜单、K 线图、板块概览各自独立异步加载，不互相阻塞。
- K 线图指标切换响应时间 ≤ 500ms（前端算）。
- 后端接口响应时间：
  - 指数查询接口 ≤ 200ms（单次 4 个 ts_code 查询）
  - 市场温度接口 ≤ 500ms（聚合统计 daily_quote 全表当日数据，需 `trade_date` 索引）
  - 换手率榜接口 ≤ 300ms（JOIN daily_quote + daily_basic，需 `ts_code + trade_date` 联合索引）
  - 板块概览接口（复用 015）≤ 300ms

### 4.2 主题统一

- 仪表盘所有图表（K 线图、榜单涨跌色、卡片涨跌色）必须复用 `ChartsTheme` 主题色变量。
- 三主题（azure / mist / cyber）切换后，所有图表颜色同步变化，不允许出现「切换主题后图表颜色不更新」的 bug。
- 涨跌色统一用 `theme.css` 的 `.stock-up`（红）/ `.stock-down`（绿）类名，不硬编码颜色值。

### 4.3 数据降级

- **指数数据未拉取时**：D1 卡片显示「暂无数据」占位，**绝不显示硬编码 mock**（这是 D1 的核心验收点）。
- **市场温度数据缺失时**：D2 卡片显示「数据加载中...」或「暂无数据」。
- **换手率榜数据缺失时**：D3 榜单显示「暂无数据」占位，不显示空表格。
- **板块行情模块（015）未上线时**：D5 卡片显示「板块行情功能开发中」占位，不阻塞其他 4 项功能。
- **K 线指标算法异常时**：D4 K 线主图正常渲染，副图显示「指标计算异常」占位。
- **数据时效提示**：所有卡片显示「数据日期」字段，让用户清楚看到数据是何日的（避免周末看到周五数据以为今日数据）。

### 4.4 兼容性

- 浏览器：Chrome / Edge / Firefox 最新版（不兼容 IE）。
- 分辨率：最低 1366x768，推荐 1920x1080。
- 移动端：不做响应式适配（仪表盘是 PC 端专业工具，不在手机使用）。

## 7. 风险与注意事项

### 7.1 数据时效风险

- **指数数据延迟**：`index_daily` 接口每日收盘后采集，盘中仪表盘显示的是前一日收盘数据，需在卡片上明确标注「数据日期」避免误导。
- **降级方案**：盘中访问仪表盘时，指数卡片显示「截至 YYYY-MM-DD 收盘」提示。

### 7.2 涨跌停判断准确性

- **板块涨跌停规则不同**：主板 10%、创业板/科创板 20%、ST 5%、北交所 30%。FR-2 市场温度的涨停数/跌停数需按板块区分判断，不能一刀切用 `pct_chg >= 9.9`。
- **建议实现**：后端 `MarketService.getMarketTemperature` 方法先 JOIN `stock_basic` 表取 `market` 与 `name` 字段，按板块规则判断涨停/跌停。

### 7.3 K 线指标算法口径一致性

- **仪表盘 D4 与个股诊断 Tab 1 必须共用同一套指标算法**（基于 `akquant.talib` 函数），保证同一只股票在两处看到的 MACD/KDJ/RSI/BOLL 数值完全一致。
- **KDJ 的 J 值自算**：`akquant.talib.STOCH` 返回 `(K, D)`，J 值需自算 `J = 3K - 2D`，仪表盘 D4 与个股诊断 Tab 1 都按此口径。

### 7.4 北向资金卡片砍掉的替代路径

- D6 北向资金卡片砍掉后，北向资金能力**不在仪表盘承担**，改由 016 资金流向模块的「北向 Tab」用 `hk_hold` 接口实现（持股比例曲线 + 十大重仓股聚合）。
- 仪表盘不提供北向资金入口，用户需从侧边栏「资金流向」-> 北向 Tab 查看。
