# K线接口真实数据实现 — 研发文档

## 一、现状分析

### 1.1 当前问题

`StockServiceImpl.getKlineData()` 是纯 mock 实现：
- 从内存 `stockMap` 取一个基准价，循环 90 天生成随机 `open/close/high/low/volume`
- 没有读取数据库中的 `daily_quote`、`adj_factor`、`dividend` 表
- 没有做复权计算，没有日线→周月线聚合

### 1.2 已有数据基础

| 数据 | 表 | 对应 Entity | 增量同步状态 |
|------|-----|-------------|-------------|
| 日线行情 | `daily_quote` | `DailyQuoteDO` | ✅ 每日 16:00 定时同步 |
| 复权因子 | `adj_factor` | `AdjFactorDO` | ✅ 每日 16:00 定时同步 |
| 分红送股 | `dividend` | `DividendDO` | ✅ 每日 16:00 定时同步 |
| 交易日历 | `trade_cal` | — | ✅ 已有 |

### 1.3 已有查询能力

| Service | 方法 | 用途 |
|---------|------|------|
| `IDailyQuoteService` | `queryByCodeAndDateRange(tsCode, startDate, endDate)` | 按股票+日期范围查日线 |
| `IAdjFactorService` | `queryByCodeAndDateRange(tsCode, startDate, endDate)` | 按股票+日期范围查复权因子 |

两个方法返回的是 DTO 对象（`DailyQuoteDTO`、`AdjFactorDTO`），字段与 DO 一致。

`KlineDataVO` 当前字段：`date, open, close, low, high, volume`

**无需修改**，这个结构已经满足日/周/月 K 线展示需求。`date` 字段：
- 日K → 交易日日期 `yyyyMMdd`
- 周K → 该周最后一个交易日日期 `yyyyMMdd`
- 月K → 该月最后一个交易日日期 `yyyyMMdd`

## 三、核心计算逻辑

### 3.1 前复权价格公式

> 来源：`doc/复权公式.md`

```
前复权价格 = 原始价格 × 当日复权因子
即: P_adj = P_raw × adj
```

仅价格（open/high/low/close）需要复权，**成交量/成交额永远使用原始值**。

### 3.2 日K线计算

日K线最简单——直接取原始日线数据，对价格字段做前复权：

```
对于每个交易日:
  open_adj  = daily_quote.open  × adj_factor.adjFactor
  close_adj = daily_quote.close × adj_factor.adjFactor
  high_adj  = daily_quote.high  × adj_factor.adjFactor
  low_adj   = daily_quote.low   × adj_factor.adjFactor
  volume    = daily_quote.vol（原始值）
```

### 3.3 周K线聚合规则

> 来源：`doc/K线.md`，与同花顺规则一致

按自然周分组（周一~周五），每周生成一根 K 线：

```
对于每个自然周内所有交易日:
  open   = 当周第一个交易日的前复权开盘价
  close  = 当周最后一个交易日的前复权收盘价
  high   = 当周所有交易日前复权最高价中的最大值
  low    = 当周所有交易日前复权最低价中的最小值
  volume = 当周所有交易日原始成交量之和
  date   = 当周最后一个交易日的日期
```

**关键**：先对每个交易日分别复权，再取 high/low 的最值。不能先聚合再复权。

### 3.4 月K线聚合规则

与周K逻辑完全一致，仅分组维度改为自然月：

```
对于每个自然月内所有交易日:
  open   = 当月第一个交易日的前复权开盘价
  close  = 当月最后一个交易日的前复权收盘价
  high   = 当月所有交易日前复权最高价中的最大值
  low    = 当月所有交易日前复权最低价中的最小值
  volume = 当月所有交易日原始成交量之和
  date   = 当月最后一个交易日的日期
```

### 3.5 分组日期计算

```java
// 周分组key = 该自然周的周五日期
LocalDate friday = date.plusDays(5 - date.getDayOfWeek().getValue());

// 月分组key = 该自然月的最后一天
LocalDate lastDayOfMonth = date.withDayOfMonth(date.lengthOfMonth());
```

注意：分组 key 用于分组，`date` 字段取的是该周期内**实际最后一个交易日**的日期，不一定是周五或月末。

## 四、缓存策略

### 4.1 设计原则

> 来源：`doc/K线.md` 缓存永不过期 + 复权因子变化失效机制

| 数据类型 | 缓存策略 |
|----------|----------|
| 历史已完成周期（过去的周/月） | 永久缓存，不自动过期 |
| 当天日K / 当周周K / 当月月K | 短时间过期（1小时），盘中会更新 |
| 全部缓存 | 复权因子变化时主动失效 |

### 4.2 实现方案

当前项目使用 Caffeine + Spring `@Cacheable`，全局配置 5 分钟过期。K 线缓存需要独立配置：

**方案**：在 `CacheConfig` 中为 `kline` 缓存单独配置更长的过期时间（如 24 小时），配合手动失效机制。

缓存 key 设计：
```
kline::{stockCode}::daily
kline::{stockCode}::weekly
kline::{stockCode}::monthly
```

### 4.3 缓存失效触发

两种检测机制并行：

1. **dividend 表提前预警**（优化手段）：每日同步 dividend 数据时，检查是否有当日除权的股票，如有则主动清除该股票的 K 线缓存
2. **adj_factor 变化兜底检测**（最终保障）：每日同步 adj_factor 后，对比该股票最新复权因子是否变化，变化则清除缓存

## 五、数据流设计

### 5.1 查询流程

```
前端请求 GET /api/stocks/kline/600519?period=weekly
    │
    ▼
Controller 接收 stockCode + period
    │
    ▼
StockService.getKlineData(stockCode, period)
    │
    ├─ 1. 查缓存 kline::{stockCode}::{period}
    │     命中 → 直接返回
    │
    ├─ 2. 缓存未命中，查数据库
    │     ├─ DailyQuoteService.queryByCodeAndDateRange(tsCode, "上市日", "今天")
    │     └─ AdjFactorService.queryByCodeAndDateRange(tsCode, "上市日", "今天")
    │
    ├─ 3. 内存计算
    │     ├─ 建立 日期→复权因子 Map
    │     ├─ 对每日数据做前复权
    │     ├─ 按 period 聚合（daily 直接输出，weekly/monthly 分组聚合）
    │     └─ 生成 List<KlineDataVO>
    │
    ├─ 4. 写入缓存（带复权因子版本号）
    │
    └─ 5. 返回结果
```

### 5.2 stockCode → tsCode 转换

前端传入的是 6 位股票代码（如 `600519`），数据库中存储的是 Tushare 格式（如 `600519.SH`）。

需要做一次转换：
```java
// 在 StockBasicMapper 或 StockBasicService 中查询
String tsCode = stockBasicService.resolveTsCode(stockCode);
// 根据代码规则拼接：6开头→.SH，0/3开头→.SZ
```

## 七、KlineCalculator 工具类设计

### 7.1 类结构

```java
package com.arthur.stock.util;

public class KlineCalculator {

    /** 前复权价格：P_adj = P_raw × adj */
    public static BigDecimal forwardAdj(BigDecimal rawPrice, BigDecimal adjFactor);

    /** 将日线数据转换为前复权日K */
    public static List<KlineDataVO> buildDailyKline(
            List<DailyQuoteDTO> dailies,
            List<AdjFactorDTO> adjFactors);

    /** 将日线数据聚合为前复权周K */
    public static List<KlineDataVO> buildWeeklyKline(
            List<DailyQuoteDTO> dailies,
            List<AdjFactorDTO> adjFactors);

    /** 将日线数据聚合为前复权月K */
    public static List<KlineDataVO> buildMonthlyKline(
            List<DailyQuoteDTO> dailies,
            List<AdjFactorDTO> adjFactors);

    // ---- 内部辅助方法 ----

    /** 周分组 key：该自然周的周五日期字符串 */
    static String weekGroupKey(String tradeDate);

    /** 月分组 key：该自然月的最后一天日期字符串 */
    static String monthGroupKey(String tradeDate);
}
```

### 7.2 计算步骤（以周K为例）

```
输入: List<DailyQuoteDTO> dailies (按日期升序)
      List<AdjFactorDTO> adjFactors (按日期升序)

Step 1: 建立 Map<tradeDate, adjFactor>
Step 2: 按 weekGroupKey(tradeDate) 分组
Step 3: 每组内:
         - 按 tradeDate 排序确保交易日顺序
         - first = 组内第一个交易日
         - last  = 组内最后一个交易日
         - open  = first.open  × adjMap[first.tradeDate]
         - close = last.close  × adjMap[last.tradeDate]
         - high  = max(每个 d.high × adjMap[d.tradeDate])
         - low   = min(每个 d.low  × adjMap[d.tradeDate])
         - volume = sum(每个 d.vol)
         - date  = last.tradeDate
Step 4: 按日期升序排序输出
```

### 7.3 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 某交易日缺少复权因子 | 跳过该交易日（不应出现，做防御性处理） |
| 节假日导致某周只有 1 个交易日 | 正常聚合为 1 根 K 线 |
| 节假日导致某月只有几个交易日 | 正常聚合 |
| 新股上市第一天 | 只有 1 条日线数据，生成 1 根日K |

### 8.2 getKlineData 实现

```java
@Override
@Cacheable(value = CACHE_KLINE, key = "#stockCode + '::' + #period")
public List<KlineDataVO> getKlineData(String stockCode, String period) {
    // 1. stockCode → tsCode
    String tsCode = resolveTsCode(stockCode);

    // 2. 查询全部日线和复权因子
    String startDate = "19900101";  // 足够早的起始日
    String endDate = LocalDate.now().format(DateTimeFormatter.BASIC_ISO_DATE);
    List<DailyQuoteDTO> dailies = dailyQuoteService.queryByCodeAndDateRange(tsCode, startDate, endDate);
    List<AdjFactorDTO> adjFactors = adjFactorService.queryByCodeAndDateRange(tsCode, startDate, endDate);

    if (dailies.isEmpty()) {
        return Collections.emptyList();
    }

    // 3. 按周期计算
    return switch (period) {
        case "weekly"  -> KlineCalculator.buildWeeklyKline(dailies, adjFactors);
        case "monthly" -> KlineCalculator.buildMonthlyKline(dailies, adjFactors);
        default        -> KlineCalculator.buildDailyKline(dailies, adjFactors);
    };
}
```

### 8.3 stockCode 转换

简单规则即可，无需查库：

```java
private String resolveTsCode(String stockCode) {
    // 6开头 → .SH（上交所），0/3开头 → .SZ（深交所）
    String suffix = stockCode.startsWith("6") ? ".SH" : ".SZ";
    return stockCode + suffix;
}
```

## 九、性能考量

### 9.1 数据量估算

| 股票 | 上市年数 | 日线条数 | 内存占用(估算) |
|------|---------|---------|--------------|
| 老股（如平安银行） | ~30 年 | ~7,200 条 | < 1 MB |
| 新股 | ~1 年 | ~240 条 | < 0.1 MB |

单次全量查询 + 内存计算耗时 < 50ms，在 Caffeine 缓存加持下，首次查询后几乎零开销。

### 9.2 不做过度优化

- 不需要预计算周月线存表——数据量小，实时聚合足够快
- 不需要分页查询日线——全量加载到内存后一次聚合
- 不需要引入 Redis——单机 Caffeine 完全够用
