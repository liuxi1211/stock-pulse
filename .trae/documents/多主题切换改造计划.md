# 多主题切换改造计划

## 一、概述

将 stock-watcher 前端从"单一深色主题"改造为"三主题可切换",并同步更新 `01-frontend.md` 规范,使后续新页面开发有明确遵循。

**最终三套主题**(去掉现有深色):

| 主题 key | 名称 | 定位 | 是否默认 |
|---|---|---|---|
| `azure` | 云隙蔚蓝 | 明亮、轻盈、护眼浅色 | ✅ 默认 |
| `mist` | 晨雾青瓷 | 冷静、低饱和、长时间盯盘护眼 | 否 |
| `cyber` | 深空赛博 | 近黑深蓝底 + 霓虹青/品红高光 + 扫描线,最强科技感 | 否 |

**已确认的决策**:
1. 交互:顶栏单个图标按钮,点击在三主题间循环切换,图标随主题变化。
2. 持久化:纯前端 `localStorage`,head 内联阻塞脚本防首屏闪烁,不改 16 个 HTML 模板、不动后端。
3. 图表:`charts-theme.js` 改造为读取 CSS 变量,切换主题时重绘所有 ECharts/ Lightweight Charts 实例。
4. 默认主题:`azure`(云隙蔚蓝)。
5. 范围:完整改造(架构 + 全部 CSS 硬编码清理 + 图表跟随 + 规范文档),一次性交付。

---

## 二、现状分析(基于探索)

### 2.1 架构红利

所有颜色已走 `theme.css :root` 的 CSS 变量,`components.css` 几乎零硬编码。**切多主题的核心改动几乎只在 `theme.css` 一个文件**——把 `:root` 拆成三套 `[data-theme="..."]` 选择器,所有 `var()` 引用自动跟随。

### 2.2 必须清理的硬编码(按工作量排序)

| 文件 | 问题 | 行数 |
|---|---|---|
| `strategy.css` | 48 处 `rgba(主题色,透明度)` 写死(蓝/青/紫/红/黄系),需改 `color-mix` | 1984 行,最重 |
| `factor-library.css` | 30 处 `rgba()`,其中 17 处主题色硬编码(蓝/青/紫系),13 处带 var fallback | 291 行 |
| `condition-tree-builder.css` | **引用了不存在的 `--accent-red` token**(4 处,行 110/111/139/140),fallback 色值深浅混乱 | 305 行 |
| `components.css` | 1 处 `#10b981` 硬编码(行 543,`.card-title-icon.fall`) | 1299 行 |
| `screener.css` | 几乎零问题,仅 3 处 `#fff`(中性白,可保留) | 910 行 |
| `charts-theme.js` | **全部硬编码深色**,不读 CSS 变量,需整体改造 | 163 行 |

### 2.3 关键落点

- **顶栏切换按钮**:`fragments/common.html` 行 142(通知按钮后、用户 dropdown 前),复用现有 `.top-bar-btn` 样式。
- **`<html>` 标签**:16 个模板的 `<html>` 都未设 `data-theme`。**不改模板**,由 head 内联脚本在解析期设置 `document.documentElement.dataset.theme`,避免闪烁与回流。
- **公共 JS**:`common.js` 用全局 `StockApp` 对象 + DOM Ready 模式,主题切换逻辑照此风格融入,不引入模块化。

## 三、改动清单(分 6 步,按依赖顺序)

### 步骤 1:重构 `theme.css` —— 三主题 token 体系

**文件**:`stock-watcher/src/main/resources/static/css/theme.css`

**做法**:
1. 删除现有单一深色 `:root` 里的所有颜色/阴影/渐变/模糊/背景光晕 token(保留 layout/radius/transitions/typography 这些主题无关的)。
2. 拆为四段:
   - `:root` —— 只放**主题无关 token**:`--radius-*`、`--sidebar-width`、`--navbar-height`、`--content-padding`、`--transition-*`、`--font-*`。再加一组**图表专用 token**(供 charts-theme.js 读取,见步骤 3):
     - `--chart-text`、`--chart-grid`、`--chart-axis`、`--chart-tooltip-bg`、`--chart-tooltip-border`、`--chart-split-line`
     - `--ambient-1` / `--ambient-2` / `--ambient-3`(三处氛围光颜色,供 `body::before` 用)
   - `[data-theme="azure"]` —— 云隙蔚蓝(默认浅色):浅蓝灰底、靛蓝主色、深墨字、柔和投影
   - `[data-theme="mist"]` —— 晨雾青瓷:雾白底、青瓷绿主色、墨蓝字
   - `[data-theme="cyber"]` —— 深空赛博:近黑深蓝底、霓虹青+品红高光、扫描线氛围
3. 每套主题都完整覆盖:`--bg-*`、`--text-*`、`--border-*`、`--accent-*`、`--rise-*`、`--fall-*`、`--shadow-*`、`--gradient-*`、`--blur-*`、`--ambient-*`、`--chart-*`。
4. 改写 `body::before`:背景光晕颜色从硬编码 `rgba(59,130,246,...)` 改为 `var(--ambient-1)` / `var(--ambient-2)` / `var(--ambient-3)`,三处径向渐变透明度也 token 化。
5. **深空赛博专属**:在 `[data-theme="cyber"]` 下额外加 `body::after` 扫描线动效(极淡的水平移动渐变条,`prefers-reduced-motion` 下关闭)——这是该主题的标志性元素。
6. **阴影语义反转**:浅色主题(`azure`/`mist`)的 `--shadow-*` 用 `rgba(0,0,0,0.08~0.14)` 柔和投影;`cyber` 用 `rgba(0,0,0,0.4~0.6)` + 霓虹 glow。

**颜色取值**:沿用原型 `light-mock.html` 已验证的 azure/mist 配色;cyber 参考"近黑深蓝 `#070b1a` + 霓虹青 `#00e5ff` + 品红 `#ff2d92`"。

### 步骤 2:清理页面 CSS 硬编码

**文件与具体操作**:

| 文件 | 操作 |
|---|---|
| `components.css` | 行 543:`#10b981` → `var(--accent-green)` |
| `strategy.css` | 48 处 `rgba(59,130,246,N%)` 等改为 `color-mix(in srgb, var(--accent-blue) N%, transparent)`;同法处理青/紫/红/黄系。6 处 `#fff` 保留(按钮文字两种主题都该反白) |
| `factor-library.css` | 17 处主题色 `rgba()` 同法改 `color-mix`;行 9-12 的 `--src-*` 自定义 token 保留在页面级(来源色不随主题变) |
| `condition-tree-builder.css` | ① 4 处 `var(--accent-red, ...)` → `var(--rise-color)`(修掉引用不存在 token 的隐患);② 行 304 纯硬编码 `rgba(255,255,255,0.02)` → `var(--bg-tertiary)` 或删除;③ 其余 `var(--token, fallback)` 里的 fallback 统一去掉,强制走全局 token |
| `screener.css` | 无需改(3 处 `#fff` 为中性白,保留) |

**原则**:凡是跟随主题变的颜色一律 `var()` 或 `color-mix(in srgb, var(...) N%, transparent)`;`#fff`/纯黑阴影/遮罩这些中性色保留。

### 步骤 3:改造 `charts-theme.js` —— 图表跟随主题

**文件**:`stock-watcher/src/main/resources/static/js/charts-theme.js`

**做法**:
1. `getEChartsTheme()` / `getKlineTheme()` 内部所有硬编码色值,改为**运行时读取 CSS 变量**:
   ```js
   const css = getComputedStyle(document.documentElement);
   const textSecondary = css.getPropertyValue('--text-secondary').trim();
   const chartGrid = css.getPropertyValue('--chart-grid').trim();
   // ...
   ```
2. `getChartColors()` 改为读 `--accent-blue/-purple/-cyan/-green/-yellow/-rise-color` 等,组成调色板数组。
3. `getBarGradient()` / `getAreaGradient()` 的涨跌色改读 `--rise-color` / `--fall-color`。
4. 新增 `ChartsTheme.applyTheme()` 方法:重新读取当前 CSS 变量,遍历**已注册的图表实例**(维护一个 `WeakSet` 或数组),对每个实例 `setOption(theme, true)` 或 Lightweight Charts 的 `applyOptions`。
5. 新增图表实例时,页面 JS 需调用 `ChartsTheme.register(chart)` 注册(或沿用现有页面里图表初始化的集中点,统一注册)。**降低侵入**:若现有页面图表分散,可提供一个 `ChartsTheme.resyncAll()`,由各页面在收到主题切换事件时自行调用——具体注册机制在实现时按现有���面图表代码结构选最小侵入方案。

### 步骤 4:新增 `theme.js` + 顶栏切换按钮

**新文件**:`stock-watcher/src/main/resources/static/js/theme.js`

**职责**:
1. `ThemeManager` 全局对象(沿用 `StockApp` 全局风格),提供:
   - `THEMES = ['azure', 'mist', 'cyber']` 及对应图标(`bi-sun` / `bi-droplet-half` / `bi-lightning-charge`)。
   - `init()`:DOM Ready 时读 `localStorage.getItem('sp-theme')`,默认 `'azure'`,设 `document.documentElement.dataset.theme`,绑定切换按钮点击事件。
   - `cycle()`:点击按钮,主题在数组里循环,写 localStorage,更新按钮图标,派发 `window` 的 `theme:changed` 事件(供 charts-theme.js 监听重绘)。
   - `get()`:返回当前主题 key。
2. 切换按钮加 `aria-label="切换主题"`,支持键盘操作。
3. 在 `common.js` DOM Ready 里调用 `ThemeManager.init()`;或在 `fragments/common.html` 的 `scripts` fragment 末尾引入 `theme.js` 并自初始化。

**防闪烁内联脚本**:在 `fragments/common.html` 的 `head(title)` fragment 最前面(所有 CSS link 之前)加一段内联阻塞 JS:
```html
<script>
  (function(){
    var t = localStorage.getItem('sp-theme') || 'azure';
    document.documentElement.setAttribute('data-theme', t);
  })();
</script>
```
确保首屏解析时 `<html>` 已带正确 `data-theme`,避免"先深色后跳浅色"的闪烁。

**顶栏按钮**:`fragments/common.html` 行 142 后插入:
```html
<button class="top-bar-btn" id="themeToggle" type="button"
        aria-label="切换主题" title="切换主题">
    <i class="bi bi-sun"></i>
</button>
```
图标由 `theme.js` 根据 `data-theme` 动态替换。

## 四、假设与决策

1. **去掉现有深色主题**:不保留为第 4 套。若未来要加回,只需在 theme.css 新增一个 `[data-theme="dark-legacy"]` 块,架构已支持。
2. **`--accent-red` 隐患**:condition-tree-builder.css 引用了 theme.css 不存在的 `--accent-red`。决定:**不新增该 token**,改为引用语义正确的 `--rise-color`(红色涨色),消除隐患。
3. **`--src-*` 来源色 token**(factor-library.css 行 9-12):来源色(akquant/tushare 等)是业务语义色,不随主题变,**保留在页面级 CSS**,不上提到全局。
4. **中性色保留**:`#fff`(按钮反白字)、纯黑 `rgba(0,0,0,...)` 阴影/遮罩、`transparent` 这些跨主题通用,不强行 var 化,避免过度改造。
5. **图表实例注册机制**:实现时优先看现有页面图表初始化代码的集中程度。若集中(如都在各自页面 JS 的 init 函数),在 `ThemeManager` 派发 `theme:changed` 事件后,由各页面 JS 监听并调 `ChartsTheme.applyTheme()` 重绘;若分散,提供 `ChartsTheme.resyncAll()` 遍历注册表。**实现阶段以最小侵入为原则选定**,不在此预先锁死。
6. **不改后端、不改 16 个 HTML 模板的 `<html>` 标签**:全程前端方案。`data-theme` 由内联脚本 + `ThemeManager` 管理。
7. **`custom.css` 不动**:探索显示它仅 3 处中性 box-shadow,无主题色硬编码。
