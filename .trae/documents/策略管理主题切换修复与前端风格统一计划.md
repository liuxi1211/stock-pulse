## 二、Current State Analysis（现状分析）

### 2.1 主题切换机制（已厘清，正确）
- **方案**：W3C CSS 自定义属性 + `<html data-theme="dark|light">` 属性切换（非 Tailwind、非 class 切换、非多份 CSS）。
- **真相源**：[theme.css](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/css/theme.css) 在 `:root, [data-theme="dark"]`（L6-L93）和 `[data-theme="light"]`（L98-L162）下定义同名的 ~50 个 token，两套值。
- **切换器**：[theme.js](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/theme.js) 的 `ThemeManager` 单例：localStorage(`stockpulse_theme`) 持久化 + 系统偏好兜底 + `.theme-transition` 平滑过渡 + `themechange` CustomEvent 通知图表。
- **切换按钮**：[common.html L143-L145](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/templates/fragments/common.html#L143-L145) 的 `#themeToggleBtn` → 全局 `toggleTheme()`。

### 2.2 因子库（标杆，主题切换正确）
- 文件：[factor-library.css](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/css/factor-library.css)
- **正确做法**：L7-L14 只在 `:root, [data-theme="dark"]` 顶部**新增 4 个专属 token**（`--src-akquant`/`--src-tushare`/`--src-raw`/`--src-derived`），后续所有组件（`.stat-strip`/`.stat-card`/`.ftable`/`.detail-panel`）**全部用全局 `var(--bg-card)` 等**。
- 切换主题时全局变量改值 → 页面自动跟随。

### 2.3 策略管理（问题所在，主题切换失效）
- 文件：[strategy.css](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/css/strategy.css)（~1700 行，**323 处 `--str-` 引用**）
- **根因**：[strategy.css L1-L47](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/css/strategy.css#L1-L47) 在 `.strategy-page` 选择器作用域内**重定义了约 30 个 `--str-*` 硬编码字面量变量**（如 `--str-bg-primary: #0a0e17`），不是引用全局 var。
- **三连击**：① 字面量硬编码不随主题变；② `.strategy-page` 作用域优先级高于 `:root`，压过 `[data-theme="light"]`；③ 后续 1700+ 行全用 `var(--str-xxx)` → 颜色冻结在深色。
- **附带问题**：L39-L44 还自画了 3 层径向渐变背景 + `background-attachment: fixed`，与全局 `body::before` 氛围光叠加，策略页比其他页"更重"。

### 2.5 strategy-list.js 的 `CARD_ACCENT`（已知遗留）
[strategy-list.js L29-L34](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/strategy-list.js#L29-L34) 定义了卡片装饰渐变（硬编码 `#3b82f6` 等），通过 L189 `style="--str-card-accent: ${accent}"` 动态注入。
- 这些是**装饰性强调色**（卡片左侧光带），非语义色，主题切换时视觉差异可接受。
- JS 中无法直接引用 CSS 变量值（需 `getComputedStyle`，成本高、收益低）。
- **决策**：作为已知遗留保留，不强制改。在知识库里注明"JS 动态注入的装饰色允许字面量"。

### 2.6 theme.css / components.css 缺口
- theme.css **无** `--accent-green`：strategy.css 的 `--str-accent-green: #22c55e` 用于"成功/激活"语义，色值等同全局 `--fall-color` 但语义不同（避免"激活绿=跌"混淆），需新增独立 `--accent-green`。
- components.css **无** `.stat-strip`：统计条是跨页面高频复用模式，目前只在 factor-library.css 定义、strategy.css 又用 `--str-*` 重复一套，需提炼为全局标准。

### 2.7 知识库现状
[01-frontend.md](file:///d:/lcProject/stock-pulse/.trae/rules/stock-watcher/frontend/01-frontend.md)（649 行，九大章）内容大部分正确，但：
- 偏"深色专用"，把 light 描述为"预留"（L62），未把双主题作为一等公民。
- **缺**整体视觉风格的系统性总结（玻璃态、氛围光、顶部光带等散落在各处）。
- **缺**主题切换机制的独立章节（theme.js / data-theme / 平滑过渡）。
- **缺**"禁止页面级重定义基础 token"硬约束（正是 strategy.css 翻车的根因）。
- **缺**提炼后的公共组件清单（stat-strip）。

## 四、Assumptions & Decisions（假设与决策）

1. **class 名不改**：strategy.css 的 `str-stat-card`/`str-filter-bar`/`str-strategy-card` 等 class 保持不变，只改变量引用 → HTML 结构和 JS 零改动，最小风险。
2. **自定义属性名对齐全局**：`--str-stat-accent` → `--stat-accent`、`--str-c` → `--c`、`--str-card-accent` → `--card-accent`（与因子库和全局 stat-strip 标准对齐），需同步改 HTML 内联 + CSS 引用 + JS 注入。
3. **stat-strip 提炼不强制迁移**：在 components.css 建立全局标准，现有页面（因子库已有、策略页带 `str-` 前缀）不迁移，零回归。未来新页面直接用全局 `.stat-strip`。
4. **`--accent-green` 新增**：解决 `--str-accent-green`（成功/激活）与 `--fall-color`（跌）色值相同但语义冲突的问题。
5. **策略页自画背景删除**：复用全局 `body::before` 氛围光，消除"策略页背景比其他页重"的视觉差异。
6. **JS 中 `CARD_ACCENT` 硬编码渐变保留**（已知遗留）：装饰性强调色，主题切换时视觉差异可接受；JS 引用 CSS 变量成本高收益低。
7. **知识库增量补充为主**：保留原九大章节精华，开头新增视觉风格 + 主题系统两章，不整体重写（避免丢失正确细节）。
8. **验证用人工**：启动 watcher，访问策略三件套 + 因子库，切换主题按钮验证 dark/light 都正常。
