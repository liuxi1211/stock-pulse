## 三、改动方案（Proposed Changes）

### 决策摘要（来自用户确认）
- ✅ 列表 + 编辑器 + 版本 **三页全部对齐**
- ✅ 回测指标缺失字段 → **后端补字段 + 接口**
- ✅ 视觉风格 → **对齐原型深色 + 渐变**
- ✅ 深色主题范围 → **仅策略模块页面**（新增 strategy.css，不动全局 theme.css）
- ✅ 策略 ID → **不处理，保留原 ID**（不做 STR-yyyyMMdd-XXXX 格式化）
- ✅ 后端改动 → **前后端一起改**

### 改动分层
1. **CSS 层**：新增 `strategy.css`（深色主题 + 渐变 + 毛玻璃，仅策略页生效）
2. **后端层**：新增统计接口、扩展 DTO（回测指标）、补回测对比数据
3. **前端 HTML 层**：三页面结构调整
4. **前端 JS 层**：渲染逻辑适配新结构与新数据

### 改动 1：新增 `strategy.css`（视觉对齐核心）

**文件**：`stock-watcher/src/main/resources/static/css/strategy.css`（新建）

**职责**：实现原型 `prototype.css` 的深色主题系统，**仅作用于策略三页面**（通过页面根容器加 `.strategy-page` class 作用域隔离）。

**内容要点**：
- 作用域容器：`.strategy-page`（包在 list/editor/versions 的 `#strategyApp`/`#strategyEditor`/`#strategyVersions` 外层）
- 深色背景层：`--str-bg-primary: #0a0e17` / `--str-bg-card: rgba(21,29,43,0.7)` + `backdrop-filter: blur(10px)`
- 蓝青渐变：`--str-gradient-primary: linear-gradient(135deg, #3b82f6, #06b6d4)`
- 涨跌色（A 股涨红跌绿）：`--str-up: #ef4444` / `--str-down: #22c55e`
- 数据字体：`.str-mono { font-family: 'JetBrains Mono', 'SF Mono', Consolas, monospace; }`
- 卡片顶线渐变条：`.strategy-card::before` 2px 渐变
- 状态徽章：`.str-badge-active/verified/draft/archived`（半透明背景+描边+圆点）
- 分类徽章：`.str-cat-tech/fund/mixed/custom`
- 统计卡：`.str-stat-card`（彩色顶线+hover上移）
- 分页：`.str-page-btn`（胶囊+active渐变）
- 编辑器：Tab 图标彩色化、radio 卡片选中态、sticky 保存条、JSON 语法高亮（`.tok-key/.tok-str/.tok-num`）
- 版本页：双栏布局 `.vdiff-layout: 320px 1fr`、hunk 风格 diff 行

**引入方式**：三页面 `<head>` 后追加 `<link th:href="@{/css/strategy.css}" rel="stylesheet">`（或在 common.html head fragment 增加按页面条件引入）。**推荐在各页面单独加 link**，避免影响其他页面加载。

#### 2.1 新增 `/api/strategies/stats` 聚合接口

**文件**：

```
  "code": 200,
  "data": {
    "total": 12,
    "verified": 7,
    "active": 3,
    "draft": 2,
    "archived": 0
  }
}
```

**实现**：基于现有 `StrategyStatusEnum` 做 `GROUP BY status` 计数（Mapper 加一个查询，或 Service 内存聚合）。

#### 2.2 扩展 `StrategyDTO` 回测指标字段

**文件**：[StrategyDTO.java](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/java/com/arthur/stock/dto/strategy/StrategyDTO.java)

**新增字段**（来自最近一次回测结果，可空）：
```java
private Double lastReturnPct;      // 累计收益%
private Double lastSharpe;         // 夏普
private Double lastMaxDrawdownPct; // 最大回撤%
private String  lastBacktestTime;  // 最近回测时间
private Integer validationErrorCount; // 校验错误数（0=无）
```

**数据来源**：
- 若已有回测记录表（stock-engine 侧）→ 联查最近一次；
- 若无 → **本次先留空字段 + 前端显示"—"**，待回测中心打通后回填。
- `validationErrorCount`：可由 config_json 做 Schema 校验后得出（engine 侧已有校验逻辑），或暂默认 0。

> ⚠️ **此为后端改动量最大处**。若回测数据表尚未建立,本次仅加 DTO 字段(默认null)+接口返回,真实数据等回测中心对接。需用户知晓。

#### 2.3 版本对比回测指标（C6）

**文件**：`StrategyDiffDTO.java` 或新建 `StrategyVersionCompareDTO.java`

**接口**：`GET /api/strategies/{id}/versions/compare?from=v4&to=v5`
```json
{
  "metrics": [
    {"label":"total_return_pct","fromVal":27.4,"toVal":34.7,"delta":7.3},
    {"label":"sharpe_ratio","fromVal":1.42,"toVal":1.84,"delta":0.42}
  ]
}
```
**数据来源**：同 2.2，依赖回测记录。若无数据，返回空数组，前端表格区显示"暂无回测对比数据"。

## 七、风险与注意

1. **回测指标数据空窗**：DTO 字段加了但无数据源时，前端必须优雅降级显示"—"，不能报错或显示 NaN。
2. **strategy.css 作用域泄漏**：必须严格用 `.strategy-page` 前缀，避免污染全站。验收时重点检查其他页面。
3. **版本页重构幅度大**：从单列时间线 → 双栏 + 4 Tab，JS 逻辑近乎重写，需充分回归测试回滚功能。
4. **JSON 语法高亮性能**：大 config_json（>10KB）正则 tokenize 可能卡顿，需做防抖或限制大小。
5. **emoji 跨平台显示**：Windows/Mac/Linux 的 emoji 渲染不一致，可接受。
