# 策略管理页面两项问题修复计划

## 一、任务概要

策略管理「新增策略」页面存在两个问题需修复：

1. **选股范围 tab - 手动股票池录入改造**：当前「股票池=手动指定」时是一个裸 `<textarea>`（逗号/换行分隔录入代码），交互体验差。需改造为选股中心同款的「搜索建议 + 标签云」便捷输入模式。
2. **可视化条件录入 - 比较符下拉框深色主题配色修复**：在黑色主题下，条件树里的比较符（`>` `<` `>=` 等）`<select>` 下拉框文字看不清，配色不正确。

### 关键差异与约束

- **数据结构差异**：选股中心 `state.manualStocks` 是对象数组 `[{tsCode,code,name}]`；策略编辑器 `screen_config.stocks` 是字符串数组 `["600000.SH"]`。
  → **决策：保持策略侧字符串数组契约不变**，内部用对象数组维护以便展示，保存时只提取 tsCode。
- **SearchSuggest 已是通用组件**，可直接复用，无需重写。
- **比较符下拉框问题根因**：`.ctb-comparator-badge select` 用 `color: inherit` 且 `background: transparent`，`<option>` 用浏览器原生配色。深色主题下 badge 容器是深底浅字（正常），但 select 文字在某些层级 inherit 失效会回落到黑色，且 option 展开层是系统白底。
  → **决策：采用 CSS 显式配色**（给 select 与 option 显式写主题变量颜色）。

#### 1.2 strategy-editor.js 改造

**(a) 新增内部状态**（在构造或 init 时）：

**(b) 改 `applyUniverse`**（L1250）：除切换显隐外，初始化 SearchSuggest 绑定（懒初始化，只绑一次）：
```javascript

    if (wrap) wrap.style.display = value === 'manual' ? '' : 'none';
    if (value === 'manual') this._ensureManualStockSuggest();
```

**(c) 新增 `_ensureManualStockSuggest`**：绑定 SearchSuggest 到 `#f-stocks-search`，`onSelect` 回调里调 `addManualStock`：
```javascript
_ensureManualStockSuggest() {
    if (this._manualSuggestBound) return;
    const input = document.getElementById('f-stocks-search');
    if (!input || typeof SearchSuggest === 'undefined') return;
    new SearchSuggest(input, {
        onSelect: (item) => {
            this.addManualStock(item);
            input.value = '';
            input.focus();
        },
    });
    this._manualSuggestBound = true;
},
```
