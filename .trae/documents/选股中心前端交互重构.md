# 选股中心前端交互重构计��

## 概述

当前选股中心存在三大交互问题：① 用浏览器原生 `prompt`/`confirm` 弹窗不适配主题；② "运行时静默 PUT"是唯一保存路径，切换/刷新会丢失改动；③ 三栏常驻布局让中栏编辑区拥挤、新建会复用当前编辑器内容造成误解。

本次重构以 **"配置空间最大化 + 用户对保存完全掌控 + 弹窗主题统一"** 为核心理念，按四个方向落地。

涉及文件（仅前端，无后端改动）：
- [screener.html](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/templates/pages/screener.html) — 结构调整
- [screener.js](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/screener.js) — 交互逻辑
- [screener.css](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/css/screener.css) — 样式
- [common.js](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/common.js) — 新增 `StockApp.prompt()` 弹窗工厂（仅追加，不改现有 `confirm`/`alert`）

## 现状分析（Phase 1 探索结论）

### 已有可复用资产
- `StockApp.confirm({title,message,...})` 基于 Bootstrap Modal 的 Promise 风格弹窗（[common.js:156-225](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/common.js#L156-225)），返回 `boolean`。
- `StockApp.alert({title,message,level})` 同模式（[common.js:227+](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/common.js#L227)）。
- `StockApp.toast(msg, type)` 轻提示。
- `StockApp.get/post/put/delete` API 封装。
- CSS 主题变量：`--bg-card`、`--border-color`、`--accent-blue`、`--accent-cyan`、`--accent-red`、`--text-primary/secondary/muted`、`--bg-card-hover`。

### 当前痛点（已确认）
1. **新建**：[screener.js:128-145](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/screener.js#L128-145) 用 `window.prompt`，且 `screenConfig: collectConfig()` 会复用当前编辑器内容 → 用户误以为"新建=空白"。
2. **删除**：[screener.js:148](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/screener.js#L148) 用 `window.confirm`。
3. **保存**：无独立保存按钮；唯一持久化路径是 [runScreen()](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/screener.js#L554-586) 内的 PUT。切换方案 ([selectPlan](file:///d:/lcProject/stock-pulse/stock-watcher/src/main/resources/static/js/screener.js#L114-126)) 或刷新页面会丢改动。
4. **校验**：前端对条件树零结构校验，空 factor / NaN value / 零权重都能保存。
5. **布局**：三栏 `200px 1fr 300px` 常驻，中栏编辑器（尤其条件树叶子节点的因子下拉+参数+输出+比较器+右操作数）在 1280px 屏就拥挤。

## 决策汇总（来自 Phase 2/3）

| 维度 | 决策 |
|---|---|
| 布局 | **抽屉式结果面板**：左栏方案库 + 中栏编辑器常驻；结果改为运行时从右侧滑入的全屏抽屉 |

| 状态可视化 | **顶部状态条**：方案名 + 保存状态(已保存/未保存改动) + 最近保存时间 |

state.dirty = false;                  // 是否有未保存改动
state.lastSavedConfig = null;         // 上次保存/加载时的 config 快照（JSON 字符串）

// 监听编辑器变化（统一入口）
function markDirty() {
    if (!state.currentPlanId) return;  // 未选中方案不标脏
    state.dirty = true;
    updateStatusUI();
}
function clearDirty() {
    state.dirty = false;
    state.lastSavedConfig = JSON.stringify(collectConfig());
    state.lastSavedAt = Date.now();
    updateStatusUI();
}
function updateStatusUI() {
    const dot = document.querySelector('#planStatus .sw-status-dot');
    const text = document.querySelector('#planStatus .sw-status-text');
    const time = document.getElementById('planStatusTime');
    const saveBtn = document.getElementById('saveBtn');

    if (!state.currentPlanId) {
        dot.style.color = 'var(--text-muted)';
        text.textContent = '未选中方案';
        time.textContent = '';
        saveBtn.disabled = true;
        return;
    }
    const planName = document.getElementById('planName').value.trim() || '未命名方案';
    if (state.dirty) {
        dot.style.color = 'var(--accent-orange, #f97316)';
        text.textContent = planName + ' · 未保存改动';
        saveBtn.disabled = false;
        saveBtn.classList.add('btn-warning');  // 高亮
    } else {
        dot.style.color = 'var(--accent-cyan)';
        text.textContent = planName + ' · 已保存';
        saveBtn.classList.remove('btn-warning');
        saveBtn.disabled = true;   // 干净时禁用，避免重复保存
    }
    time.textContent = state.lastSavedAt
        ? '· ' + new Date(state.lastSavedAt).toLocaleTimeString('zh-CN', {hour:'2-digit',minute:'2-digit'})
        : '';
}

#### savePlan 函数（新增独立保存逻辑）

```javascript
async function savePlan() {
    if (!state.currentPlanId) {
        StockApp.toast('请先选中或新建方案', 'warning');
        return false;
    }
    const errors = validateConfig();
    if (errors.length) {
        await StockApp.alert({
            title: '保存失败',
            message: '请修正以下问题：\n' + errors.map((e,i)=>`${i+1}. ${e}`).join('\n'),
            level: 'warning'
        });
        return false;
    }
    return new Promise(resolve => {
        StockApp.put('/screener/plans/' + encodeURIComponent(state.currentPlanId), {
            name: document.getElementById('planName').value.trim() || '未命名方案',
            description: document.getElementById('planDesc').value.trim(),
            screenConfig: collectConfig(),
        }, function (resp) {
            if (resp.code === 200) {
                clearDirty();
                StockApp.toast('方案已保存', 'success');
                loadPlans();   // 同步左侧卡片的 updatedAt
                resolve(true);
            } else {
                StockApp.toast(resp.message || '保存失败', 'danger');
                resolve(false);
            }
        });
    });
}
```

async function createPlan() {
    const name = await StockApp.prompt({
        title: '新建选股方案',
        message: '将为新方案创建空白配置，创建后可在编辑器中配置条件。',
        placeholder: '请输入方案名称',

        confirmText: '创建',
    });
    if (!name) return;

    const desc = '';   // 不再复用 planDesc 输入框
    const body = {
        name: name.trim(),
        description: desc,
        screenConfig: defaultConfig(),   // ⭐ 空白配置，不再 collectConfig()
    };
    StockApp.post('/screener/plans', body, function (resp) {
        if (resp.code === 200 && resp.data) {
            StockApp.toast('方案已创建', 'success');
            state.currentPlanId = resp.data.id || resp.data;
            loadPlans();
            // 加载新方案并清空编辑器（确保从空白起步）
            selectPlan(state.currentPlanId);
        } else {
            StockApp.toast(resp.message || '创建失败', 'danger');
        }
    });
}

### 改动 6：基础结构校验（screener.js）

新增 `validateConfig()` 函数，保存/运行前调用，返回错误字符串数组（空数组=通过）。

```javascript
function validateConfig() {
    const errors = [];

    // 1. 条件树递归校验：每个叶子 factor 必填、value 不能 NaN
    function checkNode(node, pathDesc) {
        if ('operator' in node) {
            (node.conditions || []).forEach((c, i) => checkNode(c, `${pathDesc}[${i}]`));
            return;
        }
        // 叶子
        ['left', 'right'].forEach(side => {
            const op = node[side];
            if ('factor' in op) {
                if (!op.factor) {
                    errors.push(`条件 ${pathDesc} 的${side==='left'?'左':'右'}操作数未选择因子`);
                }
            } else {
                if (op.value == null || Number.isNaN(op.value)) {
                    errors.push(`条件 ${pathDesc} 的${side==='left'?'左':'右'}操作数值无效`);
                }
            }
        });
    }
    if (state.tree.conditions && state.tree.conditions.length) {
        checkNode(state.tree, '');
    }

    // 2. 排序打分校验
    const method = document.querySelector('input[name="rankMethod"]:checked').value;
    if (method === 'single') {
        if (!document.getElementById('rankSingleFactor').value) {
            errors.push('排序打分：单因子模式下未选择排序因子');
        }
    } else {
        const rows = document.querySelectorAll('#weightList .sw-cond-row');
        if (!rows.length) {
            errors.push('排序打分：复合加权模式下未添加任何权重因子');
        } else {
            let total = 0;
            let hasEmpty = false;
            rows.forEach(row => {
                const f = row.querySelector('.w-factor').value;
                const w = parseFloat(row.querySelector('.w-weight').value);
                if (!f) hasEmpty = true;
                if (!Number.isNaN(w)) total += Math.abs(w);
            });
            if (hasEmpty) errors.push('排序打分：存在未选择因子的权重行');
            if (total === 0) errors.push('排序打分：复合权重绝对值之和为 0，无法归一化');
        }
    }

    // 3. topN
    const topN = parseInt(document.getElementById('topN').value, 10);
    if (!topN || topN < 1) errors.push('排序打分：topN 必须为正整数');

    // 4. 上市天数
    const minDays = parseInt(document.getElementById('fMinListDays').value, 10);
    if (Number.isNaN(minDays) || minDays < 0) errors.push('静态过滤：最低上市天数需为非负整数');

    return errors;
}
```

**注意**：空条件树**不报错**（用户可能只用静态过滤+排序选股，这是合法场景）。校验只在叶子存在时检查完整性。
