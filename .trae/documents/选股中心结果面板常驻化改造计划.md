# 选股中心：右侧抽屉改造为常驻固定面板

## Summary

把选股中心（`screener.html`）运行选股后的「右侧滑出抽屉」改造为「页面最右侧常驻固定面板」：
- 工作区由当前的两栏（方案库 + 编辑器）扩展为三栏（方案库 + 编辑器 + 结果面板）。
- 结果面板初始即常驻显示空状态（运行前提示「配置条件后点击运行」）。
- 保留一个「折叠/展开」按钮，允许用户临时收起右侧面板腾出编辑区空间（替代原抽屉的关闭/ESC/遮罩交互）。
- 布局栅格沿用现有节奏：`220px 1fr 360px`，最小改动、复用现有主题变量与组件风格，不引入原型 `003` 的纸张/印章红配色。

改造后要求代码整洁：删除无用的抽屉专属样式与逻辑（mask、translateX、ESC 等），保持函数命名与现有命名风格（如 `openResultDrawer/closeResultDrawer` 重命名为 `expandResultPanel/collapseResultPanel`）一致。

## Current State Analysis

### 当前 HTML 结构（`templates/pages/screener.html`）
- `.screener-workspace`：grid 两栏 `220px 1fr`（左 `.sw-plan-rail` + 中 `.sw-editor`）。
- 运行控制条在 `.page-header > .page-actions.screener-runbar`（日期/候选池/保存/运行）。
- 抽屉：`<aside class="sw-result-drawer" id="resultDrawer">` 位于 `.content-area` 内，`position:fixed; right:0; width:min(560px,90vw); transform:translateX(100%)`，靠 `.open` 类切换。
- 配套遮罩：`<div class="sw-drawer-mask" id="drawerMask">`。
- 抽屉头部含「锁定」与「关闭（`bi-chevron-right`）」按钮。

### 当前 CSS（`static/css/screener.css`）
- `.screener-workspace { grid-template-columns: 220px 1fr; }`（行 98-103）。
- `.sw-result-*` 系列样式（summary/table 等）已存在，可被新面板直接复用。
- 抽屉专属样式集中在行 760-828：`.sw-result-drawer`（fixed/translateX）、`.sw-drawer-head`、`.sw-drawer-close`、`.sw-drawer-mask` 及其 `.open` 态。

### 当前 JS（`static/js/screener.js`）
- `openResultDrawer()` / `closeResultDrawer()`：操作 `#resultDrawer` / `#drawerMask` 的 `.open` 类与 `aria-hidden`。
- `bindDrawerEsc()`：监听 ESC 调用 `closeResultDrawer()`。
- `runProgress.start()` 内会调用 `openResultDrawer()` 展示骨架屏。
- `runScreen()` 成功后调用 `openResultDrawer()`。
- 导出对象含 `openResultDrawer, closeResultDrawer`（行 1295）。

### 原型参考（`.trae/specs/003-multi-factor-screener/prototype/`）
- `index.html`：`workspace` 为 `220px 1fr 360px` 三栏，右侧 `.result-panel` 常驻，内含空状态 `.result-empty` 与 `.result-content`（summary + tabs + views）。
- `styles.css`：行 1063+ 定义 `.result-panel` 常驻样式（`min-height:640px; display:flex; flex-direction:column`）。
- 用户已明确：**不照搬原型视觉**，仅借鉴「常驻三栏」布局结构。

## Proposed Changes

### 1. `templates/pages/screener.html`（结构调整）

**A. 把结果抽屉改造成工作区第三栏**

- 将 `<aside class="sw-result-drawer" id="resultDrawer">` 从 `.content-area` 末尾的游离位置，移入 `.screener-workspace` 内，作为第三个直接子元素（紧跟 `.sw-editor` 之后）。
- 删除抽屉外的 `<div class="sw-drawer-mask" id="drawerMask">`（连同其 `onclick`）。
- 类名重命名：`sw-result-drawer` → `sw-result-panel`（与原型命名一致，区别于旧抽屉），保留 `id="resultDrawer"`（JS 仍用此 id 取节点，减少 JS 改动面；如需更整洁可同步重命名为 `resultPanel`，但本次按「最小改动 + 整洁」取舍：id 保留，类名换新）。
- 抽屉头部 `.sw-drawer-head` → `.sw-panel-head`；关闭按钮 `.sw-drawer-close`（`bi-chevron-right`）→ 折叠按钮 `.sw-panel-toggle`（`bi-chevron-right` 表示「折叠」，折叠后由 JS 切换为 `bi-chevron-left`）。
- 锁定按钮保留，禁用态逻辑不变。

**B. 新增「展开」入口（折叠时恢复）**

- 在 `.screener-workspace` 上通过 CSS 类 `.result-collapsed` 切换栅格（见 CSS 改动）。
- 折叠后右侧面板不可见，需要在编辑器右上角或运行控制条附近放一个「显示结果」按钮（`bi-layout-three-columns` 或 `bi-chevron-left`），点击展开。最简方案：保留一个浮动的 `.sw-result-show` 小按钮（仅在 `.result-collapsed` 时可见），避免侵入现有运行控制条布局。

**C. 面板内容容器**

- 面板内部仍保留 `.sw-result-summary`（`#resultSummary`）与 `.sw-result-body`（`#resultBody`），`renderResult` / `runProgress` 渲染逻辑无需改动。
- 初始空状态文案：`<div class="sw-empty"><i class="bi bi-search"></i><div>配置条件后点击「运行选股」</div></div>`（已是当前默认内容，无需改）。

### 3. `static/js/screener.js`（交互改造）

**A. 重命名与逻辑替换**

- `openResultDrawer()` → `expandResultPanel()`：移除 mask 操作；改为 `workspace.classList.remove('result-collapsed')` + `panel.classList.remove('collapsed')` + `toggleBtn` 图标切回 `bi-chevron-right`。
- `closeResultDrawer()` → `collapseResultPanel()`：`workspace.classList.add('result-collapsed')` + `panel.classList.add('collapsed')` + 图标切 `bi-chevron-left`（用于浮动展开按钮）。
- `bindDrawerEsc()` → **删除**（不再需要 ESC 关闭；面板常驻）。
- `init()` 内 `bindDrawerEsc()` 调用一并删除。
- `runProgress.start()` 内的 `openResultDrawer()` → `expandResultPanel()`（行为一致：确保面板可见以展示骨架屏；若用户折叠了，运行时自动展开）。
- `runScreen()` 成功后的 `openResultDrawer()` → `expandResultPanel()`。
- 导出对象（行 1295）的 `openResultDrawer, closeResultDrawer` → `expandResultPanel, collapseResultPanel`（HTML 的 `onclick` 同步改名）。

**B. 新增绑定**

- 为 `#resultPanelToggle`（折叠按钮）绑定 `click` → `collapseResultPanel()`。
- 为 `.sw-result-show`（浮动展开按钮）绑定 `click` → `expandResultPanel()`。
- 这两个绑定在 `bindEvents()` 内完成。

**C. id 与类名整理**

- `#resultDrawer` 保留（JS 取节点用），仅类名换 `sw-result-panel`。
- `#drawerMask` 相关引用全部删除（HTML 已移除该节点）。

### 4. 清理与整洁性

- HTML：删除游离的 `<div class="sw-drawer-mask" id="drawerMask">` 行；抽屉头/关闭按钮的类名与图标更新到位。
- CSS：抽屉段（行 760-828）整段替换为新面板段；无重复或死代码。
- JS：
  - 删除 `bindDrawerEsc` 函数及其调用。
  - `openResultDrawer` / `closeResultDrawer` 全部改名，无残留。
  - 顶部 `e()` 转义、`StockApp` 等依赖不变。
- 不改动：条件树渲染、因子下拉、manual 候选股、保存/校验、运行进度三阶段反馈、锁定逻辑、tab 切换、脏数据监听等。

## Assumptions & Decisions

- **id 保留 `resultDrawer`**：减少 JS 改动面，但类名换新以表达语义；如审查时希望彻底整洁可同步改为 `resultPanel`，成本极低（仅 `getElementById` 字符串）。
- **折叠态用浮动按钮恢复**：避免侵入 `.page-actions.screener-runbar`（日期/候选池/保存/运行）的现有排布，对页面顶部布局零影响。
- **不照搬原型视觉**：用户明确「最小改动复用现有节奏」，故颜色/字体/印章按钮等原型特色元素不引入；仅借鉴三栏常驻结构。
- **空状态文案**：沿用当前默认文案「配置条件后点击『运行选股』」，与 `renderResult(null)` 分支一致。
- **响应式断点**：沿用现有 1280px / 992px 两个断点，不新增。
- **max-height 对齐**：左栏方案列表用 `calc(100vh - 220px)`，结果面板用 `calc(100vh - 160px)` 略高，保证头部运行栏与页头不被遮挡；落地时按视觉实测微调。

## Verification

1. **本地启动**：`mvn spring-boot:run`（或现有启动方式）后访问 `/screener`。
2. **初始态**：页面打开即见三栏；右侧面板显示「配置条件后点击运行」空状态。
3. **运行选股**：
   - 点击「运行选股」→ 按钮进入 loading，右侧面板立即显示骨架屏与计时器（`runProgress`）。
   - 接口返回后面板渲染命中表格；toast 提示命中数与耗时。
4. **折叠/展开**：
   - 点击面板头部「折叠」按钮 → 面板隐藏，右下出现浮动「显示结果」按钮。
   - 点击浮动按钮 → 面板恢复，浮动按钮消失。
   - 折叠态下再次运行选股 → 面板自动展开（`runProgress.start` 调用 `expandResultPanel`）。
5. **键盘**：ESC 不再关闭面板（已删除绑定）；其余快捷键不受影响。
6. **响应式**：
   - ≤1280px：三栏降为「左栏 180px + 编辑器」+ 结果面板下沉整行。
   - ≤992px：单栏堆叠，所有区块纵向排列。
7. **锁定**：结果返回后「锁定」按钮启用，点击仍能正常调用锁定接口。
8. **代码整洁**：全局搜索 `drawer`/`Drawer`/`drawerMask`/`bindDrawerEsc` 应无残留（仅可能在历史 git 记录中出现）。
