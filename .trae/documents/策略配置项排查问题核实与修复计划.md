## 二、Current State Analysis（关键代码现状）

### Bug #1 根因（compiler.py:749-764 `_build_position_ctx`）
```python
account = self.get_account()  # akquant 返回 dict，不含 entry_price / position_pnl_pct
pos.entry_price = float(account.get("entry_price", 0.0) or 0.0)  # 恒 0.0
pos.position_pnl_pct = float(account.get("position_pnl_pct", 0.0) or 0.0)  # 恒 0.0
```
akquant 正确 API：`self.position.entry_price`（strategy_position.py:32，内部走 `get_position_entry_price`）；`position_pnl_pct` 需自算 `(close - entry) / entry`。

`SELL_METHODS = {"close_position", "sell", "signal_based"}`（constants.py:32）已定义但从未在 validator 中引用。compiler.py:426 `raise CompilerError(...)` 不在 `on_bar` 的 `except ConditionEvalError`（compiler.py:786）内 → 整个回测崩溃。

### Bug #5（compiler.py:1115-1125）
```python
try:
    scores = rebalance_engine.select_at_rebalance_date(...)  # 内部 _apply_universe_filter 可能抛 BacktestError
except Exception:  # noqa: BLE001 - 选股失败不阻断回测
    return  # ← 把 PIT 强制的 BacktestError 也吞了
```

### Bug #6（compiler.py:1828-1832）

## 三、Proposed Changes（按文件）

### 修复 1：Bug #1 — `_build_position_ctx` 改用正��� API
**文件**：`stock-engine/services/backtest/compiler.py`（749-764 行）

**改动**：
- `entry_price`：`self.position.entry_price`（symbol 参数化，需用 `self.position` 的 symbol-aware 形态；akquant `self.position` 默认当前 bar symbol，跨 symbol 场景需通过 `get_position_entry_price(symbol)` —— 核实 akquant 是否暴露此方法；若否，对非当前 symbol 暂用 `get_account()` 兜底或仅支持当前 symbol）。
- `position_pnl_pct`：自算 `(current_close - entry) / entry`，current_close 取当前 bar close（`bar.close` 经 `_build_snapshot` 已在 ctx 内，或 `self.close`）。
- 保留 try/except 兜底（异常时留默认 0，不阻断）。

**实现要点**：
```python
def _build_position_ctx(self, symbol: str) -> PositionCtx:
    pos = PositionCtx()
    try:
        qty = float(self.get_position(symbol) or 0.0)
        pos.position_qty = qty
        if qty != 0:
            pos.bars_held = int(self.hold_bar() or 0)
            entry = self._get_entry_price(symbol)  # 新增辅助
            pos.entry_price = float(entry) if entry else None
            if entry and entry > 0:
                close = float(self.close)  # 当前 bar close
                pos.position_pnl_pct = (close - entry) / entry
    except Exception:
        pass
    return pos
```
`_get_entry_price(symbol)`：优先 `self.position.entry_price`（当前 symbol），其他 symbol 走 akquant 底层 `get_position_entry_price`（核实 0.2.47 是否可用；若不可用，多标的 bracket/exit 跨 symbol 场景降级，仅保证当前 symbol 正确）。

**验证步骤**：构造带 `exit.rules` 止损 5% 的策略回测，确认能触发；构造 bracket 策略，确认 bracket 单能下单。

### 修复 3：Bug #5 — PIT 强制失败不再被吞
**文件**：`stock-engine/services/backtest/compiler.py`（1115-1125 行 `on_daily_rebalance` 选股段）

**改动**：把 `BacktestError` 从 `except Exception` 中剥离，向上抛；其他真异常仍降级。

### 修复 4：Bug #6 — buffer_n 分别构造 buy/hold 不按 score 截断
**文件**：`stock-engine/services/backtest/compiler.py`（1822-1832 行）

**改动**：分别构造 buy_list（按 `selected[:buy_threshold]`，保持 score 降序）和 hold_list（`hold_set` 内按 score 降序），合并时**优先保留 hold**，总数限到 top_n 但 hold 不被 buy 挤出。
```python
buy_list = selected[:buy_threshold]
hold_candidates = [s for s in selected[buy_threshold:sell_threshold]
                   if s in current_positions and (current_positions.get(s, 0) or 0) > 0]
# buy 优先填，剩余名额给 hold；buffer 意图：hold 可占用 top_n 内名额不被挤
target_symbols_ordered = (buy_list + hold_candidates)[:top_n]
# 若 buy 已超 top_n（buy_threshold ≤ top_n 通常成立），hold 名义保留但不下单
```
保留诊断字段 `selected_count`。

**验证步骤**：构造 top_n=3, buffer_n=1，5 只标的轮动场景，确认已持仓低分股在缓冲带内不被挤出。

### 修复 5：W1+W2 — 保留权重的归一化
**文件**：`stock-engine/services/backtest/compiler.py`（W2: 1879-1896 段；W1: 1898-1919 段）

**改动思路**：保留权重后，对「非保留」标的的目标权重按比例缩放，使总和回到 investable。
- 先收集 kept 权重（跌停拒卖 + min_holding 保留），记 `kept_total = Σ kept_weight`。
- 剩余可分配额度 = `investable - kept_total`（clamp ≥ 0）。
- 对非 kept 标的的原始权重（步骤 4 算出的）按 `remaining / original_non_kept_sum` 缩放。

实现上：在步骤 4（权重分配）与步骤 5/6（保留）之间，先记录 `kept` dict，保留完后做一次归一化重缩放。或：保留段改为「仅标记不重新赋值」，步骤 7 之前统一归一化。

**最小侵入方案**：在步骤 6 之后、步骤 7 之前，新增「归一化」步骤：
```python
total = sum(target_weights.values())
if total > investable + 1e-9:
    scale = investable / total
    target_weights = {s: w * scale for s, w in target_weights.items()}
```
注意：此方案会让 kept 标的也被等比缩放（不是「保留原值」），但能消除 > 1 的杠杆风险，且数学确定。若要严格保留 kept 原值，则只缩放非 kept 部分。

**采用**：只缩放非 kept 部分（更贴合「保留」语义）。

**验证步骤**：构造 min_holding_bars=5、当前有持仓不满足周期的场景，确认 target_weights 总和 ≈ investable（≤ 1）。
