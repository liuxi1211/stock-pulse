# StockPulse 主要功能模块整体设计计划

> **范围**：stock-pulse「主要功能」大模块下 7 个子模块的统一规划，含现有 3 个模块（仪表盘 / 行情中心 / 自选股）的补齐，以及 3 个新增模块（板块行情 / 资金流向 / 个股诊断）的设计与开发。
> **K 线分析**：原占位菜单移除，K 线渲染能力合并到「个股诊断」页面的 Tab。
> **后端范围**：基于 **Tushare ≤2000 积分可调用接口** 接入（已按 `tushare 接口汇总.md` 核查，原计划 14 个新接口中 7 个不可用，已重构为 10 个可用接口，详见 §10 附录与 §4）。

---

## 1. 总体设计决策

### 1.1 页面融合 vs 独立的最终决策

**结论：6 个菜单项 + 1 个下钻详情页，不做融合。**

| 模块 | 形态 | 菜单项 | 路由 | 入口 |
|---|---|---|---|---|
| 仪表盘 | 独立页面 | ✅ 菜单项 | `/` | 总览入口 |
| 行情中心 | 独立页面 | ✅ 菜单项 | `/page/stock-list` | 全市场列表入口 |
| 自选股 | 独立页面 | ✅ 菜单项 | `/page/watchlist` | 用户专属管理 |
| 板块行情 | 独立页面 | ✅ 菜单项 | `/page/sector` | 行业板块视角（仅申万行业，无概念板块） |
| 资金流向 | 独立页面 | ✅ 菜单项 | `/page/moneyflow` | 主力资金视角（5 Tab，原 6 Tab 砍 1） |
| 个股诊断 | **下钻详情页** | ❌ 不上菜单 | `/page/stock-detail/{code}` | 从其他页面点击股票代码进入（4 Tab，原 5 Tab 砍 1） |
| ~~K 线分析~~ | **合并**到个股诊断的 Tab | ❌ 移除占位 | - | 不独立 |

**决策理由（老股民视角）**：
1. **板块/资金/个股是三个独立维度的视图**，强行融合会让页面臃肿且切换路径复杂。
2. **老股民的使用路径**是「列表/排行 -> 点击个股 -> 看详情做诊断」，所以「个股诊断」是从其他页面下钻的入口，不应放菜单。
3. **K 线分析不是独立功能**，老股民看票时一定会联动看 K 线 + 基本面 + 资金 + 板块，单独做一个 K 线页面太单薄。把 K 线作为「个股诊断」的一个 Tab 才合理。

### 1.2 接口能力约束下的模块裁剪

> 原计划使用了 7 个 tushare 不存在或需 >2000 积分的接口（`concept` / `concept_detail` / `moneyflow_dc` / `moneyflow_hsgt` / `hsgt_top10` / `stock_company` / `share_float`），核查后做如下裁剪与替代（详见 §10 附录）：

| 原功能 | 处理方式 | 替代方案 |
|---|---|---|
| 概念板块 Tab（concept / concept_detail） | ❌ 砍掉 | tushare 无此接口；板块行情只保留「申万一级行业」单 Tab |
| 板块资金流向（moneyflow_dc） | ❌ 砍掉 | 不引入派生聚合数据，保持数据单源 |
| 北向资金每日净买入（moneyflow_hsgt） | ✅ 替代 | 用 `hk_hold`（沪深股通持股明细，2000 起）做持股比例曲线；砍掉每日净买入柱状图与十大成交股表格 |
| 北向十大成交股（hsgt_top10） | ❌ 砍掉 | 无 ≤2000 积分替代接口 |
| F10 公司资料（stock_company） | ❌ 砍掉 | 个股诊断 Tab 4 整体删除，后续 tushare 增加低积分 F10 接口再补 |
| 限售解禁（share_float 需 3000 分） | ❌ 砍掉 | 风险提示 Tab 不含解禁明细 |
| 减持公告 | ✅ 补充 | 新增 `stk_holdertrade`（股东增减持，2000）填充 |
| 基本面股东人数 | ✅ 补充 | 新增 `stk_holdernumber`（股东人数，2000）填充 Tab 2 |

### 1.3 侧边栏最终结构

```
主要功能：
  - 仪表盘         (/)                          [已实现，补齐]
  - 行情中心       (/page/stock-list)           [已实现，补齐]
  - 自选股         (/page/watchlist)            [已实现，补齐]
  - 板块行情       (/page/sector)               [新增，仅行业 Tab]
  - 资金流向       (/page/moneyflow)            [新增，5 Tab]
（个股诊断 /page/stock-detail/{code} 不在菜单，从其他页面下钻）
量化研究：
  - 因子库 / 选股中心 / 策略管理 / 回测中心
```

### 4.3 工程落地步骤（按表执行）

每张新表需按 `02-tushare-integration-guide.md` 11 步流程：

1. 在 `schema-mysql.sql` + `schema-sqlite.sql` 同步加 CREATE TABLE
2. 在 `schema-mysql-comments.sql` 加 ALTER TABLE 注释

## 7. 假设与约束

1. **Tushare 积分约束**：本项目仅能调用 ≤2000 积分的接口。原计划 14 个新接口中，7 个不可用（`concept` / `concept_detail` / `moneyflow_dc` / `moneyflow_hsgt` / `hsgt_top10` / `stock_company` 不在 tushare 接口汇总表；`share_float` 需 3000 分）。详见 §10 附录。
2. **数据源单源性**：所有数据严格来自 tushare，不引入 akshare 或其他数据源，不引入派生聚合数据（如「个股 moneyflow 按行业聚合」视为派生数据，不采用）。
3. **北向资金替代**：用 `hk_hold`（沪深股通持股明细，2000 起）替代原 `moneyflow_hsgt` / `hsgt_top10`，仅做「持股比例曲线」+「十大重仓股聚合」，不做「每日净买入柱状图」。
4. **个股诊断 Tab 4 砍掉**：原「公司资料 F10」依赖 `stock_company` 接口（不存在），整体砍掉，页面顺延为 4 Tab。
5. **限售解禁砍掉**：`share_float` 需 3000 积分超限，风险提示 Tab 不含解禁明细；用 `stk_holdertrade`（股东增减持）部分替代「减持公告」项。
6. **schema 双库同步**：每张新表需同步改 `schema-mysql.sql` + `schema-sqlite.sql` + `schema-mysql-comments.sql` 三份。
7. **前端技术栈不变**：保持 Thymeleaf SSR + Bootstrap + ECharts + LightweightCharts，不引入 Vue/React。
8. **Tushare 拉取调度**：所有新增接口按各自频率（日线类日频、龙虎榜日频、财务季频）加入调度任务，避免高峰时段限流。
9. **个股诊断入口**：不在侧边栏菜单出现，从其他页面点击股票代码下钻进入；也支持用户在顶栏全局搜索框输入代码直接跳转。
10. **K 线分析菜单移除**：原 `fragments/common.html` 中的 K 线分析占位（`href="#"`）直接删除，不再保留。
11. **现有页面脚本抽出**：`stock-list.html` 和 `watchlist.html` 当前脚本内联，补齐时顺手抽成独立 `stock-list.js` / `watchlist.js`。

## 9. 总结

- **页面形态**：6 个独立菜单项 + 1 个下钻详情页（个股诊断），不做融合。

- **实施顺序**：Phase A（数据底座）-> Phase B（个股诊断）-> Phase C（板块行情）-> Phase D（资金流向）-> Phase E（现有模块补齐）-> Phase F（联调）。
