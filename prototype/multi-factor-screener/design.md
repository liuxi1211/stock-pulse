# 多因子选股中心 · 前端原型设计说明

## 1. 主题与目标

- **主题**：A 股量化研究场景下的「多因子选股中心」。
- **目标受众**：个人量化投资者、研究员。
- **核心目标**：把抽象的因子条件、过滤规则、排序权重转化为可拼装、可运行、可追溯的选股工作流。

## 2. 设计主张

把选股工作区设计成**「交易大厅的公告板 + 证券报纸的排版精度」**：

- 方案不是表格行，而是可以被翻看、锁定、贴标签的「卡片」。
- 规则树不是 JSON，而是像拼积木一样被组合的条件块。
- 结果不是普通列表，而是像行情终端一样实时滚动的命中名单。

这种纸质感与数据密度的对比，能让严肃的量化工具保留「研究手稿」的温度，同时不失专业感。

## 3. 设计令牌

### 色彩

| 名称 | 色值 | 用途 |
|---|---|---|
| `paper` | `#F7F5F0` | 页面主背景，模拟账本草稿纸 |
| `ink` | `#161823` | 主文字、关键标题 |
| `graphite` | `#6B7280` | 次要说明、网格线、占位符 |
| `seal-red` | `#C92A2A` | 主操作按钮、A 股上涨、选中态、印章效果 |
| `jade-green` | `#2F8E7A` | A 股下跌、通过/成功提示 |
| `highlight-gold` | `#D4A03C` | rank 徽章、高亮标记 |

### 字体

| 角色 | 字体 | 用途 |
|---|---|---|
| 展示字体 | `Noto Serif SC` | 页面大标题、方案名称、印章文字 |
| 正文字体 | `Noto Sans SC` | 表单标签、说明文案、按钮 |
| 数据字体 | `JetBrains Mono` | 股票代码、因子值、排名、涨跌幅 |

### 布局

**三栏指挥台**：左侧「方案与模式」、中间「规则构建画布」、右侧「结果与追踪」。

```text
+------------------------------------------------------------------+
|  StockPulse · 多因子选股中心     [日期] [候选池]      [运行]       |
+-----------+--------------------------------+---------------------+
| 选股模式  |                                |                     |
|  快照选股  |    规则树构建画布               |    结果预览          |
|  区间选股  |                                |                     |
|           |   [AND]                         |   命中 12 只         |
| 方案库    |   ├─ PE_TTM < 30                |   ┌────┬────┬────┐  |
|  低估值... |   └─ ROE_TTM > 10               |   │排名│代码│得分│  |
|  技术面... |                                 |   └────┴────┴────┘  |
| + 新建方案 |   [添加条件]  [添加分组]         |   [锁定] [追踪]      |
+-----------+--------------------------------+---------------------+
```

### 标志性元素

**印章运行按钮（Seal Button）**：「运行选股」被设计成一枚红色印章，点击后按钮短暂出现「钤印」动画，结果卡片角落出现一枚红色印章角标。它把「执行一次选股并落库」这个动作，隐喻为研究员在稿纸上盖下确认章——这是一个真实的决策瞬间，也是本页面最令人记忆深刻的元素。

## 4. 自我评审与风险点

- **评审**：最初考虑过深色终端风格（黑底+荧光绿），但这类风格在金融工具中过于常见，且压迫感强；也考虑过暖米色+赤陶色的「通用模板」组合，但与本主题关联度弱。最终选定「纸本草稿 + 印章红」既有 A 股文化符号（红涨绿跌、公章/签章），又能与普通 admin 仪表盘形成差异。
- **真实美学风险点**：印章红同时承担了「主按钮色」和「上涨色」。如果结果中大量出现红色上涨数据，可能与按钮色混淆。应对方案：按钮使用**实心印章**并带纹理/图标，数据涨跌通过**前缀箭头 + 文字标签**区分，避免仅靠颜色识别。

## 5. 适配与无障碍

- 响应式：宽屏三栏 → 中屏两栏（结果下沉） → 移动端单栏堆叠。
- 键盘焦点：所有交互元素有 `focus-visible`  outline（`ink` 色 2px）。
- 减少动效：`prefers-reduced-motion: reduce` 下关闭印章动画与渐显效果。
